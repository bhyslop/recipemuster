NOTE: This operation dispatches a container build via the vessel's
{rbtgr_vessel_trigger}
using the committed
{rbtgr_build_yaml}
in its
{rbtgr_vessel_dir}.
All substitution values are baked at inscribe time — zero custom overrides at dispatch.

NOTE: This operation supports concurrent execution per the Concurrent Session Safety invariant.
Each invocation dispatches an independent trigger execution with a unique `BUILD_ID`.
Concurrent builds of the same vessel produce independent images with different
{rbtga_consecration}
timestamps.

The
{rbtr_director}
dispatches a trigger-based remote container build:

==== Substitution Variable Inventory

All `_RBGY_*` substitution variables are baked into
{rbtgr_build_yaml}
at inscribe time by
{rbtgo_rubric_inscribe}.
Cloud Build resolves these from the committed YAML — no values are overridden at dispatch.

*Vessel identity*::
[cols="1,2"]
|===
| `_RBGY_MONIKER` | Vessel sigil. Source: {rbrv_sigil}.
| `_RBGY_DOCKERFILE` | Dockerfile path relative to build context. Source: {rbrv_conjure_dockerfile}.
| `_RBGY_PLATFORMS` | Target platforms. Source: {rbrv_conjure_platforms}.
|===

*Registry coordinates*::
[cols="1,2"]
|===
| `_RBGY_GAR_LOCATION` | Artifact Registry region. Source: {rbrr_gcp_region}.
| `_RBGY_GAR_PROJECT` | Depot project ID. Source: {rbrr_depot_project_id}.
| `_RBGY_GAR_REPOSITORY` | GAR repository name. Source: {rbrr_gar_repository}.
| `_RBGY_GAR_HOST_SUFFIX` | GAR Docker host suffix. Source: RBGC constant.
| `_RBGY_ARK_SUFFIX_IMAGE` | Image artifact tag suffix. Source: RBGC constant.
| `_RBGY_ARK_SUFFIX_ABOUT` | Metadata artifact tag suffix. Source: RBGC constant.
| `_RBGY_CRANE_TAR_GZ` | Crane release tarball URL. Source: RBGC constant.
|===

*Provenance*::
[cols="1,2"]
|===
| `_RBGY_GIT_COMMIT` | Main repo HEAD SHA at inscribe time.
| `_RBGY_GIT_BRANCH` | Main repo branch at inscribe time.
| `_RBGY_GIT_REPO` | Main repo identifier (owner/repo).
| `_RBGY_RUBRIC_REPO` | {rbtgr_rubric_repo} URL. Filled after {rbtgr_rubric_repo} push.
| `_RBGY_RUBRIC_COMMIT` | {rbtgr_rubric_repo} HEAD SHA after push. Filled after {rbtgr_rubric_repo} push.
| `_RBGY_INSCRIBE_TIMESTAMP` | Inscribe timestamp (`YYYYMMDD_HHMMSS`). Used by rbgjb01 to construct {rbtga_consecration}.
|===

Builder image references (step `name:` fields) are resolved to digest-pinned URIs from regime pins at inscribe time.
These are inlined directly into YAML — not expressed as substitution variables.

==== Script Inlining

Step scripts (`rbgjb/*.sh`) are the source of truth for build logic.
At inscribe time,
{rbtgo_rubric_inscribe}
reads each script and inlines its content into the corresponding YAML `script:` field.

Developers edit `rbgjb/*.sh` in the main repo, then run
{rbtgo_rubric_inscribe}
to regenerate
{rbtgr_build_yaml}
with the updated content.
The committed YAML in the main repo is the reviewed artifact;
the {rbtgr_rubric_repo} copy is synced by inscribe.

==== Tag Construction

Each build produces a
{rbtga_consecration}
with dual timestamps:

`i«INSCRIBE»-b«BUILD»`

* `i«YYYYMMDD_HHMMSS»` — inscribe timestamp, baked as `_RBGY_INSCRIBE_TIMESTAMP`
* `b«YYYYMMDD_HHMMSS»` — build timestamp, derived at build time by `rbgjb01-derive-tag-base.sh`

Example: `i20260224_153022-b20260224_160530`

Step `rbgjb01` constructs: `TAG_BASE="${_RBGY_INSCRIBE_TIMESTAMP}-b$(date -u +%Y%m%d_%H%M%S)"`

Complete artifact tags combine vessel moniker with consecration:

* {rbtga_ark_image}: `«MONIKER»:«TAG_BASE»«ARK_SUFFIX_IMAGE»`
* {rbtga_ark_about}: `«MONIKER»:«TAG_BASE»«ARK_SUFFIX_ABOUT»`

Multiple builds of the same inscribed YAML sort together by inscribe prefix,
each distinguished by its build timestamp.
