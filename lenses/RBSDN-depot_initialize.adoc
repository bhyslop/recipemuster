The
{at_operator}
completes one-time depot initialization after
{rbtgo_depot_create}
has provisioned infrastructure.
This procedure validates the
{rbra_rubric_repo_url}
for {rbtgr_rubric_repo} management and completes the GitHub OAuth authorization for the
{rbtgi_source_connection}
created by
{rbtgo_depot_create}
in PENDING state.

This procedure is idempotent — safe to re-run.
Steps already complete (URL already valid, connection already COMPLETE) are skipped.

// ⟦axs_inputs axd_none⟧

// ⟦axs_preconditions⟧

* {rbtgi_depot}
exists (completed
{rbtgo_depot_create})
* {rbrr_gdc_connection_name} and {rbrr_gdc_region} configured in RBRR
* {rbrr_director_rbra_file} contains {rbra_rubric_repo_url}
* Operator has browser access for GitHub OAuth authorization

// ⟦axs_behavior⟧

. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}

. *Load Rubric Repo URL*:
.. {rbbc_call}
{rbtoe_rbra_load}
for
{rbrr_director_rbra_file}
.. {rbbc_store}
`«RUBRIC_REPO_URL»` from {rbra_rubric_repo_url}
.. If `«RUBRIC_REPO_URL»` is empty:
... {rbbc_show}
"RBRA_RUBRIC_REPO_URL is not set in Director RBRA file"
... {rbbc_show}
guidance: "Create a repository, generate a PAT with Contents:write, construct the authenticated URL as https://x-access-token:<PAT>@<host>/<org>/<repo>.git, and add RBRA_RUBRIC_REPO_URL to the Director RBRA file."
... {rbbc_fatal}
"Set RBRA_RUBRIC_REPO_URL in Director RBRA file and re-run"

. *Validate Rubric Repo URL*:
.. {rbbc_call}
`git ls-remote «RUBRIC_REPO_URL» HEAD` to verify reachability and credential validity
.. If `git ls-remote` fails:
... {rbbc_fatal}
"Rubric repo URL is unreachable or credentials are invalid — check RBRA_RUBRIC_REPO_URL and ensure PAT has Contents:write permission"
.. {rbbc_show}
"Rubric repo URL validated"

. *Check Developer Connect Connection Status*:
.. Construct connection path: `projects/{rbrr_depot_project_id}/locations/{rbrr_gdc_region}/connections/{rbrr_gdc_connection_name}`
.. {rbbc_call}
link:https://cloud.google.com/developer-connect/docs/reference/rest/v1/projects.locations.connections/get[developerconnect.connections.get] for `«CONNECTION_PATH»`
.. {rbbc_require}
HTTP 200 — connection exists
.. {rbbc_store}
`«CONNECTION_STAGE»` from `.installationState.stage`
.. If `«CONNECTION_STAGE»` is `COMPLETE`:
... {rbbc_show}
"Developer Connect connection already active — skipping OAuth step"
... Go to step 7

. *Guide GitHub OAuth Authorization*:
.. {rbbc_store}
`«ACTION_URI»` from `.installationState.actionUri` of the connection response
.. {rbbc_require}
`«ACTION_URI»` is present and non-empty
.. {rbbc_show}
"Developer Connect connection requires GitHub OAuth authorization"
.. {rbbc_show}
"Open this URL in your browser to authorize the Google Developer Connect app:"
.. {rbbc_show}
`«ACTION_URI»`
.. {rbbc_show}
"After clicking 'Authorize' on the GitHub consent page, return here and press Enter"
.. Wait for operator confirmation

. *Poll Connection Until Active*:
.. {rbbc_poll}
link:https://cloud.google.com/developer-connect/docs/reference/rest/v1/projects.locations.connections/get[developerconnect.connections.get] for `«CONNECTION_PATH»`
.. Extract `.installationState.stage` from each response
.. Poll at `RBGC_EVENTUAL_CONSISTENCY_SEC` intervals until stage reaches `COMPLETE` or `RBGC_MAX_CONSISTENCY_SEC` timeout
.. {rbbc_require}
`installationState.stage` is `COMPLETE`
.. {rbbc_fatal}
if timeout: "Developer Connect OAuth did not complete — re-open the authorization URI and try again"

. *Final Validation*:
.. {rbbc_show}
"Running final validation..."
.. Verify {rbtgr_rubric_repo} URL: repeat `git ls-remote` call — {rbbc_require} successful exit
.. Verify Developer Connect: repeat connections.get call — {rbbc_require} `installationState.stage` is `COMPLETE`
.. {rbbc_show}
"Depot initialization complete"

// ⟦axs_outputs⟧

The outputs of depot initialization are:

- Validated {rbra_rubric_repo_url} — confirmed reachable via `git ls-remote`
- Active {rbtgi_source_connection} — `installationState.stage` is `COMPLETE`
- OAuth token automatically stored in Secret Manager (managed by Developer Connect, not by operator)

// ⟦axs_completion⟧

Successful execution confirms the depot is fully initialized for
{rbtgo_rubric_inscribe}
operations.
The
{rbra_rubric_repo_url}
enables {rbtgr_rubric_repo} management and the
{rbtgi_source_connection}
enables Cloud Build to access the {rbtgr_rubric_repo}.
Proceed to
{rbtgo_rubric_inscribe}
to set up per-vessel build infrastructure.
