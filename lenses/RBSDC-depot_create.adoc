//axhob_operation
{rbtgo_depot_create}

NOTE: This operation creates an
{rbtgi_depot}
- a cohesive unit of Recipe Bottle infrastructure including the depot project, build bucket, container repository, Mason service account, and
{rbtgi_cbv2_connection}
that share their productive lifecycle.

The
{rbtr_payor}
creates a new depot infrastructure from the
{rbtgi_payor_project}:

//axhop_parameter_from_type
{rbtgi_depot}
. *Depot Name*:
.. {rbbc_require}
`«INPUT_DEPOT_NAME»` provided as first argument
.. Validate depot name: lowercase letters, numbers, hyphens only, max 20 chars

//axhos_step
. *Validate Rubric Repo URL (GitLab required)*:
.. {rbbc_require}
{rbrr_rubric_repo_url}
is set in regime (`RBRR_RUBRIC_REPO_URL` in `rbrr.env`) or {rbbc_show} tabtarget hint for `rbw-Pgl` then {rbbc_fatal} "Cannot proceed without GitLab rubric repo"
.. {rbbc_require}
URL begins with `https://gitlab.com/` or {rbbc_show} tabtarget hint for `rbw-Pgl` (GitLab setup guide) then {rbbc_fatal} "Cannot proceed without GitLab rubric repo"

//axhop_parameter_from_stdin
. *GitLab Project Access Token (stdin)*:
.. {rbbc_show}
rubric repo URL
.. Read one line from stdin: `«GITLAB_TOKEN»`
.. If empty, {rbbc_show} tabtarget hint for `rbw-Pgl` (GitLab setup guide) then {rbbc_fatal} "Token is empty"

//axhos_step
. *Validate Rubric Repo URL reachability*:
.. Construct authenticated URL using bash builtin: `https://oauth2:«GITLAB_TOKEN»@` + URL path from
{rbrr_rubric_repo_url}
.. {rbbc_call}
`git ls-remote «AUTHENTICATED_URL» HEAD` to verify reachability and credential validity, stderr captured to temp file
.. {rbbc_require}
successful exit or {rbbc_fatal} "Rubric repo URL unreachable or token invalid — see «stderr_file»"
.. Discard `«AUTHENTICATED_URL»` from memory (token must not persist)

//axhos_step
. *Authenticate as
{rbtr_payor}*:
.. Execute
{rbtoe_payor_authenticate}

//axhos_step
. *Generate Depot Project ID*:
.. {rbbc_store}
`«TIMESTAMP»` as current time in YYYYMMDDHHmm format
.. Generate project ID: `rbw-«INPUT_DEPOT_NAME»-«TIMESTAMP»`
.. {rbbc_store}
as `«DEPOT_PROJECT_ID»`
.. {rbbc_require}
total length ≤ 30 characters

//axhos_step
. *Create Depot Project*:
.. {rbbc_submit}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/create[projects.create] with:
... projectId: `«DEPOT_PROJECT_ID»`
... displayName: `RB Depot: «INPUT_DEPOT_NAME»`
... parent: `{rbrp_parent_type}s/{rbrp_parent_id}` (if parent_type != `none`)
.. {rbbc_await}
returned operation
.. {rbbc_require}
project creation successful

//axhos_step
. *Link Billing Account*:
.. {rbbc_call}
link:https://cloud.google.com/billing/docs/reference/rest/v1/projects/updateBillingInfo[cloudbilling.projects.updateBillingInfo] for `«DEPOT_PROJECT_ID»`
.. Set billingAccountName: `billingAccounts/{rbrp_billing_account_id}`
.. {rbbc_require}
HTTP 200 response confirming billing linkage

//axhos_step
. *Get Depot Project Number*:
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/get[projects.get] for `«DEPOT_PROJECT_ID»`
.. {rbbc_store}
`«PROJECT_NUMBER»` from `.name` field (CRM v3 returns `projects/{number}`)
.. {rbbc_require}
valid numeric project number after extracting from name

//axhos_step
. *Enable Depot Project APIs*:
.. For each service in [`artifactregistry`, `cloudbuild`, `containeranalysis`, `iam`, `secretmanager`, `serviceusage`, `storage`]:
... Execute
{rbtoe_api_enable}
with service name and `«DEPOT_PROJECT_ID»`
.. {rbbc_require}
all APIs enabled successfully

//axhos_step
. *Verify IAM Propagation*:
.. NOTE: After API enablement, IAM permissions may not be immediately available due to eventual consistency.
.. {rbbc_poll}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories/list[artifactregistry.repositories.list] for `«DEPOT_PROJECT_ID»` in `{rbrr_gcp_region}`
.. Poll at `RBGC_EVENTUAL_CONSISTENCY_SEC` intervals until HTTP 200 or `RBGC_MAX_CONSISTENCY_SEC` timeout
.. {rbbc_require}
HTTP 200 response confirming IAM propagation complete
.. {rbbc_fatal}
if timeout: "IAM propagation timeout - artifactregistry API not accessible"

//axhos_step
. *Create Build Bucket*:
.. Construct bucket name: `rbw-«INPUT_DEPOT_NAME»-bucket`
.. {rbbc_call}
link:https://cloud.google.com/storage/docs/json_api/v1/buckets/insert[storage.buckets.insert] with:
... name: `rbw-«INPUT_DEPOT_NAME»-bucket`
... location: `{rbrr_gcp_region}`
... project: `«DEPOT_PROJECT_ID»`
... lifecycle rule: delete objects after 1 day
.. {rbbc_store}
`«BUILD_BUCKET»` as created bucket name

//axhos_step
. *Create Container Repository*:
.. Construct repository name: `rbw-«INPUT_DEPOT_NAME»-repository`
.. {rbbc_submit}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories/create[artifactregistry.repositories.create] with:
... repositoryId: `rbw-«INPUT_DEPOT_NAME»-repository`
... format: `DOCKER`
... location: `{rbrr_gcp_region}`
... parent: `projects/«DEPOT_PROJECT_ID»/locations/{rbrr_gcp_region}`
.. {rbbc_await}
returned operation
.. {rbbc_store}
`«GAR_REPOSITORY»` as created repository name

//axhos_step
. *Create
{rbtr_mason}
Service Account*:
.. NOTE: The
{rbtr_mason}
replaces Cloud Build's implicit service account to provide deterministic permissions and predictable authorization within the depot infrastructure.
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/create[iam.serviceAccounts.create] with:
... accountId: `mason-«INPUT_DEPOT_NAME»`
... displayName: `Mason for RB Depot: «INPUT_DEPOT_NAME»`
... project: `«DEPOT_PROJECT_ID»`
.. {rbbc_store}
`«MASON_EMAIL»` from response `.email` field

//axhos_step
. *Configure
{rbtr_mason}
Permissions*:
.. {rbbc_call}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories/setIamPolicy[repositories.setIamPolicy] for `«GAR_REPOSITORY»`
... Add `serviceAccount:«MASON_EMAIL»` with `roles/artifactregistry.writer`
.. {rbbc_call}
link:https://cloud.google.com/storage/docs/json_api/v1/buckets/setIamPolicy[buckets.setIamPolicy] for `«BUILD_BUCKET»`
... Add `serviceAccount:«MASON_EMAIL»` with `roles/storage.objectViewer`
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/setIamPolicy[projects.setIamPolicy] for `«DEPOT_PROJECT_ID»`
... Add `serviceAccount:«MASON_EMAIL»` with `roles/viewer`
... Add `serviceAccount:«MASON_EMAIL»` with `roles/logging.logWriter`

//axhos_step
. *Enable Cloud Build service agent to impersonate
{rbtr_mason}*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/setIamPolicy[serviceAccounts.setIamPolicy] for `«MASON_EMAIL»`
.. Add member: `serviceAccount:service-«PROJECT_NUMBER»@gcp-sa-cloudbuild.iam.gserviceaccount.com`
.. Grant role: `roles/iam.serviceAccountTokenCreator`
.. NOTE: Cloud Build service agent is Google-provisioned, not newly created.
.. {rbbc_require}
IAM binding successful (enables Cloud Build to execute as Mason)

//axhos_step
. *Store GitLab credentials in Secret Manager (3 secrets)*:
.. Generate `«WEBHOOK_SECRET»` as random UUID via `uuidgen` (output to temp file, read with bash builtin)
.. Base64-encode `«GITLAB_TOKEN»` (used for both api and read_api secrets)
.. Base64-encode `«WEBHOOK_SECRET»`
.. For each secret (`RBGC_CBV2_API_TOKEN_SECRET_NAME`, `RBGC_CBV2_READ_TOKEN_SECRET_NAME`, `RBGC_CBV2_WEBHOOK_SECRET_NAME`):
... {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets/create[secrets.create] with:
.... parent: `projects/«DEPOT_PROJECT_ID»`
.... secretId: secret name constant
.... body: `{"replication": {"automatic": {}}}`
... {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets.versions/add[secrets.versions.add] with appropriate base64 payload
... {rbbc_store}
`«SECRET_VERSION»` from response `.name` field
.. NOTE: api and read_api secrets store the same token value. Webhook secret is auto-generated UUID. The GitLab project access token must have both `api` and `read_api` scopes — CB v2 verifies `read_api` independently.

//axhos_step
. *Grant Cloud Build service agent Secret Manager access (all 3 secrets)*:
.. Construct service agent: `service-«PROJECT_NUMBER»@gcp-sa-cloudbuild.iam.gserviceaccount.com`
.. For each of the 3 secrets:
... {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets/getIamPolicy[secrets.getIamPolicy] — must succeed (HTTP 200) before modification. Die on failure; never fall back to empty policy.
... Add Cloud Build service agent to `roles/secretmanager.secretAccessor` in the existing policy (read-modify-write)
... {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets/setIamPolicy[secrets.setIamPolicy] with the updated policy
.. {rbbc_require}
all IAM bindings successful — existing bindings preserved

//axhos_step
. *Grant Cloud Build service agent connection admin*:
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/setIamPolicy[projects.setIamPolicy] for `«DEPOT_PROJECT_ID»`
.. Add `serviceAccount:service-«PROJECT_NUMBER»@gcp-sa-cloudbuild.iam.gserviceaccount.com` with `roles/cloudbuild.connectionAdmin`
.. {rbbc_require}
IAM binding successful

//axhos_step
. *Create CB v2 GitLab Connection*:
.. Construct connection name: `rbw-«INPUT_DEPOT_NAME»«RBGC_CBV2_CONNECTION_SUFFIX»` (suffix is `-gitlab`)
.. {rbbc_call}
link:https://cloud.google.com/build/docs/api/reference/rest/v2/projects.locations.connections/create[connections.create] with:
... parent: `projects/«DEPOT_PROJECT_ID»/locations/{rbrr_gcp_region}`
... connectionId: `rbw-«INPUT_DEPOT_NAME»-gitlab`
... body:
+
----
{
  "gitlabConfig": {
    "authorizerCredential": {
      "userTokenSecretVersion": "«API_TOKEN_SECRET_VERSION»"
    },
    "readAuthorizerCredential": {
      "userTokenSecretVersion": "«READ_TOKEN_SECRET_VERSION»"
    },
    "webhookSecretSecretVersion": "«WEBHOOK_SECRET_VERSION»"
  }
}
----
.. {rbbc_await}
returned {gcp_lro}
.. {rbbc_store}
`«CBV2_CONNECTION_NAME»` as `rbw-«INPUT_DEPOT_NAME»-gitlab`
.. {rbbc_poll}
link:https://cloud.google.com/build/docs/api/reference/rest/v2/projects.locations.connections/get[connections.get] until `installationState.stage` reaches `COMPLETE`
.. *Retry with backoff on IAM propagation failure*:
The connection creation LRO serves as the IAM propagation gate (see RBSCIP).
If the Cloud Build Service Agent cannot yet access the secrets (grant not propagated),
the LRO fails.  On failure:
... Delete the zombie connection resource via
link:https://cloud.google.com/build/docs/api/reference/rest/v2/projects.locations.connections/delete[connections.delete]
(a failed LRO leaves a non-functional connection that blocks re-creation with the same name)
... Wait per backoff profile (initial 3s, 2x multiplier, cap 20s, deadline 420s)
... Retry `connections.create` from the top of this step
.. {rbbc_require}
connection is COMPLETE or {rbbc_warn} diagnostic checklist ({rbbc_fatal} after):
... GitLab project access token has both `api` and `read_api` scopes
... Token is a project access token (not personal)
... RBRR_RUBRIC_REPO_URL points to correct GitLab project

//axhos_step
. *Create CB v2 Repository*:
.. {rbbc_call}
link:https://cloud.google.com/build/docs/api/reference/rest/v2/projects.locations.connections.repositories/create[repositories.create] with:
... parent: `projects/«DEPOT_PROJECT_ID»/locations/{rbrr_gcp_region}/connections/«CBV2_CONNECTION_NAME»`
... repositoryId: `«RBGC_CBV2_REPOSITORY_ID»` (constant, currently `rubric`)
... body: `{"remoteUri": "«RBRR_RUBRIC_REPO_URL»"}`
.. {rbbc_await}
returned {gcp_lro}
.. {rbbc_require}
repository created successfully

//axhoo_output_of_type
{rbtgi_depot}
. *Display Depot Configuration*:
.. {rbbc_show}
depot creation successful
.. {rbbc_show}
required RBRR configuration values:
... `RBRR_DEPOT_PROJECT_ID=«DEPOT_PROJECT_ID»`
... `RBRR_GCP_REGION={rbrr_gcp_region}`
... `RBRR_GAR_REPOSITORY=«GAR_REPOSITORY»`
... `RBRR_CBV2_CONNECTION_NAME=«CBV2_CONNECTION_NAME»`
.. {rbbc_show}
Mason service account: `«MASON_EMAIL»`
.. {rbbc_show}
CB v2 connection `«CBV2_CONNECTION_NAME»` active with GitLab
.. {rbbc_show}
GitLab token stored in Secret Manager (3 secrets: api, read_api, webhook)

//axhoo_output_of_type
{rbtgi_depot}
. *Update Depot Tracking*:
.. Execute
{rbtoe_depot_list_update}
to refresh depot project tracking

//axhoc_completion
Successful execution creates depot infrastructure with configured build bucket, container repository, Mason service account, CB v2 GitLab connection (COMPLETE), and CB v2 repository linking to the
{rbtgr_rubric_repo}.
Proceed to
{rbtgo_governor_reset}
to create the
{rbtr_governor}
for this depot.
