//axhob_operation
{rbtgo_director_create}

The
{rbtr_governor}
creates a
{rbtr_director}
{giam_service_account}
with build management permissions:

//axhoq_precondition
* Execute {rbtoe_check_rubric_repo_url} (validates {rbrr_rubric_repo_url} using PAT from Secret Manager)

//axhoq_precondition
* Execute {rbtoe_check_cbv2_connection} to verify {rbtgi_cbv2_connection} is active

//axhop_parameter_from_type
{rbtr_director}
. *Validate Input Parameters*:
.. {rbbc_require}
`«INPUT_INSTANCE»` provided as first argument
.. {rbbc_store}
`«DIRECTOR_ACCOUNT_NAME»` as `director-«INPUT_INSTANCE»`
.. {rbbc_store}
`«DIRECTOR_EMAIL»` as `«DIRECTOR_ACCOUNT_NAME»@{rbrr_depot_project_id}.iam.gserviceaccount.com`

//axhoq_precondition
* {rbbc_require}
`BURD_OUTPUT_DIR` environment variable set and directory exists

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` for subsequent API calls

//axhos_step
. *Create
{rbtr_director}
Service Account*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/create[iam.serviceAccounts.create] with:
... accountId: `«DIRECTOR_ACCOUNT_NAME»`
... displayName: `Recipe Bottle Director («INPUT_INSTANCE»)`
... description: `Create/destroy container images for «INPUT_INSTANCE»`
... project:
{rbrr_depot_project_id}
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«DIRECTOR_UNIQUE_ID»` from response `.uniqueId` field

//axhos_step
. *Verify Service Account Propagation*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/get[iam.serviceAccounts.get] for `projects/-/serviceAccounts/«DIRECTOR_EMAIL»`
.. {rbbc_poll}
with backoff until HTTP 200 (newly-created SA may not be immediately resolvable by email)
.. NOTE: Poll must use the email address, not the numeric ID. The email-based lookup path has independent propagation from the UID-based path, and subsequent steps (keys.list, keys.create) all use the email address. Polling by UID can succeed while the email path still returns 404.
.. {rbbc_require}
service account accessible by email

//axhos_step
. *Preflight: Ensure Pristine Key State*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/list[serviceAccounts.keys.list] for `«DIRECTOR_EMAIL»`
.. {rbbc_store}
`«EXISTING_KEYS»` from response `.keys[]`
.. {rbbc_require}
no keys with `keyType: USER_MANAGED` exist
.. {rbbc_fatal}
if USER_MANAGED keys found: "Manual cleanup required via Console"

//axhos_step
. *Generate Service Account Key*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/create[serviceAccounts.keys.create] for `«DIRECTOR_EMAIL»` with:
... privateKeyType: `TYPE_GOOGLE_CREDENTIALS_FILE`
.. {rbbc_store}
`«DIRECTOR_PRIVATE_KEY_DATA»` from response `.privateKeyData` field
.. {rbtoe_rbra_generate}
converts `«DIRECTOR_PRIVATE_KEY_DATA»` to `«DIRECTOR_RBRA_FILE»`

//axhos_step
. *Get Depot Project Number*:
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/get[projects.get] for
{rbrr_depot_project_id}
.. {rbbc_store}
`«PROJECT_NUMBER»` from `.projectNumber` field

//axhos_step
. *Grant Project-Level Permissions*:
.. {rbtoe_iam_grant_project} for
{rbrr_depot_project_id}
.. Add `serviceAccount:«DIRECTOR_EMAIL»` with roles:
... `roles/cloudbuild.builds.editor` (submit and manage builds)
... `roles/viewer` (read project resources)
.. Policy version: 3
.. {rbbc_require}
IAM policy update successful

//axhos_step
. *Grant Secret Manager Access for Inscribe*:
.. NOTE:
{rbtr_director}
needs to fetch the API token from Secret Manager during
{rbtgo_rubric_inscribe}
to construct authenticated git URLs.
.. {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets/getIamPolicy[secrets.getIamPolicy] for `projects/{rbrr_depot_project_id}/secrets/RBGC_CBV2_API_TOKEN_SECRET_NAME`
.. {rbbc_require}
HTTP 200 — existing policy must be read successfully before modification. Die on failure; a silent fallback to empty policy would destroy existing bindings (e.g., the Cloud Build service agent binding set by {rbtgo_depot_create}).
.. *Complete policy write*: Add ALL expected `secretAccessor` bindings to the read policy, regardless of what the read returned:
... `serviceAccount:«DIRECTOR_EMAIL»` — Director needs secret access for inscribe
... `serviceAccount:service-«PROJECT_NUMBER»@gcp-sa-cloudbuild.iam.gserviceaccount.com` — Cloud Build service agent needs secret access at build time
.. NOTE: This complete-policy-write pattern prevents the IAM eventual consistency race: if `getIamPolicy` returns stale data missing the CB service agent binding (set by {rbtgo_depot_create}), the `setIamPolicy` still includes it.
.. {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets/setIamPolicy[secrets.setIamPolicy] with the complete expected policy
.. {rbbc_require}
IAM policy set with all expected secretAccessor bindings

//axhos_step
. *Grant
{rbtr_mason}
Impersonation*:
.. NOTE:
{rbtr_director}
submits builds that execute as
{rbtr_mason}.
This requires `serviceAccountUser` permission on
{rbtr_mason}.
.. {rbtoe_iam_grant_sa} on
{rbtr_mason}
service account
.. Add `serviceAccount:«DIRECTOR_EMAIL»` with `roles/iam.serviceAccountUser`
.. {rbbc_require}
IAM binding successful

//axhos_step
. *Grant
{rbtgi_build_bucket}
Access*:
.. {rbtoe_iam_grant_bucket} on
{rbtgi_build_bucket}
.. Add `serviceAccount:«DIRECTOR_EMAIL»` with roles:
... `roles/storage.objectCreator` (upload sources)
... `roles/storage.objectViewer` (verify uploads)
.. {rbbc_require}
bucket IAM update successful

//axhos_step
. *Grant
{rbtgi_repository}
Admin Access*:
.. {rbbc_call}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories/setIamPolicy[repositories.setIamPolicy] for
{rbtgi_repository}
.. *Complete policy write*: Add ALL expected bindings to the read policy, regardless of what the read returned:
... `serviceAccount:«DIRECTOR_EMAIL»` with `roles/artifactregistry.repoAdmin` — Director manages images
... `serviceAccount:«MASON_EMAIL»` with `roles/artifactregistry.writer` — Mason pushes images at build time
.. NOTE: Same complete-policy-write pattern as the secret IAM step. Prevents stale read from overwriting Mason's writer binding set by {rbtgo_depot_create}.
.. {rbbc_require}
repository IAM set with all expected bindings

//axhoo_output_of_type
{rbtgi_rbra_file}
. *Display Installation Instructions*:
.. {rbbc_show}
Director created successfully
.. {rbbc_show}
RBRA file location: `«DIRECTOR_RBRA_FILE»`
.. {rbbc_show}
installation command:
... `cp "«DIRECTOR_RBRA_FILE»" "{rbrr_director_rbra_file}"`
.. {rbbc_warn}
if
{rbrr_director_rbra_file}
already exists

//axhoc_completion
Successful execution creates director service account.
