The
{at_operator}
performs one-time manual setup to establish
Google Developer Connect (GDC) infrastructure linking the main GitHub repository
to Google Cloud Build for trigger-based builds.
Developer Connect uses an OAuth GitHub App — no GitHub PAT is required.
This procedure creates the connection and repository link but does NOT create build triggers.

// ⟦axs_inputs axd_none⟧

// ⟦axs_preconditions⟧

* Google Cloud project exists with billing enabled ({rbrr_depot_project_id})
* Developer Connect API enabled: `developerconnect.googleapis.com`
* Secret Manager API enabled: `secretmanager.googleapis.com`
* Operator has `Developer Connect Admin` IAM role on depot project
* GitHub account has owner or admin access on the main repository

// ⟦axs_behavior⟧

. *Enable Required APIs*:
.. Using an authenticated gcloud session or Console:
... Enable Developer Connect API
... Enable Secret Manager API (Developer Connect stores OAuth token here)

. *Create Developer Connect Connection*:
.. Run:
+
----
gcloud developer-connect connections create CONNECTION_NAME \
    --location=REGION \
    --github-config-app=DEVELOPER-CONNECT
----
+
.. Replace `CONNECTION_NAME` with chosen name (e.g., `rbw-github-connection`)
.. Replace `REGION` with Developer Connect region (e.g., `us-central1`)
.. The connection is created in a *pending* state — OAuth authorization is not yet complete

. *Complete GitHub OAuth Authorization*:
.. Retrieve the authorization URI:
+
----
gcloud developer-connect connections describe CONNECTION_NAME \
    --location=REGION \
    --format="value(installationState.actionUri)"
----
+
.. Open the returned URI in a browser
.. GitHub presents the standard OAuth authorization page for the "Google Developer Connect" app
.. Click *Authorize* to grant access
.. GitHub redirects back to Google; Developer Connect stores the OAuth token as a Secret Manager secret

. *Verify Connection Is Active*:
.. Poll until `installationState.stage` reaches `COMPLETE`:
+
----
gcloud developer-connect connections describe CONNECTION_NAME \
    --location=REGION \
    --format="value(installationState.stage)"
----
+
.. Expected output: `COMPLETE`
.. If still `PENDING_USER_OAUTH` or `PENDING_INSTALL_APP`, re-open the `actionUri`

. *Create Git Repository Link*:
.. Link the main GitHub repository to the connection:
+
----
gcloud developer-connect connections git-repository-links create REPO_LINK_NAME \
    --clone-uri=https://github.com/OWNER/REPO.git \
    --connection=CONNECTION_NAME \
    --location=REGION
----
+
.. Replace `REPO_LINK_NAME` with chosen name (e.g., `rbw-main-repo`)
.. Replace `OWNER/REPO` with the actual GitHub owner and repository name

. *Record Resource Names in RBRR*:
.. Update `rbrr.env` with the actual values:
... `RBRR_GDC_CONNECTION_NAME` = the CONNECTION_NAME used in step 2
... `RBRR_GDC_REGION` = the REGION used in step 2
... `RBRR_GDC_REPO_LINK` = the REPO_LINK_NAME used in step 5
.. These are short names, not full resource paths — the Foundry constructs full paths from
   `RBRR_DEPOT_PROJECT_ID` + `RBRR_GDC_REGION` + connection/link names

. *Validate via REST API Health Check*:
.. Verify the connection is healthy using curl (consistent with Foundry's REST-only pattern):
+
----
curl -s -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  "https://developerconnect.googleapis.com/v1/projects/${RBRR_DEPOT_PROJECT_ID}/locations/${RBRR_GDC_REGION}/connections/${RBRR_GDC_CONNECTION_NAME}" \
  | jq '.installationState.stage'
----
+
.. Expected: `"COMPLETE"`
.. Verify the repository link exists:
+
----
curl -s -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  "https://developerconnect.googleapis.com/v1/projects/${RBRR_DEPOT_PROJECT_ID}/locations/${RBRR_GDC_REGION}/connections/${RBRR_GDC_CONNECTION_NAME}/gitRepositoryLinks/${RBRR_GDC_REPO_LINK}" \
  | jq '.cloneUri'
----
+
.. Expected: the GitHub clone URI of the main repository

// ⟦axs_outputs⟧

The outputs of GDC establishment are:

- Connection name (`RBRR_GDC_CONNECTION_NAME`) — Developer Connect connection resource
- Region (`RBRR_GDC_REGION`) — regional location of the connection
- Repository link name (`RBRR_GDC_REPO_LINK`) — git-repository-link resource for the main repo
- OAuth token automatically stored in Secret Manager (managed by Developer Connect, not by operator)

Full resource paths constructed as:

----
projects/{RBRR_DEPOT_PROJECT_ID}/locations/{RBRR_GDC_REGION}/connections/{RBRR_GDC_CONNECTION_NAME}
projects/{RBRR_DEPOT_PROJECT_ID}/locations/{RBRR_GDC_REGION}/connections/{RBRR_GDC_CONNECTION_NAME}/gitRepositoryLinks/{RBRR_GDC_REPO_LINK}
----

// ⟦axs_completion⟧

NOTE: GDC establishment creates the GitHub-to-GCP bridge but does not create build triggers.
Proceed to trigger creation (per-vessel) to complete the build pipeline setup.
The connection serves all vessel triggers — this procedure is performed once, not per vessel.
