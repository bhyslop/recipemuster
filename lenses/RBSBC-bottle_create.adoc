NOTE: This sequence within
{opbr_bottle_run}
creates a transient
{at_bottle_container}
from the configured image, establishing network connectivity and volume mounts without persisting the container after execution.

// ⟦axs_behavior⟧
. *Configure Container Parameters*:
.. {rbbc_use}
{rbrn_bottle_vessel}:{rbrn_bottle_consecration} as source image
.. {rbbc_use}
{at_enclave_network} for exclusive network attachment
.. {rbbc_use}
{at_sentry_container} address as DNS resolver
.. {rbbc_use}
{rbrn_volume_mounts} specifications for filesystem access

. *Create Container*:
.. {rbbc_call}
podman create with:
... Image: {rbrn_bottle_vessel}:{rbrn_bottle_consecration}
... Network: {at_enclave_network} (exclusive)
... DNS: {at_sentry_container} address
... Volumes: {rbrn_volume_mounts}
... Cleanup flag: `--rm` (automatic removal on exit)
.. Allow podman to assign unique container name
.. {rbbc_require}
container created successfully
.. {rbbc_store}
`«CONTAINER_ID»` from creation response

// ⟦axs_completion⟧
Successful execution creates a configured transient container ready for command execution.
