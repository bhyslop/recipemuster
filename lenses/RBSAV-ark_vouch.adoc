//axhob_operation
{rbtgo_ark_vouch}

NOTE: This operation vouches for an
{rbtga_ark}
in
{rbtgi_repository}
by verifying SLSA provenance on each per-platform image digest and publishing the verification results as a
{rbtga_ark_vouch}
artifact.
The
{rbtr_director}
submits a Cloud Build job via `builds.create` (direct submission), which runs as the
{rbtr_mason}
service account on the default pool.

NOTE: Vouch uses `builds.create` rather than trigger dispatch because the
{rbtga_consecration}
is known only after conjure completes.
The default pool is acceptable: vouch steps only need internet access (binary download, GAR registry, Sigstore root of trust).

NOTE: {rbsk_no_gcloud} applies.
Registry authentication uses the GCE metadata server to obtain a service account token and writes a static Docker config.

The
{rbtr_director}
vouches for an ark:

{rbtga_ark}

.. {rbbc_require}
vessel directory provided as first argument
.. {rbbc_require}
consecration provided as second argument

. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}

`«DIRECTOR_TOKEN»` for subsequent API calls

.. Image: {rbrg_alpine_image_ref}

.. Compute SHA256 checksum of downloaded binary
.. {rbbc_require}
checksum matches {rbrg_slsa_verifier_sha256}
.. {rbbc_store}

//axhos_step
. *Cloud Build Step 2: Discover Digests and Verify Provenance*:
.. Image: {rbrg_alpine_image_ref}
.. Authenticate to GAR via GCE metadata server token, write Docker config
.. {rbbc_call}
Docker Registry API: GET manifests for `«CONSECRATION»-image` to obtain OCI image index
.. {rbbc_store}
`«PLATFORM_DIGESTS»` as list of `(platform, digest)` pairs extracted from the manifest list
.. For each `(platform, digest)` in `«PLATFORM_DIGESTS»`:
... {rbbc_call}
`slsa-verifier verify-image` with `--source-uri` set to `git+«RBRR_RUBRIC_REPO_URL»` and `--print-provenance`
... {rbbc_store}
per-platform verification output to `/workspace/verify-«PLATFORM_SUFFIX».json`
.. {rbbc_require}

//axhos_step
. *Cloud Build Step 3: Assemble and Push Vouch Artifact*:
.. Image: {rbrg_docker_image_ref}
.. Compose `vouch_summary.json`: consecration, vessel, verifier identity (URL + SHA256), per-platform verification verdicts
.. Build `FROM scratch` container with summary and per-platform verification files as layers
.. Push to registry tagged `«CONSECRATION»-vouch`
.. {rbbc_require}
push succeeds

//axhos_step

.. {rbbc_poll}
`GET «GCB_PROJECT_BUILDS_URL»/«BUILD_ID»` every 5 seconds
.. Continue while status in [`PENDING`, `QUEUED`, `WORKING`]
.. {rbbc_fatal}
if timeout or final status is not `SUCCESS` -- no partial vouch artifacts are pushed

.. {rbbc_show}

//axhoc_completion
Successful execution creates a
{rbtga_ark_vouch}
artifact in the registry tagged as `«VESSEL»:«CONSECRATION»-vouch`.
The vouch artifact is a single-platform container (verification results are architecture-independent metadata).
+
*Source URI matching*: The `--source-uri` must be `git+«RBRR_RUBRIC_REPO_URL»` to match the source recorded in provenance by trigger-dispatched builds.
Format mismatch causes verification failure.
+
*Re-vouch idempotency*: Re-running vouch re-executes verification and pushes a new vouch artifact, overwriting the previous.
Content is presumed deterministic (same digests + same provenance + same verifier = identical output).

==== Vouch Substitution Variables

All `_RBGV_*` substitution variables are passed in the `builds.create` request body.

*Registry and identity*::
[cols="1,2"]
|===
| `_RBGV_GAR_HOST` | GAR Docker host (e.g., `us-central1-docker.pkg.dev`). Source: regime constants.
| `_RBGV_GAR_PATH` | GAR repository path (`«PROJECT»/«REPOSITORY»`). Source: regime constants.
| `_RBGV_VESSEL` | Vessel sigil. Source: loaded vessel configuration.
| `_RBGV_CONSECRATION` | Full consecration string. Source: input parameter.
|===

*Verification*::
[cols="1,2"]
|===
| `_RBGV_SOURCE_URI` | `git+«RBRR_RUBRIC_REPO_URL»`. Source: regime configuration.
| `_RBGV_VERIFIER_URL` | slsa-verifier download URL. Source: {rbrg_slsa_verifier_url}.
| `_RBGV_VERIFIER_SHA256` | slsa-verifier binary checksum. Source: {rbrg_slsa_verifier_sha256}.
|===
