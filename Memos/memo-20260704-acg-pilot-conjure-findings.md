# ACG Pilot 3 — Conjure Pair Audit Findings

## Origin

Third and final ACG audit pilot — the scale test on the richest surface. Audited
the conjure pair: RBSAC against `rbfd_director.sh`'s conjure path (`rbfd_ordain`,
`rbfd_build`, `zrbfd_preflight_reliquary`, `zrbfd_registry_preflight`,
`zrbfd_stitch_build_json`, `zrbfd_push_build_context`), plus support where the
spec reaches: `rbfca_assembly.sh`, `rbfcg_gar.sh`, `rbfcp_plumb.sh`,
`rbfc0_core.sh`, `rbfcb_host.sh`, `rbfcv_resolve.sh`, cloud steps
`rbgjb01`–`rbgjb07` and `rbgjr01`. Spec neighbors opened to judge homes: RBSAG,
RBSAP, RBSAB, RBSCJ, RBSLE, RBSCL, RBSIR, and the RBS0 kernels
(`rbsk_in_pool_preflight`, `rbsk_pinning_boundary`, `rbsk_human_present`) and
fact/chain quoins.

Run under the full detect-only move set with pilots 1–2's amendments in force
(cite-then-restate signature, citation-collapse default, residue discipline,
104/105 unit boundary, deliberate-absence ruling, axd_inlaid name-identity).

## Headline

Three verdicts, one per pilot question:

1. **The hardened guide held.** Nearly every detected comment resolved by
   pilot-1/2 machinery with no new judgment required: the spec side is mature,
   so detection degenerated to citation-collapse almost everywhere, exactly as
   ACGm_104 now predicts. The detect hook (grep spec acronyms and `RBr_` inside
   comment blocks) found the collapse population mechanically.
2. **New ground found — the verifier-tier gap.** The audit's sharpest finding
   (F10) is a *live* defect ACG's licensing model cannot yet talk about: a magic
   step-index whose only verifier is a paid cloud build. Reveille cannot catch
   it; the assert it feeds has been silently dead since 2026-05-01, proven from
   the tabtarget logs. ACGm_101's "mutate-now — the verifier fails loud" story
   assumes the verifier is free.
3. **The pair fiction does not scale as-is.** `rbfd_director.sh` is three
   operations answering to three spec subdocs; the support modules serve every
   pair at once; the findings concentrated in shared support and at spec seams
   (RBSAC↔RBSAP↔RBSCL/RBSIR). Roster consequence in the final section.

## ACG amendment candidates (twin deliverable)

- **A1 — Verifier tiers.** ACGm_101/108's mutate-now licensing assumes the
  verifier is cheap (shellcheck + reveille). Name the verifier *tier* per move;
  a value only a live cloud build exercises (step indexes, substitution names,
  workspace filenames) is mutate-now only with an explicit
  verify-on-next-live-run note, else detect-only. F10 is the motivating
  specimen.
- **A2 — Foreign-environment constant home.** Bless the pattern: when a file's
  environment cannot source the constant home (cloud step, jailer script), the
  literal at the foreign site is licensed, the host constant is the home, and
  the home's comment declares the pairing + grep gate
  (`RBGC_IMAGE_LABEL_RESOLVED_BASE` precedent).
- **A3 — Allocation is orthogonal to dialect.** One-file-one-guide governs
  dialect; ACG's moves reach into BCG/RCG/WSG/CBG/JDG files as the allocation
  axis, and a collapse writes its residue in the host guide's comment idiom.
  (Also note: cloud-step comments ship in the build JSON yet remain edit-time —
  no re-read forcing function at run.)
- **A4 — Contract-header boundary test.** A header sentence survives if it
  states what a caller/editor must know *at the code* — args, preconditions,
  side effects, result channel, announce posture. Sentences about why the
  system is shaped so go to the home. (The audit found headers that are 80%
  design restatement wrapped around 20% contract.)
- **A5 — Authority is never a comment.** Extend ACGm_104 detect: a citation
  whose target is another comment ("see the comment in rbfd_graft") is itself a
  finding; repoint to the home.
- **A6 — Multi-symbol name-identity** (feed to AXLA/MCM): what ACGm_102 means
  for an N-mode operation fronted by thin public wrappers over a shared core
  (the `rbtgo_ark_inspect` case) — which symbol is the inlay, or whether the
  no-single-symbol exception applies.
- **A7 — Spec-side rivet residue** (needs operator ruling): when a *spec* cites
  a rivet defined in a sibling spec (RBSAP citing `RBr_b4e`), does it get a
  one-clause reader gist or full residue discipline? This is the DRY-the-
  rationale vs keep-the-load-bearing tension at its hardest.
- **A8 — Residue form in shipped source** (operator-raised mid-landing; ruling
  pending). Shipped source never cites a spec acronym: the `.sh` files ship
  while the closed specs do not, so a readable spec name leaks the veiled
  tree's structure and dangles for consumers — and it drifts on spec
  renames/decompositions, where a cited rivet is never renumbered. The residue
  form is: a rivet ID where a normative proposition needs an anchor at the
  code (the dirty-tree/pure-chain-head dictum, the graft-ungated camber);
  *nothing* where name-identity already carries the link (move rule 3 — the
  symbol/inlay reaches the spec through the census). Spec acronyms remain fine
  spec-to-spec and in veiled context files. Consequences once ratified: the
  first tranche's acronym residues re-form (two rivets minted, the rest drop
  to bare code); the pre-existing acronym citations across shipped source
  become a detectable corpus-sweep population — roster material, not this
  pace.

## Roster recommendation (recorded for the post-pilot groom)

**No fourth pilot.** The guide is hardened: detection is mechanical, the
collapse machinery covered everything, and the two genuinely new items (A1,
A2) are licensing refinements, not detection gaps.

**But do not roster naive pairs.** The scale test's structural lesson: the
pair unit must be the *operation* (spec subdocument), not the module, and
shared support must not be re-audited N times. Proposed cut:

1. **Per-operation pair paces** — one pace per spec subdoc with a declared
   support closure ("audit RBSAS against rbfr_retriever.sh where the spec
   reaches"). Delegable at opus tier once this pace's amendments land; the
   detect recipe is now: grep spec acronyms + `RBr_` in comments, read homes,
   default to collapse.
2. **One shared-support sweep pace** — rbfc0/rbfca/rbfcb/rbfcg headers plus
   the rbgc/rbcc constant-comment blocks, audited once as a unit.
3. **Cross-document sweeps stay corpus-wide** (105 word-cancer, 107 skidmark)
   per the standing cinch; F6 and F12b feed them.
4. **Hot lanes wait**: sitting-lifecycle comments, delete/banish, federation
   surfaces — audit after their heats land.
5. **Fable-tier reserved** for: the amendment landings, the A7 ruling, and the
   RBSCJ decision-record drain (F12b) if approved.
