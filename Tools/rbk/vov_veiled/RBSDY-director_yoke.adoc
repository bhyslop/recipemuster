//axhob_operation
{rbtgo_director_yoke}

NOTE: This operation writes `RBRV_RELIQUARY=«STAMP»` into an ordain-path vessel's `rbrv.env` so that subsequent Cloud Build operations (conjure, bind, graft, enshrine) consume the named tool image set as GCB step images.
The about/vouch pipeline (gcloud, docker, alpine, syft) consumes reliquary tool images uniformly across all ordain modes; bind additionally uses skopeo for image-copy.
Both arguments are validated before any file write: the stamp against GAR (all six tool packages must be present under the reliquary subtree), the vessel against its RBRV enrollment.
On validation failure the operation errors diagnostically without touching any file.

The
{rbtr_director}
yokes a reliquary stamp into a vessel's regime:

//axhopt_typed_parameter
{rbtr_director}
. *Validate Input Parameters*:
.. {rbbc_require}
`«VESSEL_SIGIL»` provided as folio (e.g., `rbev-sentry-deb-tether`)
.. {rbbc_require}
`«STAMP»` provided as first positional argument (e.g., `r260327172456`)

//axhos_step
. *Load Vessel* (CLI furnish, BUZ_FOLIO-gated):
.. Resolve `«VESSEL_SIGIL»` to vessel directory `${RBRR_VESSEL_DIR}/«VESSEL_SIGIL»`
.. {rbbc_require}
vessel directory contains `rbrv.env`
.. Source `rbrv.env` to populate `RBRV_*` variables
.. Kindle RBRV enrollment (no enforce — yoke writes `RBRV_RELIQUARY`,
which is a kindle constant with min=1 enrollment; pre-write enforce
would self-veto on marshal-zero state. The strict validator path is
`rbw-rvv` ValidateVesselRegime.)

//axhos_step
. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}
.. {rbbc_store}
`«DIRECTOR_TOKEN»` for subsequent API calls

//axhos_step
. *List Reliquary Packages in GAR*:
.. {rbbc_store}
`«DEPOT_PROJECT_ID»` as `${RBRR_CLOUD_PREFIX}d-${RBRR_DEPOT_MONIKER}` (resolved by `RBDC_DEPOT_PROJECT_ID`)
.. {rbbc_store}
`«GAR_REPOSITORY»` as `${RBRR_CLOUD_PREFIX}${RBRR_DEPOT_MONIKER}-gar` (resolved by `RBDC_GAR_REPOSITORY`)
.. {rbbc_call}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories.packages/list[packages.list] for
`«DEPOT_PROJECT_ID»`
/
{rbrr_gcp_region}
/
`«GAR_REPOSITORY»`
with `pageSize=1000`
.. {rbbc_require}
HTTP 200 response
.. Filter response package names to those starting with `{gar_reliquaries_namespace}/«STAMP»/`
.. Strip the subtree prefix to yield the bare tool-name set present in GAR

//axhos_step
. *Verify Tool Image Completeness*:
.. Expected tool name set: `gcloud`, `docker`, `alpine`, `syft`, `binfmt`, `skopeo` (per
{rbtgo_depot_inscribe})
.. {rbbc_require}
all six names present in the filtered list
.. {rbbc_fatal}
if any expected tool is missing — the error message names `«STAMP»`, the reliquary subtree path, and the specific missing tool(s); remediation points at
{rbtgo_depot_inscribe}

//axhos_step
. *Rewrite rbrv.env*:
.. Load the vessel's `rbrv.env` into memory, line by line
.. For each line matching `^RBRV_RELIQUARY=` — substitute with `RBRV_RELIQUARY=«STAMP»`, preserving position
.. All other lines pass through unchanged
.. If no `RBRV_RELIQUARY=` line was found — append one under a `# Tool Image Reliquary` comment header
.. Write the composed content to a temp file, then atomically replace `rbrv.env` via rename

//axhoc_completion
Successful execution writes `RBRV_RELIQUARY=«STAMP»` into the vessel's `rbrv.env`; no GAR state is modified.
The
{rbtr_director}
commits the modified `rbrv.env` with their usual git workflow.
