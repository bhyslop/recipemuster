//axhob_operation
{rbtgo_retriever_create}

The
{rbtr_governor}
creates a
{rbtr_retriever}
{giam_service_account}
with read-only
{rbtgi_repository}
access:

//axhoq_precondition
* Execute {rbtoe_check_rubric_repo_url} (validates {rbrr_rubric_repo_url} using PAT from Secret Manager)

//axhoq_precondition
* Execute {rbtoe_check_cbv2_connection} to verify {rbtgi_cbv2_connection} is active

//axhop_parameter_from_type
{rbtr_retriever}
. *Validate Input Parameters*:
.. {rbbc_require}
`«INPUT_INSTANCE»` provided as first argument
.. {rbbc_store}
`«RETRIEVER_ACCOUNT_NAME»` as `retriever-«INPUT_INSTANCE»`
.. {rbbc_store}
`«RETRIEVER_EMAIL»` as `«RETRIEVER_ACCOUNT_NAME»@{rbrr_depot_project_id}.iam.gserviceaccount.com`

//axhoq_precondition
* {rbbc_require}
`BURD_OUTPUT_DIR` environment variable set and directory exists

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` for subsequent API calls

//axhos_step
. *Create
{rbtr_retriever}
Service Account*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/create[iam.serviceAccounts.create] with:
... accountId: `«RETRIEVER_ACCOUNT_NAME»`
... displayName: `Recipe Bottle Retriever («INPUT_INSTANCE»)`
... description: `Read-only access to Google Artifact Registry - instance: «INPUT_INSTANCE»`
... project:
{rbrr_depot_project_id}
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«RETRIEVER_UNIQUE_ID»` from response `.uniqueId` field

//axhos_step
. *Verify Service Account Propagation*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/get[iam.serviceAccounts.get] for `projects/-/serviceAccounts/«RETRIEVER_EMAIL»`
.. {rbbc_poll}
with backoff until HTTP 200 (newly-created SA may not be immediately resolvable by email)
.. NOTE: Poll must use the email address, not the numeric ID. The email-based lookup path has independent propagation from the UID-based path, and subsequent steps (keys.list, keys.create) all use the email address. Polling by UID can succeed while the email path still returns 404.
.. {rbbc_require}
service account accessible by email

//axhos_step
. *Preflight: Ensure Pristine Key State*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/list[serviceAccounts.keys.list] for `«RETRIEVER_EMAIL»`
.. {rbbc_store}
`«EXISTING_KEYS»` from response `.keys[]`
.. {rbbc_require}
no keys with `keyType: USER_MANAGED` exist
.. {rbbc_fatal}
if USER_MANAGED keys found: "Manual cleanup required via Console"

//axhos_step
. *Generate Service Account Key*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/create[serviceAccounts.keys.create] for `«RETRIEVER_EMAIL»` with:
... privateKeyType: `TYPE_GOOGLE_CREDENTIALS_FILE`
.. Retry up to `RBGC_SA_KEY_CREATE_RETRY_MAX` times on HTTP 404 with `RBGC_SA_KEY_CREATE_RETRY_DELAY_SEC` delay between attempts
.. NOTE: SA read operations (GET by email, keys.list) may succeed while keys.create POST returns 404 "does not exist" due to write-path propagation delay. The 404 confirms no key was created, making retry safe. See <<rbscig_iam_grant_contracts>>.
.. {rbbc_store}
`«RETRIEVER_PRIVATE_KEY_DATA»` from response `.privateKeyData` field
.. {rbtoe_rbra_generate}
converts `«RETRIEVER_PRIVATE_KEY_DATA»` to `«RETRIEVER_RBRA_FILE»`

//axhos_step
. *Grant
{rbtgi_repository}
Read Access*:
.. {rbtoe_iam_grant_project} for
{rbrr_depot_project_id}
.. Add `serviceAccount:«RETRIEVER_EMAIL»` with role:
... `roles/artifactregistry.reader` (read-only access to all Artifact Registry repositories)
.. Policy version: 3
.. {rbbc_require}
IAM policy update successful

//axhoo_output_of_type
{rbtgi_rbra_file}
. *Display Installation Instructions*:
.. {rbbc_show}
Retriever created successfully
.. {rbbc_show}
RBRA file location: `«RETRIEVER_RBRA_FILE»`
.. {rbbc_show}
installation command:
... `cp "«RETRIEVER_RBRA_FILE»" "{rbrr_retriever_rbra_file}"`
.. {rbbc_warn}
if
{rbrr_retriever_rbra_file}
already exists

//axhoc_completion
Successful execution creates retriever service account with read-only Artifact Registry access and provides RBRA file installation command.
