//axhob_operation
{rbtgo_ark_abjure}

NOTE: This operation formally renounces an
{rbtga_ark}
from
{rbtgi_repository}
using the Docker Registry HTTP API V2 implemented by
{gar_service}.

An
{rbtga_ark}
is the coherent unit of a Recipe Bottle build: a set of artifacts sharing a common
{rbtga_hallmark}
but distinguished by suffix.
The
{rbtga_ark_image}
(`-image`) contains the container image itself.
The
{rbtga_ark_about}
(`-about`) contains SBOM and build metadata.
The
{rbtga_ark_vouch}
(`-vouch`) contains verification results.
The
{rbtga_ark_pouch}
(`-pouch`) contains the build context (conjure/bind only, absent for graft).
A fifth artifact, `-diags`, is present for conjure arks only (build-time diagnostics forwarded to the about pipeline).
Abjuring an ark removes all existing artifacts as a single logical operation.

The
{rbtr_director}
abjures an ark:

//axhopt_typed_parameter
{rbtga_ark}
. *Validate Input Parameters*:
.. {rbbc_require}
`«INPUT_VESSEL»` provided as first argument
.. {rbbc_require}
`«INPUT_HALLMARK»` provided as second argument
.. {rbbc_fatal}
if vessel missing: "Vessel parameter required"
.. {rbbc_fatal}
if hallmark missing: "Hallmark parameter required"

//axhopt_typed_parameter
{rbtga_hallmark}
. *Validate Hallmark Format*:
.. {rbbc_require}
`«INPUT_HALLMARK»` matches pattern `[cbg]\d{12}-r\d{12}` (mode prefix + dual timestamp)
.. {rbbc_fatal}
if malformed: "Hallmark format invalid"

//axhos_step
. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}
.. {rbbc_store}
`«DIRECTOR_TOKEN»` for subsequent API calls

//axhos_step
. *Construct Registry Endpoint*:
.. {rbbc_store}
`«REGISTRY_HOST»` as `{rbrr_gcp_region}-docker.pkg.dev`
.. {rbbc_store}
`«REGISTRY_PATH»` as `{rbrr_depot_project_id}/{rbrr_gar_repository}`
.. {rbbc_store}
`«REGISTRY_BASE»` as `https://«REGISTRY_HOST»/v2/«REGISTRY_PATH»`

//axhos_step
. *Construct Ark Tags*:
.. {rbbc_store}
`«IMAGE_TAG»` as `«INPUT_VESSEL»:«INPUT_HALLMARK»-image`
.. {rbbc_store}
`«ABOUT_TAG»` as `«INPUT_VESSEL»:«INPUT_HALLMARK»-about`
.. {rbbc_store}
`«VOUCH_TAG»` as `«INPUT_VESSEL»:«INPUT_HALLMARK»-vouch`
.. {rbbc_store}
`«POUCH_TAG»` as `«INPUT_VESSEL»:«INPUT_HALLMARK»-pouch`
.. {rbbc_store}
`«DIAGS_TAG»` as `«INPUT_VESSEL»:«INPUT_HALLMARK»-diags`

//axhos_step
. *Verify Ark Existence*:
.. {rbbc_call}
link:https://github.com/opencontainers/distribution-spec/blob/main/spec.md#pulling-manifests[Docker Registry API: HEAD manifests] for `«IMAGE_TAG»`:
... method: `HEAD`
... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-image`
... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
... header: `Accept: application/vnd.docker.distribution.manifest.v2+json, application/vnd.docker.distribution.manifest.list.v2+json, application/vnd.oci.image.index.v1+json, application/vnd.oci.image.manifest.v1+json`
.. {rbbc_store}
`«IMAGE_EXISTS»` as true if HTTP 200, false if HTTP 404
.. {rbbc_call}
same HEAD request for `«ABOUT_TAG»`:
... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-about`
.. {rbbc_store}
`«ABOUT_EXISTS»` as true if HTTP 200, false if HTTP 404
.. {rbbc_call}
same HEAD request for `«VOUCH_TAG»`:
... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-vouch`
.. {rbbc_store}
`«VOUCH_EXISTS»` as true if HTTP 200, false if HTTP 404
.. {rbbc_call}
same HEAD request for `«POUCH_TAG»`:
... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-pouch`
.. {rbbc_store}
`«POUCH_EXISTS»` as true if HTTP 200, false if HTTP 404 (optional — conjure/bind only, absent for graft)
.. {rbbc_call}
same HEAD request for `«DIAGS_TAG»`:
... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-diags`
.. {rbbc_store}
`«DIAGS_EXISTS»` as true if HTTP 200, false if HTTP 404 (optional — conjure-only, absent for bind/graft)

//axhos_step
. *Evaluate Ark State*:
.. If all of `«IMAGE_EXISTS»`, `«ABOUT_EXISTS»`, and `«VOUCH_EXISTS»` are false:
... {rbbc_fatal}
"Ark not found: no artifacts exist"
.. If exactly one exists:
... {rbbc_warn}
"Orphaned artifact detected"
... {rbbc_show}
which artifact exists and which is missing
.. {rbbc_store}
`«ARTIFACTS_TO_DELETE»` as list of existing artifacts

//axhos_step
. *Confirm Abjuration*:
.. Unless `--force` flag provided:
... {rbbc_show}
"Will abjure ark «INPUT_VESSEL»/«INPUT_HALLMARK»:" followed by each artifact in `«ARTIFACTS_TO_DELETE»`
... {rbbc_require}
user types `yes` to confirm
... {rbbc_fatal}
if not confirmed: "prompt not confirmed."

//axhos_step
. *Delete Image Artifact* (if exists):
.. If `«IMAGE_EXISTS»`:
... {rbbc_call}
link:https://github.com/opencontainers/distribution-spec/blob/main/spec.md#deleting-tags[Docker Registry API: DELETE manifests] with:
.... method: `DELETE`
.... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-image`
.... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
... {rbbc_require}
HTTP 202 (Accepted) or 204 (No Content) response
... {rbbc_fatal}
if delete fails: "Failed to delete -image artifact (HTTP «STATUS_CODE»)"

//axhos_step
. *Delete About Artifact* (if exists):
.. If `«ABOUT_EXISTS»`:
... {rbbc_call}
Docker Registry API: DELETE manifests with:
.... method: `DELETE`
.... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-about`
.... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
... {rbbc_require}
HTTP 202 (Accepted) or 204 (No Content) response
... {rbbc_fatal}
if delete fails: "Failed to delete -about artifact (HTTP «STATUS_CODE»)"

//axhos_step
. *Delete Vouch Artifact* (if exists):
.. If `«VOUCH_EXISTS»`:
... {rbbc_call}
Docker Registry API: DELETE manifests with:
.... method: `DELETE`
.... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-vouch`
.... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
... {rbbc_require}
HTTP 202 (Accepted) or 204 (No Content) response
... {rbbc_fatal}
if delete fails: "Failed to delete -vouch artifact (HTTP «STATUS_CODE»)"

//axhos_step
. *Delete Pouch Artifact* (if exists):
.. If `«POUCH_EXISTS»`:
... {rbbc_call}
Docker Registry API: DELETE manifests with:
.... method: `DELETE`
.... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-pouch`
.... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
... {rbbc_require}
HTTP 202 (Accepted) or 204 (No Content) response
... {rbbc_fatal}
if delete fails: "Failed to delete -pouch artifact (HTTP «STATUS_CODE»)"

//axhos_step
. *Delete Diags Artifact* (if exists):
.. If `«DIAGS_EXISTS»`:
... {rbbc_call}
Docker Registry API: DELETE manifests with:
.... method: `DELETE`
.... endpoint: `«REGISTRY_BASE»/«INPUT_VESSEL»/manifests/«INPUT_HALLMARK»-diags`
.... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
... {rbbc_require}
HTTP 202 (Accepted) or 204 (No Content) response
... {rbbc_fatal}
if delete fails: "Failed to delete -diags artifact (HTTP «STATUS_CODE»)"

//axhoot_typed_output
{rbtga_ark}
. *Display Results*:
.. {rbbc_show}
"Ark abjured: «INPUT_VESSEL»/«INPUT_HALLMARK»"
.. For each deleted artifact:
... {rbbc_show}
`  - «TAG» deleted`

//axhoc_completion
Procedure completes successfully when all existing ark artifacts have been deleted from the registry and the abjuration confirmation is displayed.
The ark is now formally renounced and no longer part of the repository corpus.
