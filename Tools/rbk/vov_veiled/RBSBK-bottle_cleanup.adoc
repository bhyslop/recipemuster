//axhob_operation
{mkr_bottle_cleanup}

NOTE: Cleanup of prior state is handled by `docker compose down --remove-orphans` within
{opbs_bottle_start}.
This removes all containers and networks for the nameplate's compose project in a single command.
Tolerates missing projects (first start on a fresh system).

After compose-down, a subnet conflict detection step checks whether any foreign Docker network occupies this nameplate's enclave subnet.
If a stale network from a different compose project is found (e.g., containers died without compose-down after a reboot), the operation fails with an actionable cleanup command.
