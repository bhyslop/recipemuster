[[RBr_5b8]]
//axvc_spoor
RBr_5b8::
docker's registry-auth client carries a hardcoded, non-configurable 15s timeout (moby/registry/auth.go);
against a healthy-but-slow GAR auth backend it fires prematurely,
surfacing the Go net/http stdlib string held in `RBGC_DOCKER_LOGIN_TRANSIENT_SIGNATURE`
(`Client.Timeout exceeded while awaiting headers` — a standard-library invariant, stable across docker versions).
Neither `docker login` nor the initial resolve leg of pull / manifest inspect (token fetch + manifest HEAD)
carries an internal retry — the docker client retries only layer downloads, after resolve —
so a single slow token response kills the command.
Upstream: moby/moby#44350.
Surveyed twice: 2026-05-30 at the login site (commit `40c429ddb`),
and 2026-07-10 at the pull resolve leg (gauntlet attempt 7:
the manifest inspect seconds earlier succeeded on its own token fetch,
and the next attempt ran the entire ladder clean — a single-request transient).
Real auth failures emit `unauthorized` and never match the signature.
