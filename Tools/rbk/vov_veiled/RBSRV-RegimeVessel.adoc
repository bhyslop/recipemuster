//axhrb_regime
{rbrv_regime}

This regime specifies variables that define container vessel configurations within the Recipe Bottle system.
Each
{rbrv_regime}
instance represents a vessel image source: bound from an external registry, conjured from a Dockerfile, or grafted from a local build.
RBRV assignment files are found in `rbev-vessels/*/rbrv.env` and configure image sourcing for bottle deployments.

//axhrv_variable
{rbrv_sigil}

Unique identifier for the vessel.
Must be 1-64 characters forming a valid cross-context identifier.
Must match the vessel directory name.

//axhrv_variable
{rbrv_description}

Human-readable description of the vessel purpose.
Optional; 0-512 characters.

//axhrv_variable
{rbrv_vessel_mode}

Selects vessel provisioning strategy.
Must be
{rbrv_vessel_mode_bind},
{rbrv_vessel_mode_conjure},
or
{rbrv_vessel_mode_graft}.

//axhrgv_variable
{rbrv_egress_mode}

Selects which worker pool the primary build operation uses.
Must be `tether` (public egress) or `airgap` (`NO_PUBLIC_EGRESS`).
Inscribe and enshrine always use tether regardless of this setting.
About and vouch always use airgap regardless of this setting.
Conjure and bind use the pool selected by this variable.

//axhrgb_group
{rbrv_group_binding}

//axhrgc_gate
{rbrv_vessel_mode}
{rbrv_vessel_mode_bind}

Variables in this group apply only when
{rbrv_vessel_mode}
is
{rbrv_vessel_mode_bind}.
Binding retrieves a pre-built image from an external registry without local build steps.

//axhrgv_variable
{rbrv_bind_image}

Fully-qualified image reference to copy from an external registry.
Must be a valid FQIN; 1-512 characters.

//axhrgv_variable
{rbrv_bind_optional_dockerfile}

Optional path to reference Dockerfile for the upstream image.
When configured, the Dockerfile content is passed to
{rbtgo_ark_about}
and embedded in the about artifact as `recipe.txt`.
Optional; 0-512 characters.

//axhrgb_group
{rbrv_group_conjuring}

//axhrgc_gate
{rbrv_vessel_mode}
{rbrv_vessel_mode_conjure}

Variables in this group apply only when
{rbrv_vessel_mode}
is
{rbrv_vessel_mode_conjure}.
Conjuring builds a vessel image locally from a Dockerfile and build context in the repository.

//axhrgv_variable
{rbrv_conjure_dockerfile}

Path to Dockerfile relative to repository root.
Must be 1-512 characters.

//axhrgv_variable
{rbrv_conjure_bldcontext}

Build context path relative to repository root.
Must be 1-512 characters.

//axhrgv_variable
{rbrv_conjure_platforms}

Space-separated list of target platforms for multi-platform builds.
Must be 1-512 characters.
Examples: `linux/amd64`, `linux/amd64 linux/arm64`.

//axhrgb_group
{rbrv_group_image}

//axhrgc_gate
{rbrv_vessel_mode}
{rbrv_vessel_mode_conjure}

Variables in this group declare base image dependencies for multi-stage Dockerfiles.
Up to 3 indexed slots (n = 1, 2, 3).
{rbtgo_ark_enshrine}
resolves each
{rbrv_image_origin}
to a GAR-mirrored
{rbrv_image_anchor}.

//axhrgv_variable
{rbrv_image_origin}

Upstream base image tag declaration (e.g., `python:3.11-slim`).
Vessel author declares intent; the build system resolves to a concrete reference.
Optional; 0-512 characters per slot.

//axhrgv_variable
{rbrv_image_anchor}

GAR-mirrored, content-addressed anchor for the base image.
Written by
{rbtgo_ark_enshrine}.
Format: `{sanitized-origin}-{10-char-sha256}`.
When present, conjure substitutes the full GAR reference.
When absent, conjure passes
{rbrv_image_origin}
through as the upstream tag.
Optional; 0-512 characters per slot.

//axhrgb_group
{rbrv_group_grafting}

//axhrgc_gate
{rbrv_vessel_mode}
{rbrv_vessel_mode_graft}

Variables in this group apply only when
{rbrv_vessel_mode}
is
{rbrv_vessel_mode_graft}.
Grafting accepts a locally-built image pushed to GAR via native docker/podman push.
No Cloud Build involvement for the image itself — about and vouch run in Cloud Build after push.

//axhrgv_variable
{rbrv_graft_image}

Local image reference for graft operations.
Must be 1-512 characters.
Provides the image argument to
{rbtgo_ark_graft}.

//axhrgv_variable
{rbrv_graft_optional_dockerfile}

Optional path to Dockerfile used for the local build.
When configured, the Dockerfile content is passed to
{rbtgo_ark_about}
and embedded in the about artifact as `recipe.txt`.
Optional; 0-512 characters.

//axhro_kindle
{rbkro_kindle}

Sets defaults for all RBRV_ variables via `VAR="${VAR:-}"` pattern.
Detects unexpected RBRV_ variables via compgen prefix scan.
Builds ZRBRV_ROLLUP serialization string.

//axhro_validate
{rbkro_validate}

Validates fields via buv_env_* type checkers.
Checks mode-appropriate presence invariant: RBRV_BIND_IMAGE for bind, RBRV_CONJURE_DOCKERFILE for conjure, optional RBRV_GRAFT_IMAGE for graft.
Validates optional Dockerfile paths when configured: RBRV_BIND_OPTIONAL_DOCKERFILE (bind), RBRV_GRAFT_OPTIONAL_DOCKERFILE (graft).

//axhro_render
{rbkro_render}

Renders via rbcr_* shared module.
Sections: Core Vessel Identity, Egress Mode, Binding Configuration, Conjuring Configuration, Grafting Configuration.
Displays unexpected variables in red.
Validates after full render completes.

//axhro_list
{rbkro_list}

Enumerates available vessel instances by scanning subdirectories of RBRR_VESSEL_DIR for rbrv.env files.
Returns sigil identifiers suitable for instance selection.
