//axhob_operation
{rbtgo_director_divest}

NOTE: This operation divests a
{rbtr_director}
{giam_service_account}
from the
{rbtgi_depot_project}.
The
{rbtr_governor}
reads the role's installed RBRA, extracts the SA email from the `client_email` field, issues a REST DELETE against the IAM API, and removes the production RBRA file from disk to mirror cloud state and prevent stale-credential reuse.
No arguments required; the target SA is derived entirely from the kindled regime and the role's installed credential file.

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` for subsequent API calls

//axhos_step
. *Read Role RBRA and Extract Service Account Email*:
.. {rbbc_require}
the role's installed RBRA file exists at
{rbrr_director_rbra_file}
.. {rbbc_fatal}
if the file is missing: "Director RBRA file not found — director may not have been invested"
.. {rbbc_store}
`«SA_EMAIL»` from the `client_email` field of
{rbrr_director_rbra_file}
.. {rbbc_fatal}
if `client_email` is empty or absent: "Director RBRA file present but client_email field is missing or empty"

//axhos_step
. *Divest Director Service Account*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/delete[iam.projects.serviceAccounts.delete] with:
... method: `DELETE`
... endpoint: `https://iam.googleapis.com/v1/projects/{rbtgi_depot_project}/serviceAccounts/«SA_EMAIL»`
... header: `Authorization: Bearer «GOVERNOR_TOKEN»`
.. {rbbc_require}
HTTP 200 response OR HTTP 404 (already divested)
.. {rbbc_warn}
if HTTP 404: "Service account not found (already divested)"

//axhos_step
. *Remove Production RBRA File*:
.. Remove the role's RBRA file at
{rbrr_director_rbra_file}
from disk
.. {rbbc_require}
file removal successful or file already absent
.. NOTE: Removing the RBRA file mirrors cloud state locally and prevents any process from reusing a credential that no longer corresponds to an active
{giam_service_account}.

//axhos_step
. *Display Result*:
.. {rbbc_show}
single-line success naming `«SA_EMAIL»` and confirming the
{rbtr_director}
{giam_service_account}
was divested, or a notice that it was already absent

//axhoc_completion
Successful execution removes the
{rbtr_director}
{giam_service_account}
from
{rbtgi_depot_project}
and deletes the local RBRA credential file.
Cloud state and local credential state are left in agreement: no stale key file remains after a successful divest.
A 404 response from the cloud DELETE is accepted as success — the
{giam_service_account}
is absent either way, and the RBRA file is removed regardless.
Proceed to
{rbtgo_director_invest}
to re-issue a fresh
{rbtr_director}
{giam_service_account}
when needed.
