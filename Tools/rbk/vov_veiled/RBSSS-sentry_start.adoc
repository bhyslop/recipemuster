//axhob_operation
{opss_sentry_start}

NOTE: Docker Compose orchestrates the full container lifecycle declaratively.
The sentry starts first; its health check (`/tmp/rbjh_healthy`) gates the pentacle; the pentacle's health check (`/tmp/rbjph_healthy`) gates the bottle.
Network creation, container instantiation, and startup ordering are all managed by compose — the sub-operations below describe what happens inside each container at startup, not imperative orchestration steps.

//axhos_step
. *Compose creates networks and starts sentry*:
.. Compose creates
{at_transit_network}
(`transit`) and
{at_enclave_network}
(`enclave` with static subnet from nameplate)
.. Sentry container starts on both networks with privileged mode
.. Baked-in entrypoint (`rbjs_sentry.sh`) executes

//axhos_step
. *Sentry discovers interfaces*:
.. Sentry uses `ip -o addr show to «enclave_ip»` to identify which interface carries the enclave IP
.. The other global-scope interface is identified as the uplink
.. All iptables rules use discovered interface names, not hardcoded eth0/eth1
.. NOTE: Docker Compose does not guarantee network-to-interface ordering; IP-based discovery is required

//axhos_step
. *Sentry configures iptables*:
.. Default policies: DROP on INPUT, FORWARD, OUTPUT
.. Custom chains: RBM-INGRESS, RBM-EGRESS, RBM-FORWARD
.. ICMP allowed within enclave only
.. Entry port DNAT if enabled
.. Access and DNS rules per nameplate configuration (allowlist/global/disabled modes)

//axhos_step
. *Sentry starts dnsmasq*:
.. Listens on enclave interface at
{rbrn_enclave_sentry_ip}
.. Domain filtering per
{rbrn_uplink_dns_mode}
(allowlist or global)
.. DNS firewall rules added to RBM-INGRESS and RBM-EGRESS

//axhos_step
. *Sentry signals health*:
.. Creates `/tmp/rbjh_healthy`
.. Compose health check detects this and allows pentacle to start

//axhoc_completion
After sentry is healthy, the
{at_sentry_container}
has dual network connectivity with iptables enforcement and DNS services active, ready to serve the
{rbsc_crucible}.
