The
{rbtt_theurge}
is Recipe Bottle's test-orchestration engine: it composes the project's fixtures
(BUS0 `busr_fixture`) into named suites (`busr_suite`) and runs them against real
regimes, real cloud, and real crucibles.
This subdoc homes the cosmology the engine embodies — the vocabulary code cites —
and defers every containment, membership, and identity fact to its production home.

== The wrapper(inner) combinator

A runnable `busr_suite` is, in the theurge model, exactly one product:
one set of inner bodies run under one
{rbtt_wrapper}.
The inner bodies are the suite's
{rbtt_case_s};
the wrapper is the substrate posture they run against.
The suite-composition oracle (`rbtdtc_crucible.rs`) proves every literal suite in
`RBTDRA_SUITES` remains expressible as this wrapper(inner) product, and asserts the
structural laws the product implies.

[[rbtt_theurge]]
{rbtt_theurge}::
Recipe Bottle's test-orchestration engine (the `rbtd` crate): the composer and
runner of fixtures and suites, and the cosmology this subdoc homes.

[[rbtt_wrapper]]
{rbtt_wrapper}::
The outer factor of the combinator: the substrate posture a set of
{rbtt_case_s}
runs under.
Four wrappers exhaust the model —
*Base* (substrate-independent — the reveille
{rbtt_stratum},
no freehold),
the
{rbtt_reuse}
(a standing
{rbtt_freehold}),
the
{rbtt_lifecycle}
(an ephemeral create→destroy
{rbtt_leasehold}),
and *Local* (fully local, no freehold).
A wrapper selects the substrate the inner bodies run against; it never alters a
fixture's credless arming — that is the credless membrane, and it is why suite
membership is a set of *whole* fixtures, each keeping the credless flag it was
registered with.

[[rbtt_case]]
{rbtt_case}::
An inner body: one atomic test returning a Pass / Fail / Skip verdict, run in its
own isolated temp directory.
The unit a
{rbtt_wrapper}
runs, and the *inner* of the wrapper(inner) product.

[[rbtt_reuse]]
{rbtt_reuse}::
The wrapper that runs its inner bodies against a STANDING
{rbtt_freehold}:
reuse the freehold if it is live, else levy one once, then heal credentials —
never tearing down.
The durable-substrate posture behind picket, bivouac, echelon, skirmish,
dogfight, and blockade.
Completeness law: the complete reuse product is service ∪ crucible.

[[rbtt_lifecycle]]
{rbtt_lifecycle}::
The wrapper that stands up a fresh
{rbtt_leasehold}
and tears it down within the run — the create→destroy proof behind the gauntlet
ladder.
The moniker picker's `max + 1` rule is the safety boundary: a lifecycle run always
mints a leasehold ABOVE the standing freehold, so its tear-down can never reach the
freehold.
Ladder law: blockade ⊆ skirmish ⊆ gauntlet.

== The three strata

A
{rbtt_stratum}
is a dependency tier — an RB instance of the BUS0 `busr_suite` dependency-tier axis
— gating which fixtures a suite may host by what external dependency they need.

[[rbtt_stratum]]
{rbtt_stratum}::
A dependency tier over fixtures.
Three strata partition the base ladder:
*reveille* (no external dependency; credless),
*picket* (GCP credentials),
and *bivouac* (a container runtime);
*echelon* is their union (picket ∪ bivouac over reveille).
The stratum is an axis over fixtures — what a fixture *needs* — orthogonal to the
{rbtt_wrapper}
that selects *substrate*.

== The freehold composite

[[rbtt_freehold]]
{rbtt_freehold}::
The durable, reusable test substrate a
{rbtt_reuse}
wrapper stands on — homed in `rbtdrk_freehold.rs`.
A composite of a cloud-prefix base and a runtime-prefix base, a per-station
tinctured family stem, an auto-incrementing six-digit moniker family, the depot
project-id derivation, and the depot fact-file layout.
It persists across runs and across
{rbtf_foedus}
re-mints; only a deliberate, suiteless freehold-churn destroys it.

[[rbtt_leasehold]]
{rbtt_leasehold}::
The ephemeral per-run substrate a
{rbtt_lifecycle}
wrapper mints and destroys: a fresh moniker in the freehold family, always picked
ABOVE the standing freehold (the `max + 1` boundary), stood up and torn down within
one run.
A leasehold is to the
{rbtt_freehold}
as a lease is to a freehold estate — occupied, then surrendered — never the estate
itself.

== Suite membership

The membership of every named suite — which fixtures compose reveille, picket,
gauntlet, and the rest — is owned by the hand-written `RBTDRA_SUITES` literal
registry in `rbtdra_almanac.rs`, the compile-checked source of truth.
It is not transcribed here (ACG: reference the home, don't recreate): a member list
restated in prose drifts silently from the registry.
`rbtdtc_crucible.rs` is the oracle proving each literal list stays expressible as a
wrapper(inner) product.

== The standing freehold subject

The rig pins exactly one standing freehold subject —
`RBTDGC_FREEHOLD_SUBJECT`, the operator's Entra `oid`, homed in RBPC
(`rbpc_constants.sh`).
In the production identity-layers model this is the PERMANENT, pool-independent
citizen-definition layer, and it alone is the test rig's to pin.
That model — the permanent
{rbtf_citizen}
definition versus the EVOLVING
{rbtf_foedus}
and depot instances a live admission is scoped to — is defined production-side
(RBSRF); this subdoc pins the one subject and references the rest.

== Crucible substrate

The substrate the bivouac
{rbtt_stratum}
and the crucible-bearing wrappers exercise — the
{rbsc_crucible}
and its
{at_sentry_container},
{at_pentacle_container},
and
{at_bottle_container} —
is specified by the crucible-security specs (RBSIP, RBSCC, RBSCN, RBSCV, RBSPC).
This cosmology cites them; it does not restate containment.
