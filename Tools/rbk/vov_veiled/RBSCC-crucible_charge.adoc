//axhob_operation
{rbsc_charge}

NOTE: This operation starts the full
{at_sessile_service}
via Docker Compose.
Compose owns container and network lifecycle; this operation handles preflight validation, cleanup of prior state, subnet conflict detection, and the compose invocation.

//axhos_step
. *Verify Local Vouch Artifacts*:
.. {rbbc_require}
sentry vouch artifact present in local image store (`docker image inspect` on `<GCP_REGION>-docker.pkg.dev/<PROJECT>/<REPO>/{gar_hallmarks_namespace}/<hallmark>/vouch:<hallmark>`)
.. {rbbc_require}
bottle vouch artifact present in local image store (`docker image inspect` on `<GCP_REGION>-docker.pkg.dev/<PROJECT>/<REPO>/{gar_hallmarks_namespace}/<hallmark>/vouch:<hallmark>`)
.. {rbbc_fatal}
if either vouch artifact is absent: emit clear message directing operator to run summon first; no network calls attempted

//axhos_step
. *Clean Previous Instance*:
.. {rbbc_call}
`docker compose ... down --remove-orphans` to remove any existing containers and networks for this nameplate's compose project
.. Tolerates missing project (first start)

//axhos_step
. *Detect Nameplate Collisions*:
.. Enumerate Docker networks bearing the `com.docker.compose.project` label; for each, read its `com.docker.compose.network` and `com.docker.compose.project` labels
.. Match: `com.docker.compose.network` is `enclave` or `transit` AND `com.docker.compose.project` matches `^(.*-)?{rbrn_moniker}$` (covers bare moniker from pre-prefix-era residue and any `{prefix}{rbrn_moniker}` form, since `{rbrr_runtime_prefix}` is constrained to end with `-`)
.. {rbbc_fatal}
if any foreign network matches: refusal lists each offending network name with the exact `docker network rm` command(s)
.. Independently, enumerate all Docker networks and inspect each for subnet match against `{rbrn_enclave_base_ip}/{rbrn_enclave_netmask}`; skip this nameplate's own compose network (already removed by prior step)
.. {rbbc_fatal}
if any foreign network occupies this subnet (catches non-compose networks not covered by the label-based scan)

//axhos_step
. *Start via Compose*:
.. {rbbc_call}
`docker compose --profile sessile up -d` with base compose file, optional nameplate fragment, and env-files for both repo regime and nameplate regime
.. Compose creates networks, starts sentry, waits for health, starts pentacle, waits for health, starts bottle
.. The `sessile` profile includes the bottle container; omitting it starts only sentry and pentacle (infrastructure mode for agile dispatch)

//axhoc_completion
Successful execution establishes a new
{at_sessile_service}
instance with the health check chain guaranteeing that security enforcement is active before the bottle starts.
