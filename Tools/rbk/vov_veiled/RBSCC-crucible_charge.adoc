//axhob_operation
{rbsc_charge}

NOTE: This operation starts the full
{at_sessile_service}
via Docker Compose.
Compose owns container and network lifecycle; this operation handles preflight validation, cleanup of prior state, subnet reclaim, and the compose invocation.

//axhos_step
. *Verify Local Vouch Artifacts*:
.. {rbbc_require}
sentry vouch artifact present in local image store (`docker image inspect` on `<GCP_REGION>-docker.pkg.dev/<PROJECT>/<REPO>/{gar_hallmarks_namespace}/<hallmark>/vouch:<hallmark>`)
.. {rbbc_require}
bottle vouch artifact present in local image store (`docker image inspect` on `<GCP_REGION>-docker.pkg.dev/<PROJECT>/<REPO>/{gar_hallmarks_namespace}/<hallmark>/vouch:<hallmark>`)
.. {rbbc_fatal}
if either vouch artifact is absent: emit clear message directing operator to run summon first; no network calls attempted

//axhos_step
. *Clean Previous Instance*:
.. {rbbc_call}
`docker compose ... down --remove-orphans` to remove any existing containers and networks for this nameplate's compose project
.. Tolerates missing project (first start)

//axhos_step
. *Reclaim Nameplate Subnet*:
.. Enumerate all Docker networks and inspect each for subnet match against `{rbrn_enclave_base_ip}/{rbrn_enclave_netmask}`; skip this nameplate's own compose network (already removed by the prior step)
.. A crucible's subnet derives from the nameplate, not from any project prefix, so at most one network can occupy it; freeing the subnet is the whole invariant and needs no name-lineage check
.. {rbbc_warn}
on the occupying network, if any: force-remove its attached containers, then remove the network — reclaiming the subnet regardless of the network's prefix or how it was created (compose or manual `docker network create`)
.. {rbbc_fatal}
if the occupying network cannot be removed

//axhos_step
. *Start via Compose*:
.. {rbbc_call}
`docker compose --profile sessile up -d` with base compose file, optional {rbnnh_compose_fragment}, and env-files for both repo regime and nameplate regime
.. Compose creates networks, starts sentry, waits for health, starts pentacle, waits for health, starts bottle
.. The `sessile` profile includes the bottle container; omitting it starts only sentry and pentacle (infrastructure mode for agile dispatch)

//axhos_step
. *Invoke Post-Charge Hook (Conditional)*:
.. If `«moniker»/rbnnh_post_charge.sh` exists and is executable, invoke it as the last charge step (after the bottle readiness wait) with `RBRN_*` and `RBRR_*` exported in the environment
.. Absence or non-executable bit is silently tolerated (no warning, no error)
.. {rbbc_fatal}
if the hook exits nonzero — the exit status propagates through {rbsc_charge}, since a present hook expresses a precondition the operator wants enforced; contract details: {rbnnh_post_charge_hook}

//axhoc_completion
Successful execution establishes a new
{at_sessile_service}
instance with the health check chain guaranteeing that security enforcement is active before the bottle starts.
