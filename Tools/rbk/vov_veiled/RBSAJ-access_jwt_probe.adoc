//axhob_operation
{rbtgo_access_jwt_probe}

NOTE: This operation probes
{gar_service}
read access for a specified
{rbtr_role}
by performing a JWT-to-OAuth token exchange and then calling the Artifact Registry packages.list API.
It is a bounded propagation-absorbing retry, not a sustained-access check: the first successful response wins and the operation returns immediately.
A freshly-granted IAM role is not yet readable during the post-grant propagation window, surfacing as HTTP 403; the operation re-attempts — re-exchanging the token each time — until access propagates or the attempt budget is exhausted.
The operation's own defaults are a single attempt with no delay (`--count` 1, `--delay_ms` 0); the SA-role propagation budget (e.g. 60 attempts at 3000ms spacing) is supplied by the dispatching call-site, not baked into this operation.

The operation accepts a role parameter selecting among
{rbtr_governor},
{rbtr_director},
and
{rbtr_retriever},
each with Owner, repoAdmin, and reader permissions respectively.
All three roles have Artifact Registry read access, so a successful response confirms both token validity and role-specific permission.

//axhos_step
. *Validate Input Parameters*:
.. {rbbc_require}
`«INPUT_ROLE»` provided as first argument: one of `governor`, `director`, or `retriever`
.. {rbbc_fatal}
if role missing: "Role parameter required (governor|director|retriever)"
.. {rbbc_fatal}
if role unrecognized: "Unknown role: «INPUT_ROLE» — must be governor, director, or retriever"
.. {rbbc_store}
`«MAX_ATTEMPTS»` from `--count` flag (default: 1) — the maximum number of probe attempts, not a fixed iteration count; the first successful response ends the loop early
.. {rbbc_store}
`«DELAY_MS»` from `--delay_ms` flag (default: 0) — milliseconds to sleep between retries
.. {rbbc_fatal}
if `«MAX_ATTEMPTS»` is not a positive integer: "Count must be a positive integer"
.. {rbbc_fatal}
if `«DELAY_MS»` is not a non-negative integer: "Delay must be a non-negative integer"

//axhos_step
. *Resolve Role Configuration*:
.. If `«INPUT_ROLE»` is `governor`:
... {rbbc_store}
`«ROLE_DISPLAY»` as `Governor`
... {rbbc_store}
`«RBRA_FILE»` from
{rbrr_governor_rbra_file}
.. If `«INPUT_ROLE»` is `director`:
... {rbbc_store}
`«ROLE_DISPLAY»` as `Director`
... {rbbc_store}
`«RBRA_FILE»` from
{rbrr_director_rbra_file}
.. If `«INPUT_ROLE»` is `retriever`:
... {rbbc_store}
`«ROLE_DISPLAY»` as `Retriever`
... {rbbc_store}
`«RBRA_FILE»` from
{rbrr_retriever_rbra_file}
.. {rbbc_require}
`«RBRA_FILE»` exists on the filesystem
.. {rbbc_fatal}
if file missing: "«ROLE_DISPLAY» RBRA file missing at «RBRA_FILE»"

//axhos_step
. *Construct Verification Endpoint*:
.. {rbbc_store}
`«DEPOT_PROJECT_ID»` as `${RBRD_CLOUD_PREFIX}d-${RBRD_DEPOT_MONIKER}` (resolved by `RBDC_DEPOT_PROJECT_ID`)
.. {rbbc_store}
`«GAR_REPOSITORY»` as `${RBRD_CLOUD_PREFIX}${RBRD_DEPOT_MONIKER}-gar` (resolved by `RBDC_GAR_REPOSITORY`)
.. {rbbc_store}
`«VERIFY_URL»` as:
+
`https://artifactregistry.googleapis.com/v1/projects/«DEPOT_PROJECT_ID»/locations/{rbrd_gcp_region}/repositories/«GAR_REPOSITORY»/packages`

//axhos_step
. *Execute Bounded Probe Attempts*:
.. For each attempt `«ATTEMPT»` from 1 to `«MAX_ATTEMPTS»` (the loop ends early on the first successful response):

... *Acquire «ROLE_DISPLAY» Token*:
.... Execute
{rbtoe_jwt_oauth_exchange}
with `«RBRA_FILE»` — the token is re-exchanged on every attempt, so each retry carries a freshly-minted token
.... {rbbc_store}
`«ROLE_TOKEN»` from exchange result
.... {rbbc_fatal}
if token exchange fails: "Attempt «ATTEMPT»: JWT-to-OAuth exchange failed for «ROLE_DISPLAY»"

... *Call Artifact Registry Packages API*:
.... {rbbc_call}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories.packages/list[artifactregistry.projects.locations.repositories.packages.list] with:
..... method: `GET`
..... endpoint: `«VERIFY_URL»`
..... header: `Authorization: Bearer «ROLE_TOKEN»`
.... The call absorbs transient server-side errors internally: on HTTP 500, 502, 503, or 504 it re-executes up to 3 total attempts with exponential backoff (2s, then 4s), and {rbbc_fatal} if all 3 return a 5xx status: "«ROLE_DISPLAY» repeated transient HTTP «HTTP_STATUS» after 3 attempts". The status surfaced to *Evaluate Response* is therefore always non-5xx.
.... {rbbc_store}
`«HTTP_STATUS»` from response status code
.... {rbbc_store}
`«RESPONSE_BODY»` from response body
.... {rbbc_store}
`«ERROR_SUMMARY»` as the `.error.message` field extracted from `«RESPONSE_BODY»` via jq (with a fallback string for empty, missing, or non-JSON bodies)

... *Evaluate Response*:
.... If `«HTTP_STATUS»` is 200 or 206:
..... {rbbc_show}
`Attempt «ATTEMPT»: «ROLE_DISPLAY» OK — packages.list HTTP «HTTP_STATUS»`
..... The operation succeeds and returns immediately; no further attempts run
.... If `«HTTP_STATUS»` is 403 (access not yet propagated):
..... If `«ATTEMPT»` is less than `«MAX_ATTEMPTS»`:
...... {rbbc_show}
`Attempt «ATTEMPT»: «ROLE_DISPLAY» 'not yet' (HTTP 403); retrying`
...... If `«DELAY_MS»` is greater than 0: sleep `«DELAY_MS»` milliseconds before the next attempt
..... Otherwise (the final attempt is still 403):
...... {rbbc_fatal}
"Attempt «ATTEMPT»: «ROLE_DISPLAY» propagation budget exhausted (HTTP 403): «ERROR_SUMMARY»"
.... Otherwise (any other status, including 401):
..... {rbbc_fatal}
"Attempt «ATTEMPT»: «ROLE_DISPLAY» unexpected HTTP «HTTP_STATUS»: «ERROR_SUMMARY»"

//axhos_step
. *Loop Exhaustion Guard*:
.. Reaching the end of the attempt loop without having returned success or died is an internal-logic error (a 200/206 returns, 403 either retries or exhausts, every other status dies).
.. {rbbc_fatal}
"«ROLE_DISPLAY» JWT SA probe: loop exited without verdict — internal logic error"

//axhoc_completion
Procedure completes successfully on the first attempt whose response confirms that the selected role's JWT-derived OAuth token grants read access to the
{gar_service}
packages list endpoint.
Completion demonstrates both that JWT-to-OAuth token exchange is operational for the role and that the resulting token carries the expected Artifact Registry read permissions, having absorbed any post-grant propagation delay within the attempt budget.
