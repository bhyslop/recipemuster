//axhob_operation
{rbtgo_image_delete}

NOTE: This operation jettisons a single image tag from
{rbtgi_repository}
using the Docker Registry HTTP API V2 implemented by
{gar_service}.

`rbfl_jettison` is locator-generic: the same procedure backs
{rbtga_ark}
hallmark members,
reliquary tool images, and enshrinement images.
The package path embedded in the
{rbst_locator}
selects the domain — the procedure itself does not branch.

NOTE: To delete an entire {rbtga_ark} (the full `{gar_hallmarks_namespace}/«HALLMARK»/` subtree — every basename including `image`, `about`, `vouch`, and any `pouch`/`diags`/`attest` tags present), use {rbtgo_ark_abjure} instead.

NOTE: Jettisoning an interior member of a reliquary subtree (e.g., a single tool image) leaves a hole in an otherwise-coherent set. The operator accepts this piecemeal-by-image semantic; recovery is nuclear: re-run {rbtgo_depot_inscribe} to restore the full reliquary corpus. The operation provides no integrity guard against such interior holes.

The
{rbtr_director}
jettisons an image by {rbst_locator}:

//axhopt_typed_parameter
{rbst_locator}
. *Validate Input Parameters*:
.. {rbbc_require}
`«INPUT_LOCATOR»` provided as first argument in `package-path:tag` format
.. {rbbc_fatal}
if missing: "Locator parameter required (package-path:tag)"
.. {rbbc_fatal}
if malformed (no colon): "Invalid locator format. Expected package-path:tag"
.. {rbbc_store}
`«PACKAGE_PATH»` as portion before the final colon (e.g., `{gar_hallmarks_namespace}/«HALLMARK»/image`, `{gar_reliquaries_namespace}/«STAMP»/syft`, `{gar_enshrines_namespace}/«ANCHOR»`)
.. {rbbc_store}
`«TAG»` as portion after the final colon
.. {rbbc_fatal}
if `«PACKAGE_PATH»` is empty: "Package path is empty in locator"
.. {rbbc_fatal}
if `«TAG»` is empty: "Tag is empty in locator"

//axhos_step
. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}
.. {rbbc_store}
`«DIRECTOR_TOKEN»` for subsequent API calls

//axhos_step
. *Confirm Jettison*:
.. {rbbc_show}
`Will jettison: «INPUT_LOCATOR»`
.. {rbbc_require}
user types `yes` to confirm
.. {rbbc_fatal}
if not confirmed: "prompt not confirmed."

//axhos_step
. *Jettison Tag*:
.. {rbbc_call}
link:https://github.com/opencontainers/distribution-spec/blob/main/spec.md#deleting-tags[Docker Registry API: DELETE manifests] with:
... method: `DELETE`
... endpoint: `«REGISTRY_BASE»/«PACKAGE_PATH»/manifests/«TAG»`
... header: `Authorization: Bearer «DIRECTOR_TOKEN»`
.. {rbbc_require}
HTTP 202 (Accepted) or 204 (No Content) response
.. {rbbc_fatal}
if jettison fails: "Jettison failed with HTTP «STATUS_CODE»"

//axhoot_typed_output
{rbst_locator}
. *Display Results*:
.. {rbbc_show}
`Jettisoned or nonexistent: «INPUT_LOCATOR»`

//axhoc_completion
Procedure completes successfully when the tag has been jettisoned from the registry and the jettisoned message is displayed.
The locator-generic shape means a hallmark member, a reliquary tool image, or an enshrinement image are all deleted by the same call — domain identity lives in the locator's package path, not in branched logic.
