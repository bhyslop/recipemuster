//axhob_operation
{rbtgo_director_vouch}

NOTE: This operation vouches for all pending
{rbtga_ark}
consecrations across all
{rbtgi_vessel}
repositories.
The
{rbtr_director}
enumerates vessels, identifies consecrations with `-about` but no `-vouch`, pre-checks SLSA provenance availability for conjure-mode consecrations, and submits individual vouch builds for each vouchable consecration.
Conjure consecrations lacking provenance are skipped with a warning; bind and graft consecrations are always vouchable.

The
{rbtr_director}
batch-vouches arks:

//axhos_step
. *Enumerate Vessels*:
.. Invoke `rbrv_list_capture` to discover all vessel sigils
.. {rbbc_require}
at least one vessel exists

//axhos_step
. *Iterate Vessels*:
.. For each vessel sigil:
... Load vessel configuration via `zrbf_load_vessel`
... {rbbc_call}
Docker Registry API: GET tags list for the vessel
... Classify each tag by consecration and suffix (`-image`, `-about`, `-vouch`)

//axhos_step
. *Classify Consecration Health*:
.. For each unique consecration across all vessels, determine health state:
... *vouched*: has `-about` and `-vouch` — healthy, skip
... *vouchable*: has `-about`, no `-vouch`, and either (a) mode is bind/graft, or (b) mode is conjure and SLSA provenance exists — queue for vouch
... *unvouchable*: has `-about`, no `-vouch`, mode is conjure, no SLSA provenance — skip with warning
... *incomplete*: has `-image` but no `-about` — about pipeline has not run, skip with warning
.. Vessel mode is derived from the consecration mode prefix (`c`=conjure, `b`=bind, `g`=graft)

//axhos_step
. *Pre-check Provenance* (conjure only):
.. For each conjure-mode consecration with `-about` but no `-vouch`:
... {rbbc_call}
`gcloud artifacts docker images describe --show-provenance` to test provenance availability
... If provenance unavailable: classify as *unvouchable*, continue to next
.. Bind and graft consecrations skip this step — they are always vouchable

//axhos_step
. *Submit Vouch Builds*:
.. For each *vouchable* consecration:
... Invoke `rbf_vouch` with vessel directory and consecration parameters
... {rbbc_require}
vouch completes successfully

//axhoo_output_of_type
{rbtga_ark_vouch}
. *Display Summary*:
.. {rbbc_show}
per-vessel table: consecration, health state, action taken
.. {rbbc_show}
batch totals: N vouched, M skipped (no provenance), P already vouched, Q incomplete

//axhoc_completion
Successful execution creates
{rbtga_ark_vouch}
artifacts for all vouchable consecrations across all vessels.
Conjure consecrations without SLSA provenance are skipped gracefully — the batch never aborts on these.
Bind and graft consecrations are always vouchable (no provenance pre-check needed).
+
*No-provenance resilience* (conjure only): Older conjure builds may lack SLSA provenance records.
Rather than submitting a Cloud Build that fails with an opaque `invalid DSSE envelope payload` error, the batch pre-checks provenance availability and skips with a clear message.
+
*Mode-aware vouch*: The batch delegates to {rbtgo_ark_vouch}, which branches on vessel mode internally.
Bind gets digest-pin verification; graft gets GRAFTED stamp; conjure gets full SLSA verification.
