//axhob_operation
{rbtgo_image_retrieve}

NOTE: This operation wrests a container image from
{rbtgi_repository}
to the local container runtime.

`rbfl_wrest` is locator-generic: the same procedure pulls
{rbtga_ark}
hallmark members,
reliquary tool images, and enshrinement images.
The package path embedded in the
{rbst_locator}
selects the domain — the procedure itself does not branch.

NOTE: This is a {rbtr_director} operation (Director credentials, registry write/admin tier — same authority as {rbtgo_image_delete}). It is not the {rbtr_retriever}-tier counterpart of {rbtgo_ark_summon}: summon performs vouch ceremony before pulling under retriever credentials, while wrest is a direct pull with no vouch and no health classification. Use wrest for diagnostic or development pulls; use summon for production retrieval.

The
{rbtr_director}
wrests an image by {rbst_locator}:

//axhopt_typed_parameter
{rbst_locator}
. *Validate Input Parameters*:
.. {rbbc_require}
`«INPUT_LOCATOR»` provided as first argument in `package-path:tag` format
.. {rbbc_fatal}
if missing: "Locator parameter required (package-path:tag)"
.. {rbbc_fatal}
if malformed (no colon): "Invalid locator format. Expected package-path:tag"
.. {rbbc_store}
`«PACKAGE_PATH»` as portion before the final colon (e.g., `hallmarks/«HALLMARK»/image`, `reliquaries/«STAMP»/syft`, `enshrines/«ANCHOR»`)
.. {rbbc_store}
`«TAG»` as portion after the final colon
.. {rbbc_fatal}
if `«PACKAGE_PATH»` is empty: "Package path is empty in locator"
.. {rbbc_fatal}
if `«TAG»` is empty: "Tag is empty in locator"

//axhos_step
. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}
.. {rbbc_store}
`«DIRECTOR_TOKEN»` for subsequent operations

//axhos_step
. *Construct Registry Reference*:
.. {rbbc_store}
`«REGISTRY_HOST»` as `{rbrr_gcp_region}-docker.pkg.dev`
.. {rbbc_store}
`«FULL_IMAGE_REF»` as `«REGISTRY_HOST»/{rbrr_depot_project_id}/{rbrr_gar_repository}/«RBRR_CLOUD_PREFIX»«INPUT_LOCATOR»`

//axhos_step
. *Configure Container Runtime Authentication*:
.. {rbbc_call}
container runtime login with:
... registry: `«REGISTRY_HOST»`
... username: `oauth2accesstoken`
... password: `«DIRECTOR_TOKEN»`
.. {rbbc_require}
login successful
.. {rbbc_fatal}
if login fails: "Container runtime authentication failed"

//axhos_step
. *Pull Image*:
.. {rbbc_call}
link:https://github.com/opencontainers/distribution-spec/blob/main/spec.md#pulling-manifests[Docker Registry API: pull] via container runtime:
... command: `docker pull «FULL_IMAGE_REF»` or `podman pull «FULL_IMAGE_REF»`
.. {rbbc_require}
pull completes successfully
.. {rbbc_store}
`«IMAGE_ID»` from pulled image metadata
.. {rbbc_fatal}
if pull fails: "Image pull failed"

//axhoot_typed_output
{rbtgi_image}
. *Display Results*:
.. {rbbc_show}
image wrested: `«FULL_IMAGE_REF»`
.. {rbbc_show}
local image ID: `«IMAGE_ID»`

//axhoc_completion
Successful execution wrests the image and displays the local image ID.
The locator-generic shape means a hallmark member, a reliquary tool image, or an enshrinement image are all retrieved by the same call — domain identity lives in the locator's package path, not in branched logic.
