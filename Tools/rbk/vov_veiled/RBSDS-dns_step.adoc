//axhob_operation
{scr_dns_step}

Configures sentry DNS resolution via `/etc/resolv.conf` pointing to {rbrr_dns_server}.

If {rbrn_uplink_dns_mode} is disabled, the {at_sentry_container}:

* Blocks all DNS traffic in RBM-FORWARD from the interface attached to {at_enclave_network} (UDP and TCP port 53)
* Blocks all DNS traffic in RBM-EGRESS to the interface attached to {at_transit_network} (UDP and TCP port 53)

If {rbrn_uplink_dns_mode} is global or allowlist, the {at_sentry_container} executes this sequence:

//axhos_step
. Dnsmasq Configuration:
+
--
* Core Settings:
** Bind strictly to the interface attached to {at_enclave_network}
** Listen on {rbrn_enclave_sentry_ip}
** DNS service only — all DHCP functionality disabled (no-dhcp-interface)
** Isolated from host resolver (no-resolv)
** Exclude the interface attached to {at_transit_network} from listening (except-interface)
** Security: bogus-priv, domain-needed, strict-order
** Forwarding concurrency: dns-forward-max=150
** Source port range: 4096-65535

* Cache Settings:
** Cache size: 1000 entries
** Minimum TTL: 600 seconds
** Maximum TTL: 3600 seconds

* Logging (all modes):
** log-queries=extra
** log-facility=/var/log/dnsmasq.log
** log-dhcp, log-debug, log-async=20

* Query Control:
** When {rbrn_uplink_dns_mode} is global:
*** Forward all queries to {rbrr_dns_server}
*** Return all responses unmodified

** When {rbrn_uplink_dns_mode} is allowlist:
*** Resolve-then-freeze: for each domain in {rbrn_uplink_allowed_domains}, resolve A records via {rbrr_dns_server} at startup, then freeze as static `address=` entries in dnsmasq. No `server=` forwarding directives — dnsmasq synthesizes responses locally from frozen IPs. Subdomain queries (e.g., `exfil.anthropic.com`) return the frozen parent IP without generating upstream queries, closing the DNS subdomain exfiltration channel.
*** Return NXDOMAIN for all other queries (address=/#/)
--

//axhos_step
. Service Launch:
* Start dnsmasq daemon
* Wait for initialization (2 second delay)

//axhos_step
. Filter Rules:
+
--
* Accept DNS queries arriving at sentry from enclave:
** RBM-INGRESS: interface attached to {at_enclave_network}, UDP port 53
** RBM-INGRESS: interface attached to {at_enclave_network}, TCP port 53
* Accept sentry upstream DNS forwarding:
** RBM-EGRESS: interface attached to {at_transit_network}, UDP port 53 to {rbrr_dns_server}
** RBM-EGRESS: interface attached to {at_transit_network}, TCP port 53 to {rbrr_dns_server}
--
