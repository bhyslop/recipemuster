//axhob_operation
{rbtgo_lode_augur}

NOTE: This operation inspects a single
{rbtga_lode}
— listing its member tags and decoding its
{rbtga_lode_vouch}
provenance envelope — by querying GAR directly via the Registry API.
Read-only; no local container runtime.
Authenticates as
{rbtr_director}.

NOTE: Augur is the single-Lode inspect verb — the perceiving counterpart, at member grain, to
{rbtgo_lode_divine}'s
lifecycle-grain enumeration.
Divine answers which Lodes exist; augur answers what is inside one — its member tags and its decoded
{rbtga_lode_vouch}
envelope.
The fetched-side parallel of made-side
{rbtgo_ark_inspect}.

NOTE: Augur is kind-agnostic.
It decodes the kind from the
{rbtga_touchmark}
leading letter and reads the same-shaped envelope every kind carries — so a new Lode kind needs no change here.

NOTE: *Read-side chain consumer.*
The touchmark is resolved express-or-chain for a transient inspection only — augur writes no durable config — so a broken chain or an unknown kind dies loud as an ordinary fatal, categorically lower severity than the durable-leak links
{rbtgo_director_feoff}
and
{rbtgo_director_yoke},
whose rejections ride the chaining precision band.
Augur must never be extended to write config without joining that durable-leak surface and its depth-1 no-relay plus named-band-reject discipline.

The
{rbtr_director}
augurs a Lode:

{rbtga_touchmark}
. *Resolve the Touchmark* (express-or-chain):
.. `«TOUCHMARK»` resolves express-or-chain: an express folio argument (e.g., `b260602120000`) wins; absent, it falls back to the touchmark any
{rbtga_lode}
capture handed forward through the depth-1 cross-tabtarget chain — so a no-arg augur immediately after a capture inspects the just-captured Lode. The chain is read terminally — never relayed.
.. {rbbc_fatal}
if neither an express argument nor a chained touchmark is present — a broken chain
.. {rbbc_require}
the resolved `«TOUCHMARK»` decodes to any known Lode kind from its kind-letter prefix (see
{rbst_touchmark}),
via the same single-home prefix decoder
{rbtgo_director_feoff}
and
{rbtgo_director_yoke}
use — augur accepts every kind, unlike those one-kind gates
.. {rbbc_fatal}
if the resolved touchmark decodes to no known kind — this replaces a former format-shape regex that proved well-formedness but never that the prefix named a real kind

. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}
.. {rbbc_store}
`«DIRECTOR_TOKEN»` for subsequent API calls

//axhos_step
. *Fetch Member Tags*:
.. {rbbc_call}
GAR REST `tags.list` for the Lode package `{gar_lodes_namespace}/«TOUCHMARK»`
.. {rbbc_require}
at least one tag present — an empty package is the "Lode not present" signal
.. {rbbc_store}
`«MEMBER_TAGS»` — every member tag on the package

//axhos_step
. *Decode Provenance Envelope*:
.. {rbbc_call}
Registry API: extract the `:rbi_vouch` manifest from the `{gar_lodes_namespace}/«TOUCHMARK»` package and read its `vouch.json` layer
.. {rbbc_store}
the
{rbtga_lode_vouch}
fields (schema `rbld-vouch-3`, all keys sprued `rblv_`): `rblv_kind`, `rblv_acquired_at`, `rblv_acquired_by`, `rblv_capture_build`, `rblv_git_commit`, `rblv_trust_grade`, and `rblv_members[]` (each `rblv_origin` / `rblv_digest` / `rblv_verification` / `rblv_tags`, plus the podvm-only per-member `rblv_acquired_at` / `rblv_capture_build`)

//axhoot_typed_output
{rbtga_lode}
. *Display*:
.. {rbbc_show}
the kind (decoded from the touchmark prefix), the member tags, and the decoded provenance — origin, canonical digest, trust grade, and acquisition facts
.. {rbbc_show}
explicit trust-grade posture: `verified-against-published` versus `recorded-at-acquisition`, so the displayed claim never over-states what the upstream permits

//axhoc_completion
Augur is a read-only inspection of one
{rbtga_lode}
via the GAR Registry API: it lists the member tags and decodes the
{rbtga_lode_vouch}
envelope, and modifies nothing.
+
The categorical layout makes the touchmark the primary index — every member tag and the provenance envelope sit within the one `{gar_lodes_namespace}/«TOUCHMARK»` package, so no cross-Lode aggregator is needed.
+
Trust posture is reported honestly per the envelope's declared grade: a `recorded-at-acquisition` Lode states that RB attests only the digest observed at capture, never that it remains re-checkable against a vanished upstream.
