//axhob_operation
{rbtgo_director_invest}

NOTE: This operation invests a {rbtr_director} {giam_service_account} into the {rbtgi_depot_project}.
The {rbtr_governor} mints the SA, generates its credential key, and writes the RBRA bytes to the assay output slot.
The operation is singleton-per-role: no instance argument; the SA identity is derived from regime constants as `${RBCC_role_director}-${RBRR_DEPOT_MONIKER}`.
Fatal if the SA already exists — operator must {rbtgo_director_divest} first to re-key.

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` for subsequent API calls

//axhos_step
. *Derive
{rbtr_director}
Service Account Identity*:
.. {rbbc_store}
`«DIRECTOR_ACCOUNT_NAME»` as `${RBCC_role_director}-${RBRR_DEPOT_MONIKER}` (singleton per role per depot, regime-kindled via
{rbrr_depot_moniker})
.. {rbbc_store}
`«DIRECTOR_EMAIL»` as `«DIRECTOR_ACCOUNT_NAME»@${RBDC_DEPOT_PROJECT_ID}.iam.gserviceaccount.com` (depot project ID derived from `(RBRR_CLOUD_PREFIX, RBRR_DEPOT_MONIKER)`)

//axhos_step
. *Preflight: Confirm Service Account Does Not Exist*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/get[iam.serviceAccounts.get] for `projects/-/serviceAccounts/«DIRECTOR_EMAIL»`
.. {rbbc_fatal}
if HTTP 200: service account already exists — operator must execute
{rbtgo_director_divest}
before re-investing this role
.. {rbbc_require}
HTTP 404 to proceed

//axhos_step
. *Create
{rbtr_director}
Service Account*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/create[iam.serviceAccounts.create] with:
... accountId: `«DIRECTOR_ACCOUNT_NAME»`
... displayName: `Recipe Bottle Director`
... description: `Create/destroy container images for depot ${RBRR_DEPOT_MONIKER}`
... project:
{rbtgi_depot_project}
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«DIRECTOR_UNIQUE_ID»` from response `.uniqueId` field

//axhos_step
. *Verify Service Account Propagation*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/get[iam.serviceAccounts.get] for `projects/-/serviceAccounts/«DIRECTOR_EMAIL»`
.. {rbbc_poll}
with backoff until HTTP 200 (newly-created SA may not be immediately resolvable by email)
.. NOTE: Poll must use the email address, not the numeric ID. The email-based lookup path has independent propagation from the UID-based path, and subsequent steps (keys.list, keys.create) all use the email address. Polling by UID can succeed while the email path still returns 404.
.. {rbbc_require}
service account accessible by email

//axhos_step
. *Preflight: Ensure Pristine Key State*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/list[serviceAccounts.keys.list] for `«DIRECTOR_EMAIL»`
.. {rbbc_store}
`«EXISTING_KEYS»` from response `.keys[]`
.. {rbbc_require}
no keys with `keyType: USER_MANAGED` exist
.. {rbbc_fatal}
if USER_MANAGED keys found: "Manual cleanup required via Console"

//axhos_step
. *Generate Service Account Key*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/create[serviceAccounts.keys.create] for `«DIRECTOR_EMAIL»` with:
... privateKeyType: `TYPE_GOOGLE_CREDENTIALS_FILE`
.. Retry up to `RBGC_SA_KEY_CREATE_RETRY_MAX` times on HTTP 404 with `RBGC_SA_KEY_CREATE_RETRY_DELAY_SEC` delay between attempts
.. NOTE: SA read operations (GET by email, keys.list) may succeed while keys.create POST returns 404 "does not exist" due to write-path propagation delay. Google link:https://cloud.google.com/iam/docs/service-accounts-create[documents]: "you might need to wait 60 seconds or more before you use the service account." The 404 confirms no key was created, making retry safe. See <<rbscig_iam_grant_contracts>>.
.. {rbbc_store}
`«DIRECTOR_PRIVATE_KEY_DATA»` from response `.privateKeyData` field
.. {rbtoe_rbra_generate}
converts `«DIRECTOR_PRIVATE_KEY_DATA»` to `«DIRECTOR_RBRA_FILE»`

//axhos_step
. *Get Depot Project Number*:
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/get[projects.get] for
{rbtgi_depot_project}
.. {rbbc_store}
`«PROJECT_NUMBER»` from `.projectNumber` field

//axhos_step
. *Grant Project-Level Permissions*:
.. {rbtoe_iam_grant_project} for
{rbtgi_depot_project}
.. Add `serviceAccount:«DIRECTOR_EMAIL»` with roles:
... `roles/cloudbuild.builds.editor` (submit and manage builds)
... `roles/viewer` (read project resources)
... `roles/cloudbuild.workerPoolUser` (use private pool for vouch `builds.create`)
.. Policy version: 3
.. {rbbc_require}
IAM policy update successful

//axhos_step
. *Grant
{rbtr_mason}
Impersonation*:
.. NOTE:
{rbtr_director}
submits builds that execute as
{rbtr_mason}.
This requires `serviceAccountUser` permission on
{rbtr_mason}.
.. {rbtoe_iam_grant_sa} on
{rbtr_mason}
service account
.. Add `serviceAccount:«DIRECTOR_EMAIL»` with `roles/iam.serviceAccountUser`
.. {rbbc_require}
IAM binding successful

//axhos_step
. *Grant
{rbtgi_build_bucket}
Access*:
.. {rbtoe_iam_grant_bucket} on
{rbtgi_build_bucket}
.. Add `serviceAccount:«DIRECTOR_EMAIL»` with roles:
... `roles/storage.objectCreator` (upload sources)
... `roles/storage.objectViewer` (verify uploads)
.. {rbbc_require}
bucket IAM update successful

//axhos_step
. *Grant
{rbtgi_repository}
Admin Access*:
.. {rbbc_call}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories/getIamPolicy[repositories.getIamPolicy] via HTTP GET with `options.requestedPolicyVersion=3` query parameter
.. NOTE: {gar_registry} proto specifies GET for `getIamPolicy` with no `additional_bindings` for POST. Using POST fails intermittently with HTTP 404 (transcoding layer rejection, not missing repository). See <<rbscig_iam_grant_contracts>>.
.. {rbbc_require}
HTTP 200 — 404 means repository does not exist (real error, not empty policy)
.. {rbbc_call}
link:https://cloud.google.com/artifact-registry/docs/reference/rest/v1/projects.locations.repositories/setIamPolicy[repositories.setIamPolicy] for
{rbtgi_repository}
.. *Complete policy write*: Add ALL expected bindings to the read policy, regardless of what the read returned:
... `serviceAccount:«DIRECTOR_EMAIL»` with `roles/artifactregistry.repoAdmin` — Director manages images
... `serviceAccount:«MASON_EMAIL»` with `roles/artifactregistry.writer` — Mason pushes images at build time
.. NOTE: Same complete-policy-write pattern as the secret IAM step. Prevents stale read from overwriting Mason's writer binding set by {rbtgo_depot_levy}.
.. {rbbc_require}
repository IAM set with all expected bindings

//axhoot_typed_output
{rbtgi_rbra_file}
. *Deliver Credential to Assay Output Slot*:
.. {rbbc_store}
`«DIRECTOR_RBRA_FILE»` RBRA bytes to the assay output slot via the producer-output discipline (the orphan's `${BURD_OUTPUT_DIR}` credential write retires)

//axhoc_completion
Successful execution invests director service account `«DIRECTOR_ACCOUNT_NAME»` with build management permissions, generates its credential key, and writes the RBRA file to the assay output slot.
