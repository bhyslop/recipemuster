//axhob_operation
{rbtgo_retriever_invest}

NOTE: This operation invests a
{rbtr_retriever}
{giam_service_account}
into the
{rbtgi_depot_project}
with read-only
{rbtgi_repository}
access.
The
{rbtr_governor}
mints the SA, generates its credential key, and writes the RBRA bytes to the assay output slot.
The operation is singleton-per-role: no instance argument; the SA identity is derived from regime constants as `${RBCC_role_retriever}-${RBRR_DEPOT_MONIKER}`.
Fatal if the SA already exists — operator must
{rbtgo_retriever_divest}
first to re-key.

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` for subsequent API calls

//axhos_step
. *Derive
{rbtr_retriever}
Service Account Identity*:
.. {rbbc_store}
`«RETRIEVER_ACCOUNT_NAME»` as `${RBCC_role_retriever}-${RBRR_DEPOT_MONIKER}` (regime-kindled from
{rbrr_depot_moniker})
.. {rbbc_store}
`«RETRIEVER_EMAIL»` as `«RETRIEVER_ACCOUNT_NAME»@${RBDC_DEPOT_PROJECT_ID}.iam.gserviceaccount.com`
.. NOTE: `RBDC_DEPOT_PROJECT_ID` is derived from `(RBRR_CLOUD_PREFIX, RBRR_DEPOT_MONIKER)` at regime kindle time; no operator-supplied project ID field is referenced.

//axhos_step
. *Preflight: Confirm Service Account Does Not Exist*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/get[iam.serviceAccounts.get] for `projects/-/serviceAccounts/«RETRIEVER_EMAIL»`
.. {rbbc_fatal}
if HTTP 200: service account already exists — operator must execute
{rbtgo_retriever_divest}
before re-investing this role
.. {rbbc_require}
HTTP 404 to proceed

//axhos_step
. *Create
{rbtr_retriever}
Service Account*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/create[iam.serviceAccounts.create] with:
... accountId: `«RETRIEVER_ACCOUNT_NAME»`
... displayName: `Recipe Bottle Retriever`
... description: `Read-only access to Google Artifact Registry for depot ${RBRR_DEPOT_MONIKER}`
... project:
{rbtgi_depot_project}
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«RETRIEVER_UNIQUE_ID»` from response `.uniqueId` field

//axhos_step
. *Verify Service Account Propagation*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/get[iam.serviceAccounts.get] for `projects/-/serviceAccounts/«RETRIEVER_EMAIL»`
.. {rbbc_poll}
with backoff until HTTP 200 (newly-created SA may not be immediately resolvable by email)
.. NOTE: Poll must use the email address, not the numeric ID. The email-based lookup path has independent propagation from the UID-based path, and subsequent steps (keys.list, keys.create) all use the email address. Polling by UID can succeed while the email path still returns 404.
.. {rbbc_require}
service account accessible by email

//axhos_step
. *Preflight: Ensure Pristine Key State*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/list[serviceAccounts.keys.list] for `«RETRIEVER_EMAIL»`
.. {rbbc_store}
`«EXISTING_KEYS»` from response `.keys[]`
.. {rbbc_require}
no keys with `keyType: USER_MANAGED` exist
.. {rbbc_fatal}
if USER_MANAGED keys found: "Manual cleanup required via Console"

//axhos_step
. *Generate Service Account Key*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts.keys/create[serviceAccounts.keys.create] for `«RETRIEVER_EMAIL»` with:
... privateKeyType: `TYPE_GOOGLE_CREDENTIALS_FILE`
.. Retry up to `RBGC_SA_KEY_CREATE_RETRY_MAX` times on HTTP 404 with `RBGC_SA_KEY_CREATE_RETRY_DELAY_SEC` delay between attempts
.. NOTE: SA read operations (GET by email, keys.list) may succeed while keys.create POST returns 404 "does not exist" due to write-path propagation delay. Google link:https://cloud.google.com/iam/docs/service-accounts-create[documents]: "you might need to wait 60 seconds or more before you use the service account." The 404 confirms no key was created, making retry safe. See <<rbscig_iam_grant_contracts>>.
.. {rbbc_store}
`«RETRIEVER_PRIVATE_KEY_DATA»` from response `.privateKeyData` field
.. {rbtoe_rbra_generate}
converts `«RETRIEVER_PRIVATE_KEY_DATA»` to `«RETRIEVER_RBRA_FILE»`

//axhos_step
. *Grant
{rbtgi_repository}
Read Access*:
.. {rbtoe_iam_grant_project} for
{rbtgi_depot_project}
.. Add `serviceAccount:«RETRIEVER_EMAIL»` with role:
... `roles/artifactregistry.reader` (read-only access to all Artifact Registry repositories)
.. Policy version: 3
.. {rbbc_require}
IAM policy update successful

//axhoot_typed_output
{rbtgi_rbra_file}
. *Deliver Credential to Assay Output Slot*:
.. {rbbc_store}
`«RETRIEVER_RBRA_FILE»` RBRA bytes to the assay output slot via the producer-output discipline (the orphan's `${BURD_OUTPUT_DIR}` credential write retires)

//axhoc_completion
Successful execution invests retriever service account with read-only Artifact Registry access and delivers the RBRA credential bytes to the assay output slot.
