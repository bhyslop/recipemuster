//axhob_operation
{rbtgo_ark_enshrine}

NOTE: This operation enshrines upstream base images into
{gar_service}
as content-addressed, human-readable anchors.
The
{rbtr_director}
copies each declared
{rbrv_image_origin}
from its upstream registry to GAR with an anchor tag using
{rbrg_skopeo_image_ref}
(`skopeo copy --all`), preserving the manifest list and all per-platform manifests for multi-platform images.
The
{rbrv_image_anchor}
is then written back to the vessel regime.

NOTE: Enshrine is a pre-build preparation step.
It must be run before
{rbtgo_ark_conjure}
when anchored base images are desired.
Conjure works without anchors (passing ORIGIN through), but anchored images provide reproducibility and air-gap readiness.

The enshrine operation:

//axhopt_typed_parameter
{rbrv_sigil}
. *Validate Input Parameters*:
.. {rbbc_require}
vessel provided as first argument
.. {rbbc_require}
vessel contains at least one
{rbrv_image_origin}
declaration (slots 1-3)

//axhos_step
. *Enshrine Each Image Slot*:
.. For each indexed slot (n = 1, 2, 3) where
{rbrv_image_origin}
is set:
... Inspect upstream image to obtain the manifest digest
... {rbbc_store}
`<<DIGEST>>` -- the sha256 digest of the upstream manifest (manifest list digest for multi-platform images, single manifest digest for single-platform images)
... Construct anchor: sanitize origin tag (`:` `/` replaced with `-`), append first 10 hex chars of sha256 digest
... Example: `python:3.11-slim` with `sha256:abc123def4...` produces `python-3.11-slim-abc123def4`
... Copy upstream image to GAR with anchor tag using `skopeo copy --all`, transferring the manifest list and all per-platform manifests atomically
... {rbbc_require}
copy succeeds -- the anchor tag is now a permanent, content-addressed reference in
{gar_service}
containing all upstream platforms
... Single-platform upstream images are handled identically -- `skopeo copy --all` is a no-op on the platform dimension when only one platform exists
... Write anchor string to `RBRV_IMAGE_n_ANCHOR` in the vessel's `rbrv.env`

//axhoot_typed_output
{rbrv_image_anchor}
Per-slot anchor string written to the vessel regime.

//axhoc_completion
Successful execution produces anchored base image references in
{gar_service}
(with all upstream platforms preserved) and updates the vessel regime with
{rbrv_image_anchor}
values.
The vessel is now ready for reproducible
{rbtgo_ark_conjure}
builds using GAR-mirrored base images.
Multi-platform conjure vessels pull the correct platform from the anchored manifest list automatically.
Re-running enshrine with the same
{rbrv_image_origin}
but a different upstream digest produces a new anchor (upstream image was updated).
Re-running with the same digest is idempotent.
