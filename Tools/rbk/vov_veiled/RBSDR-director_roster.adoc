//axhob_operation
{rbtgo_director_roster}

The
{rbtr_governor}
lists all
{rbtr_director}
{giam_service_account}
instances within the
{rbtgi_depot_project},
emitting a presence fact per discovered account.
This is a read-only observation operation; no SA credentials are created or modified.

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` from the resulting access token

//axhos_step
. *List service accounts*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/list[iam.projects.serviceAccounts.list] with:
... method: `GET`
... endpoint: `https://iam.googleapis.com/v1/projects/{rbtgi_depot_project}/serviceAccounts`
... header: `Authorization: Bearer «GOVERNOR_TOKEN»`
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«ACCOUNTS»` from response `.accounts[]` array

//axhos_step
. *Filter to director service accounts*:
.. From `«ACCOUNTS»`, retain only entries whose `.accountId` begins with the `${RBCC_role_director}-` prefix
.. {rbbc_store}
`«DIRECTOR_ACCOUNTS»` — the filtered set
.. If `«DIRECTOR_ACCOUNTS»` is empty:
... {rbbc_show}
"No director service accounts found in project"
... Return success

//axhos_step
. *Emit per-account fact files*:
.. For each entry in `«DIRECTOR_ACCOUNTS»`:
... {rbbc_store}
`«IDENTITY»` by stripping the `${RBCC_role_director}-` prefix from the entry's `.accountId`
... {rbbc_call}
{buf_write_fact_multi} with file root, extension constant `RBCC_fact_ext_roster_director`
(expands to `roster-director`), and basename `«IDENTITY»`
... Fact content is empty — presence of the fact file is the datum (presence-as-fact convention)

//axhos_step
. *Report count*:
.. {rbbc_show}
total count of emitted director fact files

//axhoc_completion
Successful execution rosters
{rbtr_director}
service accounts via per-SA fact emission, one `roster-director` fact file per discovered director identity.
