//axhob_operation
{rbtgo_director_roster}

NOTE: This operation rosters {rbtr_director} {giam_service_account_s} in the {rbtgi_depot_project} via per-SA fact-file emission.
Read-only observation: lists all SAs, filters to director-prefixed accountIds, and emits one fact per discovered SA via {buf_write_fact_multi} against the `RBCC_fact_ext_roster_director` extension constant.
Fact content is empty (presence-as-fact); stdout assertion of individual identities is retired.

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` from the resulting access token

//axhos_step
. *List service accounts*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/list[iam.projects.serviceAccounts.list] with:
... method: `GET`
... endpoint: `https://iam.googleapis.com/v1/projects/{rbtgi_depot_project}/serviceAccounts`
... header: `Authorization: Bearer «GOVERNOR_TOKEN»`
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«ACCOUNTS»` from response `.accounts[]` array

//axhos_step
. *Filter to director service accounts*:
.. From `«ACCOUNTS»`, retain only entries whose `.accountId` begins with the `${RBCC_role_director}-` prefix
.. {rbbc_store}
`«DIRECTOR_ACCOUNTS»` — the filtered set; per-role observation per BBAAQ contract

//axhos_step
. *Emit Fact Per Discovered {rbtr_director}*:
.. For each entry in `«DIRECTOR_ACCOUNTS»`:
... {rbbc_store}
`«IDENTITY»` by stripping the `${RBCC_role_director}-` prefix from the entry's `accountId`
... {rbbc_call}
{buf_write_fact_multi}
with three positional arguments:
.... `<file_root>`: composed as `<roster_subdir>/«IDENTITY»` (the basename root for the emitted fact file)
.... `<ext>`: `RBCC_fact_ext_roster_director` (resolves at source time to `roster-director`)
.... `<value>`: `""` (empty content — presence-as-fact convention per RBS0:1081)

//axhos_step
. *Report count*:
.. {rbbc_show}
total count of emitted director fact files

//axhoc_completion
Successful execution rosters {rbtr_director} service accounts via per-SA fact emission — one `roster-director` fact file per discovered identity.
The operation is read-only; no service accounts are created or modified.
