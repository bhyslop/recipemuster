//axhob_operation
{rbtgo_director_roster}

NOTE: This operation rosters
{rbtr_director}
{giam_service_account_s}
in the
{rbtgi_depot_project}
via per-SA fact-file emission.
Read-only observation: lists all SAs, filters to director-prefixed emails, and emits one fact per discovered SA via
{buf_write_fact_multi}
against the `RBCC_fact_ext_roster_director` extension constant.
Fact value is the full SA email; `file_root` is the bare identity extracted from the email.

//axhos_step
. *Authenticate as
{rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` from the resulting access token

//axhos_step
. *List service accounts*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/list[iam.projects.serviceAccounts.list] with:
... method: `GET`
... endpoint: `https://iam.googleapis.com/v1/projects/{rbtgi_depot_project}/serviceAccounts`
... header: `Authorization: Bearer «GOVERNOR_TOKEN»`
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«ACCOUNTS»` from response `.accounts[]` array

//axhos_step
. *Filter to director service accounts*:
.. From `«ACCOUNTS»`, read the `.email` field of each entry and test whether it matches `^${RBCC_account_director}-(.+)@${RBGD_SA_EMAIL_FULL}$`
.. {rbbc_store}
`«DIRECTOR_EMAILS»` — the matching emails; per-role observation per BBAAQ contract

//axhos_step
. *Emit Fact Per Discovered {rbtr_director}*:
.. For each `«SA_EMAIL»` in `«DIRECTOR_EMAILS»`:
... {rbbc_store}
`«IDENTITY»` from the regex capture group — the portion of `«SA_EMAIL»` between `${RBCC_account_director}-` and `@${RBGD_SA_EMAIL_FULL}`
... {rbbc_call}
{buf_write_fact_multi}
with three positional arguments:
.... `<file_root>`: `«IDENTITY»` (bare identity — the basename root for the emitted fact file)
.... `<ext>`: `RBCC_fact_ext_roster_director` (resolves at source time to `roster-director`)
.... `<value>`: `«SA_EMAIL»` (the full service account email address)

//axhos_step
. *Report count*:
.. {rbbc_show}
total count of emitted director fact files

//axhoc_completion
Successful execution rosters {rbtr_director} service accounts via per-SA fact emission — one `roster-director` fact file per discovered identity, each containing the full SA email as its value.
The operation is read-only; no service accounts are created or modified.
