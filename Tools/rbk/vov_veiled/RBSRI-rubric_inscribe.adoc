//axhob_operation
{rbtgo_rubric_inscribe}

The single
{rbtgr_rubric}
lifecycle command.
Batch-processes all conjure vessels: clones the
{rbtgr_rubric_repo},
copies build contexts, generates all
{rbtgr_build_json}
directly into the clone, fills placeholders, pushes,
and ensures all triggers exist.

//axhoq_precondition
* {rbrr_cbv2_connection_name} configured in regime

//axhoq_precondition
* {rbrr_rubric_repo_url} configured in regime

//axhoq_precondition
* GitLab project access token accessible in Secret Manager (`RBGC_CBV2_API_TOKEN_SECRET_NAME`)

//axhoq_precondition
* Vessel configurations loadable ({rbrv_regime})

//axhos_step
. *Pin Staleness Gate*:
.. Verify `RBRG_IMAGE_PINS_REFRESHED_AT` is less than 1 day old
.. If stale: die with guidance to run image pin refresh ({rbtc_refresh_gcb_pins}) manually, commit, and re-run inscribe
.. `rbrr_refresh_gcb_pins` remains independently callable for manual inspection

//axhos_step
. *Fetch GitLab token from Secret Manager*:
.. {rbbc_call}
link:https://cloud.google.com/secret-manager/docs/reference/rest/v1/projects.secrets.versions/access[secrets.versions.access] for `projects/{rbrr_depot_project_id}/secrets/RBGC_CBV2_API_TOKEN_SECRET_NAME/versions/latest`
.. {rbbc_store}
`«GITLAB_TOKEN»` from response `.payload.data` (base64-decoded)
.. {rbbc_require}
token retrieved successfully or {rbbc_fatal} "Cannot access GitLab token in Secret Manager — verify Director SA has secretAccessor role"
.. Construct authenticated URL using bash builtin: `https://oauth2:«GITLAB_TOKEN»@` + URL path from
{rbrr_rubric_repo_url}
.. NOTE: Authenticated URL is ephemeral — constructed in memory, used for git operations, discarded after use. Never persisted.

//axhos_step
. *Clone Rubric Repo and Generate All Rubric JSON*:
.. Delete local clone at `.rbk/rubric/` if it exists
.. `git clone --depth 1` the
{rbtgr_rubric_repo}
using the ephemeral authenticated URL
.. For each conjure vessel in `rbev-vessels/`:
... Delete stale per-vessel directory in clone
... Copy entire vessel directory as build context to per-vessel directory in clone
... Load vessel configuration via `zrbf_load_vessel()`
... Generate
{rbtgr_build_json}
directly into the clone via `zrbf_stitch_build_json()` from step scripts (`rbgjb/*.sh`) and regime pins
... All values baked: GAR coordinates, builder image pins (digest-pinned), vessel identity, platforms
... The generated JSON produces only the `-image` artifact; about and vouch are handled by subsequent pipeline stages.
Step scripts `rbgjb06` through `rbgjb08` (syft, build-info, about push) are excluded from the stitched JSON — these are now in the standalone about pipeline (`rbgja/`).
Remaining steps `rbgjb01-05` + `rbgjb09` renumber to `rbgjb01-06`
... Substitutions block includes four placeholders: `_RBGY_RUBRIC_REPO`, `_RBGY_RUBRIC_COMMIT`, `_RBGY_GIT_COMMIT`, and `_RBGY_INSCRIBE_TIMESTAMP` — filled before push (next step)
... All other values deterministic: same inputs produce byte-identical output
.. Fill all four placeholders in each
{rbtgr_build_json}:
`_RBGY_RUBRIC_REPO` (actual URL), `_RBGY_GIT_COMMIT` (main repo HEAD), `_RBGY_INSCRIBE_TIMESTAMP` (current invocation) before commit; `_RBGY_RUBRIC_COMMIT` (rubric repo HEAD) after commit
.. One commit and push for all vessels (fully orchestrated — main repo is the reviewed source of truth)
.. Discard authenticated URL from memory

NOTE: Always-fresh-clone eliminates stale state.
The
{rbtgr_rubric_repo}
is small (per-vessel directories with Dockerfile + build context + JSON) — shallow clone is sub-second.

//axhos_step
. *Ensure Infrastructure (for each vessel)*:
.. Verify
{rbtgi_cbv2_repository}
exists within the
{rbtgi_cbv2_connection}
(created by
{rbtgo_depot_create})
.. Create
{rbtgr_vessel_trigger}
if it does not exist (via Cloud Build v1 triggers REST API)
.. Trigger body uses `repositoryEventConfig` (CB v2 repository reference, unmatchable push branch filter for manual-dispatch-only semantics) and `filename` (per-vessel path to {rbtgr_build_json})
.. `repositoryEventConfig.repositoryType` is server-inferred from the connection — never set by the client
.. Trigger name derived from constant prefix + vessel sigil

//axhoo_output_of_type
{rbtgr_rubric_repo}
* All conjure vessel directories synced in {rbtgr_rubric_repo} with current build material (including filled rubric repo metadata)
* {rbtgi_cbv2_repository} verified within the {rbtgi_cbv2_connection}
* {rbtgr_vessel_trigger} for each conjure vessel pointing to its
{rbtgr_build_json}
in the
{rbtgr_rubric_repo}

//axhoc_completion
NOTE: Inscribe is idempotent.
A fresh vessel (no directory in rubric repo, no trigger) gets all created.
An existing vessel gets its JSON and build context synced — infrastructure resources are left in place.
One rubric repo commit covers all vessels at their current pin state.
Proceed to
{rbtgo_trigger_build}
to dispatch a build for any individual vessel.
