//axhob_operation
{rbtgo_payor_refresh}

Manual procedure to install OAuth credentials for new setup or refresh expired/compromised credentials.
Use this for both initial credential installation after
{rbtgo_payor_establish}
and for renewal operations.

//axhos_step
. *Obtain OAuth Credentials*:
.. For initial setup: Use JSON file downloaded during
{rbtgo_payor_establish}
.. For refresh/renewal: Navigate to APIs & Services > Credentials in Payor Project
... Click existing "Recipe Bottle Payor" OAuth client to open details
... If secret compromised:
.... Click "+ Add secret"
.... Click download icon next to NEW secret
.... Browser downloads `client_secret_[id].apps.googleusercontent.com.json`
.... Click "Disable" on secret with older creation date
.... Click trash icon to delete that disabled secret

//axhos_step
. *Install/Refresh Credentials*:
.. Run: `rbgp_payor_install ~/Downloads/client_secret_*.json`
.. This will:
... Guide you through OAuth authorization flow
... Store secure credentials in `~/.rbw/payor/rbro.env`
... Test the authentication
... Initialize depot tracking
... Update `RBRP_OAUTH_CLIENT_ID` in `rbrp.env`

//axhog_guarantee
* `rbgp_depot_list` successfully displays current depots without authentication errors

//axhoc_completion
Procedure is complete when `rbgp_depot_list` successfully displays current depots without authentication errors.

NOTE: OAuth refresh tokens in testing mode expire after 6 months of non-use.
