//axhob_operation
{rbtgo_depot_list}

The
{rbtr_payor}
lists all
{rbtgi_depot}
instances billed to the
{rbtr_payor}
account and their status:

//axhos_step
. *Authenticate as
{rbtr_payor}*:
.. Execute
{rbtoe_payor_authenticate}

//axhos_step
. *Enumerate billing-account projects*:
.. {rbbc_call}
link:https://cloud.google.com/billing/reference/rest/v1/billingAccounts.projects/list[billingAccounts.projects.list]
for `billingAccounts/${RBRP_BILLING_ACCOUNT_ID}/projects`
.. {rbbc_store}
`«BILLED_PROJECTS»` — projects with `billingEnabled=true`

//axhos_step
. *Classify each project as a depot*:
.. For each project in `«BILLED_PROJECTS»`:
... {rbbc_call}
link:https://cloud.google.com/iam/reference/rest/v1/projects.serviceAccounts/list[projects.serviceAccounts.list]
for the project's IAM service accounts
... Inspect service account emails for a match against
`mason-«MONIKER»@<project>.iam.gserviceaccount.com`
where `«MONIKER»` matches `^[a-z][a-z0-9]*$`
... Skip projects with no matching Mason service account
... {rbbc_store}
`«MONIKER»` by stripping the `mason-` prefix from the matched SA's local-part

//axhos_step
. *Look up lifecycle state for each depot project*:
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/get[projects.get]
for each Mason-SA-bearing project via CRM v1 `projects/${project_id}`
.. Classify state:
... `lifecycleState=ACTIVE` → `COMPLETE`
... `lifecycleState=DELETE_REQUESTED` → `DELETE_REQUESTED`
... Other lifecycle states → log and skip

//axhos_step
. *Emit per-moniker fact files*:
.. {rbbc_store}
state string to `«MONIKER».depot`
(extension constant `RBGP_FACT_EXT_DEPOT`)
.. {rbbc_store}
canonical project_id to `«MONIKER».depot-project`
(extension constant `RBGP_FACT_EXT_DEPOT_PROJECT`)

//axhoot_typed_output
{rbtgi_depot}
. *Display summary*:
.. {rbbc_show}
each depot moniker with state (`COMPLETE` or `DELETE_REQUESTED`)
.. {rbbc_show}
total depot count

//axhoc_completion
Successful execution lists all depots billed to the
{rbtr_payor}
account.
