//axhob_operation
{rbtgo_depot_list}

The
{rbtr_payor}
lists all
{rbtgi_depot}
instances visible under the
{rbtr_payor}
account and their status:

//axhos_step
. *Authenticate as
{rbtr_payor}*:
.. Execute
{rbtoe_payor_authenticate}

//axhos_step
. *Search depots by displayName anchor*:
.. {rbbc_call}
link:https://cloud.google.com/resource-manager/reference/rest/v3/projects/search[cloudresourcemanager.googleapis.com/v3/projects:search]
with `query=displayName:«RBGC_DEPOT_DISPLAY_PREFIX»*`
.. {rbbc_store}
`«SEARCH_RESULTS»` — paginated list of matching projects; follow `nextPageToken` until exhausted
.. NOTE: CRM v3 search returns both `ACTIVE` and `DELETE_REQUESTED` projects in the default response — no state filter is applied.
This means soft-deleted depots within their 30-day retention window (ghost depots) are visible alongside active ones.
This visibility is load-bearing: the canonical-establish fixture's auto-pick logic must see `DELETE_REQUESTED` projects to
avoid re-selecting a moniker that is still in the soft-delete graveyard.

//axhos_step
. *Derive moniker and state per result*:
.. For each project in `«SEARCH_RESULTS»`:
... {rbbc_store}
`«MONIKER»` by stripping the `«RBGC_DEPOT_DISPLAY_PREFIX» ` prefix from the project's `displayName`
... {rbbc_require}
`«MONIKER»` matches `^[a-z][a-z0-9]*$`; if not, log and skip (defensive against non-depot displayName matches)
... Classify state:
.... `state=ACTIVE` → `COMPLETE`
.... `state=DELETE_REQUESTED` → `DELETE_REQUESTED`
.... Other states → log and skip

//axhos_step
. *Emit per-moniker fact files*:
.. {rbbc_store}
state string to `«MONIKER».depot`
(extension constant `RBGP_FACT_EXT_DEPOT`)
.. {rbbc_store}
canonical project_id to `«MONIKER».depot-project`
(extension constant `RBGP_FACT_EXT_DEPOT_PROJECT`)

//axhoot_typed_output
{rbtgi_depot}
. *Display summary*:
.. {rbbc_show}
each depot moniker with state (`COMPLETE` or `DELETE_REQUESTED`)
.. {rbbc_show}
total depot count

//axhoc_completion
Successful execution lists all depots visible to the
{rbtr_payor}
account, including soft-deleted depots still within their 30-day retention window.
