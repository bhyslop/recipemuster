== Terrier

The Terrier is a polity's durable, cloud-resident, authoritative record of which
citizen holds which mantle. It is the one record in the federation design that
outlives every operator's machine: the source of truth read by the rehearse verb
and by the read-only reconciliation diff, written by the admission verbs. It
lives in a manor-homed Cloud Storage bucket, one entry — a *muniment*, recording
(principal subject, mantle held) — per object.

=== Write semantics — atomic, server-adjudicated (settled)

Governor writes to the Terrier are made atomic by Cloud Storage itself. No
Cloud Build invocation, and no external lock, is involved.

* *Create is conditional on absence.* The object write carries the
  `ifGenerationMatch=0` precondition, so Cloud Storage performs the write only if
  no live object exists at that key. Concurrent creators race cleanly: one
  succeeds; the others receive `HTTP 412 Precondition Failed` and treat an
  already-present identical muniment as idempotent success.
* *Update is conditional on generation.* Where a muniment is rewritten rather
  than created, the write carries `ifGenerationMatch=<generation>` read
  beforehand — optimistic concurrency control. The write lands only if the object
  has not changed since the read; otherwise the 412 directs a read-and-retry.
* Cloud Storage guarantees these preconditions are evaluated atomically and free
  of race conditions. There is no multi-object transaction across muniments, and
  none is needed: each muniment is an independent object, so distinct governors
  writing distinct entries never contend, and the precondition covers the rare
  same-entry case.

Each write is therefore a single conditioned REST call: the bucket adjudicates
atomicity, the caller adds one query parameter and checks the HTTP code. This is
within bash's good case under BCG — glue over a service, carrying no lock logic
of its own.
