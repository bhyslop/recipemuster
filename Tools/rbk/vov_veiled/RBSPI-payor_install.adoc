//axhob_operation
{rbtgo_payor_install}

The
{at_operator}
completes OAuth credential installation by running the automated installation command with the downloaded OAuth client JSON file.
This step performs the OAuth authorization flow, securely stores credentials, and validates the {rbtgi_manor} is accessible.

//axhoq_precondition
* {rbhg_require}
{rbrp_billing_account_id}
is set in the current shell environment

//axhoq_precondition
* {rbhg_require}
OAuth client JSON file from
{rbtgo_payor_establish}
is saved in its durable home — the `client_secrets/` subdirectory of {rbrr_secrets_dir}

//axhos_step
. *Obtain or rotate the OAuth client JSON* (only when the client itself is lost or compromised):
.. For a first install, use the JSON saved during
{rbtgo_payor_establish}.
.. A dead refresh token needs no new JSON —
re-run this installation with the same saved file
(the {rbro_regime} lifecycle statement names the death modes).
.. If the client secret was compromised or its JSON is lost, mint a fresh secret:
... Open APIs & Services > Credentials in the {rbtgi_manor} and click the existing
"Recipe Bottle Payor" OAuth client
... Click "+ Add secret", then the download icon beside the NEW secret; the browser
downloads `client_secret_[id].apps.googleusercontent.com.json`
... Click "Disable" on the secret with the older creation date, then the trash icon to
delete that disabled secret
... Save the fresh JSON to the durable home, replacing the dead one

//axhos_step
. *Execute OAuth Credential Installation*:
.. Export
{rbrp_billing_account_id}
to environment (obtained from {rbtgo_payor_establish} step 2)
.. Run installation command with path to downloaded OAuth JSON:
+
[source,bash]
----
export RBRP_BILLING_ACCOUNT_ID=<your-billing-account-id>
rbgp_payor_install «RBRR_SECRETS_DIR»/client_secrets/client_secret_*.json
----

.. The glob assumes one saved JSON in the durable home;
if more than one exists, pass the exact path
.. Script validates OAuth JSON file structure
.. Script performs OAuth authorization flow (loopback redirect):
... Presents a Google OAuth consent URL to open in the browser
... User signs in and grants Recipe Bottle Payor access to cloud resources
... The browser lands on an unreachable localhost page — expected;
the authorization code rides the address-bar URL
... User copies the full URL from the address bar and pastes it at the terminal prompt

//axhos_step
. *Credential Storage*:
.. OAuth credentials stored securely in the payor `rbro.env` under «RBRR_SECRETS_DIR» with 600 permissions
.. Credentials include:
... `RBRO_CLIENT_SECRET`: OAuth client secret for token refresh
... `RBRO_REFRESH_TOKEN`: Refresh token for authentication (renew by re-installing)
.. {rbhg_warn}
Never commit the payor `rbro.env` under «RBRR_SECRETS_DIR» to version control

//axhoot_typed_output
{rbro_regime}
* OAuth credentials stored in the payor `rbro.env` under «RBRR_SECRETS_DIR»:
** `RBRO_CLIENT_SECRET`: OAuth client secret for token refresh
** `RBRO_REFRESH_TOKEN`: Refresh token for authentication (renew by re-installing)

//axhoot_typed_output
{rbrp_regime}
* Installation displays payor configuration values needed in the {rbrp_regime}:
** `RBRP_PAYOR_PROJECT_ID`: Payor project ID from OAuth JSON
** `RBRP_OAUTH_CLIENT_ID`: OAuth client ID from OAuth JSON
** `RBRP_BILLING_ACCOUNT_ID`: Already provided before installation

//axhoc_completion
Successful execution stores OAuth credentials and displays payor configuration values.
Record configuration values for {rbrp_regime} setup.
Proceed to
{rbtgo_manor_instaurate}
to found the manor's scriptable substrate (payor-project APIs, billing linkage,
workforce pool, terrier), then
{rbtgo_depot_levy}
to create depot projects.
