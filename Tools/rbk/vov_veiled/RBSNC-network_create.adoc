//axhob_operation
{mkr_network_create}

NOTE: Network creation is now managed declaratively by Docker Compose.
The `networks:` section of `rbob_compose.yml` defines both networks; the
{at_enclave_network}
carries a static subnet from the nameplate regime.
Compose creates both the
{at_transit_network}
and
{at_enclave_network}
automatically during `docker compose up`.

The
{at_enclave_network}
uses:

* driver: bridge
* subnet: `${RBRN_ENCLAVE_BASE_IP}/${RBRN_ENCLAVE_NETMASK}` (from nameplate env-file)
* static IPs: sentry at `${RBRN_ENCLAVE_SENTRY_IP}`, pentacle at `${RBRN_ENCLAVE_BOTTLE_IP}`

The
{at_transit_network}
provides internet connectivity via the Docker bridge.
Each nameplate uses a distinct /24 subnet to prevent cross-nameplate collisions.

//axhoc_completion
Both networks are created as part of the compose lifecycle.
The
{at_sentry_container}
connects to both networks; the
{at_pentacle_container}
connects only to the enclave; the
{at_bottle_container}
shares the pentacle's network namespace via compose `network_mode: "service:pentacle"`.
