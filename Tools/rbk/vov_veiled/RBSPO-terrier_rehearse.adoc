//axhob_operation
{rbtgo_terrier_rehearse}

NOTE: This operation recounts the manor-wide muniment roll — the
{rbtf_rehearse}
act (medieval sense: to recount a record formally, in order). It reads every
{rbtf_muniment}
across the whole
{rbtf_terrier}
and emits the (mantle, subject) record. It mutates nothing: no muniment is written or
withdrawn and no IAM binding is touched. Rehearse recounts what the record says; the
spec-interior reconciliation diff — which checks the record against the live mantle-SA
bindings — is a distinct, two-distinctions-apart concern and not this operator verb.

NOTE: The
{rbtr_governor}
performs the rehearse, wielding the donned governor
{rbtf_mantle}.
The read is bucket-level and manor-wide: a governor mantle reads every polity's terrier
folder (`roles/storage.objectViewer` on the bucket), so rehearse recounts the whole
manor, not one depot. It takes no parameters.

NOTE: *Depot-blind emission.* The manor-wide read factors the per-polity peruse's
list-and-fetch into a shared lister whose manor-wide entry point reads the whole bucket
prefix-blind. It emits the (mantle, subject) record per muniment; the depot stays the
object-name index — recoverable from the key, but unemitted — because the muniment's
folder placement already is the principal-by-depot join, and no consumer of the roll
needs a per-entry depot column.

NOTE: *Pure-read race tolerance.* The manor sweep is a list-then-fetch, so an object the
listing named can be withdrawn by a concurrent
{rbtf_unseat}
before its fetch. A read-after-list `404` is a benign vanish — the lister skips that
entry rather than failing — so a rehearsal never crashes on a holding torn out from
under it.
