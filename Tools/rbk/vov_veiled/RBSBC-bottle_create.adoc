//axhob_operation
{mkr_bottle_create}

NOTE: This sequence within
{opbr_bottle_run}
describes what Docker Compose does when creating a transient
{at_bottle_container}
via `docker compose run`.
Compose inherits the full bottle service definition — image, network mode, volumes, and dependencies — from the base compose file and optional nameplate fragment.

//axhos_step
. *Compose resolves container configuration*:
.. Image: `{rbrn_bottle_vessel}:{rbrn_bottle_consecration}` from compose service definition
.. Network: inherits `network_mode: "service:pentacle"` (shares pentacle's network namespace, which routes through sentry)
.. Volumes: declared in per-nameplate compose fragment (`«moniker».compose.yml`), not in nameplate env variables
.. Environment: forwarded from compose env-files and service `environment:` section

//axhos_step
. *Compose creates transient container*:
.. Container created from resolved service definition with `--rm` (automatic removal on exit)
.. Compose assigns container name (not user-specified for transient runs)
.. {rbbc_require}
container created successfully

//axhoc_completion
Successful creation yields a transient container with the same security posture as a sessile bottle — same network namespace, same volume mounts, same sentry enforcement.
