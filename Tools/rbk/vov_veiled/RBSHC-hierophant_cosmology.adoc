The
{rbth_hierophant}
conducts the delivery ceremony as the
{rbtt_theurge}
conducts the tests: a veiled Rust application owning four commands, one per
ceremony act —
{rbth_essai}
the repair lap,
{rbth_docimasy}
the proving act of the reveal,
{rbth_ostend}
the showing that closes it,
{rbth_harbinger}
the stranger rig.
This subdoc treats the cosmology the application embodies — the vocabulary
defined with the rest of the codex's terms at the branch head — and defers
every worker internal, fixture membership, and per-release operational step to
its own home.

== The command seam

The ceremony's one load-bearing boundary is reversible versus irreversible,
and the command structure is that boundary made executable.
{rbth_essai},
the
{rbth_docimasy},
and
{rbth_harbinger}
are wholly reversible: every artifact they produce — a
{rbth_candidate},
a
{rbth_rig},
a preview that dies with its private
{rbth_quarantine},
a
{rbth_cachet}
— is local or private, disposable, and replaceable by another run.
{rbth_ostend}
alone crosses the line: it conducts the operator through the
{rbth_disclosure}
and the promotion, and it is therefore built as machine-asserts-around-human-
pushes — the application performs every check and never holds a remote write.

The reveal ceremony is conducted in two acts split exactly at that line
(operator ruling, 260718): the
{rbth_docimasy}
proves — quarantine gates, freshness, the preview, credential preflight, the
gauntlet — and grants the
{rbth_cachet};
{rbth_ostend}
shows — the operator's eyes, the
{rbth_disclosure},
promotion.
The cachet is the bridge that keeps the ordering machine-enforced across the
act boundary: ostend refuses to conduct without a standing cachet whose tree
hash matches the standing candidate, so the once-per-cycle law survives
interruption, re-entry, and the days that may pass between the acts, rather
than living in one process's memory.

Iteration belongs entirely to the reversible tier: a
{rbth_lap}
is repeated until one runs dry — a lap surfacing no finding — and only a dry
candidate proceeds to the reveal.
The gauntlet runs once per repair cycle, inside the
{rbth_docimasy},
after the candidate stands on the
{rbth_quarantine}
— never inside a lap.

== Worker, never authority

The
{rbth_hierophant}
sequences proven workers; it absorbs orchestration, never authority —
with the one ruled exception of the cut, treated in its own section below.
Marshal zero, lustration, and feigning remain bash verbs in their own
modules, invoked as subprocesses; the proscription they share stays homed in
its shipped library, and the hierophant reads it through the same subprocess
reach the damnatio fixture already proves — a second copy of that table is
the drift this rule forbids.
Beside the cut, one other truly absorbed module is
{rbth_rig}
construction — clone, sever, hook, walk branch, prompt — which reads no shared
authority: one implementation serves both the
{rbth_essai}
rig stage and the
{rbth_harbinger}
command, differing only in clone source.
The veil assay is absorbed on the same footing: the content-grain leak scan —
census, matcher, self-proof — that once rode as a shipped theurge fixture is
authored in Rust inside the crate and run in-process by
{rbth_essai},
pre-cut on the maintainer tree and post-cut on the candidate's root context
document.
It is absorbed for a second reason the cut and the rig are not: a leak-scanner
that enumerates the withheld documents by name cannot itself ship, so its only
lawful home is behind the veil.

== The cut, and the single matcher

The cut is the
{rbth_hierophant_p}
own (operator re-ruling, 260718, reversing the 260717 expede-as-worker
cinch): the perambulation and the expede constructor are authored in Rust
inside the crate,
run in-process by
{rbth_essai}
— no bash reach, no subprocess between the conductor and the judgment that
stands between the private tree and the public remote.

The *perambulation* is the ship/withhold judgment over every tracked path —
the delivered file set as a table the project rules once and proves
mechanically, never a list of removals recited in prose.
A row is a literal path prefix; the longest matching row wins,
so precedence is a property of the rows themselves and no row can be
silently shadowed by an earlier one; there is no glob syntax, deliberately —
a glob's precedence is either order-dependent or ambiguous, and both are how
a withheld path ends up shipped.
The judgment is *total*, with no default in either direction:
a tracked path no row judges is red,
and a row that judges no tracked path — stale or shadowed — is red.

*Expede* is the constructor that cuts from it: clone the public base,
sever the origin, open the sole candidate branch, materialize the shipped
paths from the committed bytes, sterilize, transpose the consumer context,
commit once — then sweep the candidate's object *graph*, not its tree,
because a withheld path rides wherever it is reachable from the branch, at
any depth of history (the 292-MiB lesson: a clean tip over a leaking past).

The *single-matcher rule*, homed here: the ship/withhold judgment has exactly
one implementation — the Rust matcher in the hierophant crate.
Expede cuts from it, the sweep enforces it, and the crate's own self-proofs
stand for it: a planted withheld path must be caught, and the totality check
must close.
A second matcher live beside it, in any language, is forbidden — two
matchers drift, and drift is how a withheld path ships.
The rule's reach is the ship/withhold *path* judgment, and nothing wider: the
absorbed veil assay is a distinct grain — does a shipping file *name* a
withheld thing, not does a path ship — so it stands beside the perambulation
without being a second implementation of it, and it never authors a
ship/withhold verdict of its own.

One boundary is deliberately not absorbed: the candidate's own sterilize
remains a subprocess, because it must be the *candidate's copy* — a process
cannot regenerate from lustrated values unless its modules are the lustrated
modules, and invoking the clone's own script is what makes the maintainer's
modules structurally out of reach.

== Structural incapacity

Every public act is human-hands-only by structural incapacity, never by
discipline.
The
{rbth_hierophant}
never holds a push: it prints the push line for the operator to type, then
verifies the result.
The same law holds at every layer beneath it.
The base — the real public repository, remote `ENGROSSMENT_UPSTREAM` — is
read-only by construction: its push side is neutered to a fixed sentinel the
cut asserts before it will proceed.
The
{rbth_candidate}
holds zero remotes and a sole branch, `POSTULANT_LOCAL`.
The
{rbth_rig}
severs its origin and installs a pre-push refusal hook.
A push-capable remote to the public target in the maintainer repository is the
one forbidden catastrophe — a stray push landing the private tree on public
`main`.

Additive construction is the prevention; the
{rbth_quarantine}
is only containment, and containment never substitutes for prevention.
The candidate is built by addition in a clone of the real public base —
the cut materializes only the paths the perambulation ships — so no private
object ever enters the graph a push walks.

== The three public acts

Three distinct public acts, one irreversible.
Every push is an explicit URL plus a single explicit refspec; the fan-out
forms are forbidden.

The *preview* is reversible: the
{rbth_candidate}
pushed into the private
{rbth_quarantine}
as `POSTULANT_LOCAL:POSTULANT_LOCAL`.
A git commit is content-addressed, so the previewed object is, bit for bit,
the object later disclosed — that identity is what makes the preview
trustworthy.

The
{rbth_disclosure}
is the single irreversible act: the candidate pushed onto the public
repository as the unmerged `POSTULANT_LOCAL` branch, `main` untouched.
The staging push, not the promotion, carries promotion-grade gravity — a
public object store cannot be un-disclosed.

*Promotion* is discoverability, not disclosure: fast-forward-only, from a
fresh clone or fetch of the public repository, asserting tree-hash equality —
the byte claim is checked, not assumed.
The `POSTULANT_LOCAL:main` refspec occurs exactly once in the whole corpus,
at promotion; the preview and the disclosure both spell
`POSTULANT_LOCAL:POSTULANT_LOCAL`.
Standing doctrine: the greenfield walk gates promotion, never disclosure.
(The first delivery ran stage → promote → walk under an operator ruling; the
ceremony document records that exception beside this standing pattern.)

== The veil

The
{rbth_hierophant}
crate lives under the veiled tree and is referenced by no shipped manifest:
the delivered tree must build with the veiled directory absent.
It carries its own withheld build tabtarget.
Marshal operations are invisible to
{rbtr_consumer_s};
the release-hygiene fixture holds the path half of that line mechanically.
