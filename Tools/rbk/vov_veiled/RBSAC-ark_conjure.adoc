//axhob_operation
{rbtgo_ark_conjure}

NOTE: This operation conjures an
{rbtga_ark_image}
in
{rbtgi_repository}
by building a container image from a vessel specification. The
{rbtr_director}
performs the build via
{gcb_service}.

NOTE: Conjure produces both the `-image` and `-about` artifacts in a single trigger-dispatched Cloud Build job.
The about steps (`rbgja01`–`rbgja04`) are embedded via stitch alongside the build steps.
The `-vouch` artifact is created by the subsequent {rbtgo_ark_vouch} pipeline stage (always a separate Cloud Build job).

NOTE: This is the ark-level view of the build operation. For detailed build procedure steps, see
{rbtgo_trigger_build}.

The conjure operation:

//axhop_parameter_from_type
{rbtgr_vessel_dir}
. *Validate Input Parameters*:
.. {rbbc_require}
vessel directory provided as first argument
.. {rbbc_require}
vessel contains conjuring configuration (RBRV_CONJURE_DOCKERFILE, RBRV_CONJURE_BLDCONTEXT)

//axhos_step
. *Execute Build*:
.. Delegate to
{rbtgo_trigger_build}
which dispatches via the vessel's
{rbtgr_vessel_trigger}
.. All build parameters baked in
{rbtgr_build_json}
— zero custom substitution overrides
.. Waits for build completion
.. On success, the
{rbtga_ark_image}
artifact is created with the
{rbtga_consecration}
.. {rbtgr_provenance}
is auto-generated by Google as a signed attestation on the built image

//axhoc_completion
Successful execution creates the
{rbtga_ark_image}
in
{rbtgi_repository}.
The {rbtr_director} retains `«BUILD_ID»` and `«CONSECRATION»` from the conjure round-trip.
The trigger-dispatched job produces both `-image` and `-about` (about steps embedded via stitch).
`rbf_create` chains only {rbtgo_ark_vouch} after the combined job completes, passing `«BUILD_ID»` for SLSA provenance verification.
