//axhob_operation
{rbtgo_ark_conjure}

NOTE: This operation conjures an
{rbtga_ark_image}
in
{rbtgi_repository}
by building a container image from a vessel specification. The
{rbtr_director}
performs the build via
{gcb_service}.

NOTE: Conjure produces only the `-image` artifact.
The `-about` and `-vouch` artifacts are created by subsequent pipeline stages:
{rbtgo_ark_about} then {rbtgo_ark_vouch}.

NOTE: This is the ark-level view of the build operation. For detailed build procedure steps, see
{rbtgo_trigger_build}.

The conjure operation:

//axhop_parameter_from_type
{rbtgr_vessel_dir}
. *Validate Input Parameters*:
.. {rbbc_require}
vessel directory provided as first argument
.. {rbbc_require}
vessel contains conjuring configuration (RBRV_CONJURE_DOCKERFILE, RBRV_CONJURE_BLDCONTEXT)

//axhos_step
. *Execute Build*:
.. Delegate to
{rbtgo_trigger_build}
which dispatches via the vessel's
{rbtgr_vessel_trigger}
.. All build parameters baked in
{rbtgr_build_json}
— zero custom substitution overrides
.. Waits for build completion
.. On success, the
{rbtga_ark_image}
artifact is created with the
{rbtga_consecration}
.. {rbtgr_provenance}
is auto-generated by Google as a signed attestation on the built image

//axhoo_output_of_type
{rbtga_ark_image}
. *Display Results*:
.. {rbbc_show}
vessel image built with consecration tag

//axhoc_completion
Successful execution creates the
{rbtga_ark_image}
in
{rbtgi_repository}.
The {rbtr_director} retains `«BUILD_ID»` and `«CONSECRATION»` from the conjure round-trip for subsequent {rbtgo_ark_about} submission (passed as `_RBGA_BUILD_ID`).
The image is ready for the about-then-vouch pipeline.
Proceed to
{rbtgo_ark_about}
to assemble metadata, then
{rbtgo_ark_vouch}
to verify provenance.
