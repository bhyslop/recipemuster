//axhob_operation
{rbtgo_ark_conjure}

NOTE: This operation conjures an
{rbtga_ark_image}
in
{rbtgi_repository}
by building a container image from a vessel specification. The
{rbtr_director}
performs the build via
{gcb_service}
using direct `builds.create` API submission.

NOTE: Conjure produces both the `-image` and `-about` artifacts in a single Cloud Build job submitted via `builds.create`.
The about steps (`rbgja01`-`rbgja04`) are embedded via stitch alongside the build steps.
The `-vouch` artifact is created by the subsequent {rbtgo_ark_vouch} pipeline stage (always a separate Cloud Build job).

NOTE: The
{rbtga_consecration}
is minted locally by the
{rbtr_director}
before submission.
The Director knows the complete artifact tag set before the build begins -- naming authority and verification authority are the same entity.

NOTE: The conjure build job runs on the worker pool selected by the vessel's `RBRV_EGRESS_MODE`: tether pool for vessels needing public internet (e.g., `apt-get` in Dockerfile), airgap pool for vessels with all dependencies enshrined in GAR.
The subsequent {rbtgo_ark_vouch} always runs on the airgap pool regardless of vessel egress mode.

The conjure operation:

//axhopt_typed_parameter
{rbtga_ark}
. *Validate Input Parameters*:
.. {rbbc_require}
vessel provided as first argument
.. {rbbc_require}
vessel contains conjuring configuration (RBRV_CONJURE_DOCKERFILE, RBRV_CONJURE_BLDCONTEXT)

//axhos_step
. *Resolve Base Images*:
.. For each indexed slot (n = 1, 2, 3) where
{rbrv_image_origin}
is set:
... If
{rbrv_image_anchor}
is present: resolve to full GAR image reference using the anchor tag
... If
{rbrv_image_anchor}
is absent: pass
{rbrv_image_origin}
through as the upstream tag (development path)
.. Resolved references become `RBF_IMAGE_n` build-args for the Dockerfile `ARG`/`FROM` pattern

//axhos_step
. *Assign Consecration*:
.. Mint
{rbtga_consecration}
locally on the Director's workstation
.. {rbbc_store}
`«CONSECRATION»` -- complete artifact tag set is now known before build submission

//axhos_step
. *Push Pouch*:
.. Package build context (Dockerfile and context files) as a FROM SCRATCH OCI image
.. Push to GAR tagged `«VESSEL»:«CONSECRATION»-pouch`
.. {rbbc_require}
push succeeds -- the pouch delivers build context to
{gcb_service}

//axhos_step
. *Execute Build*:
.. Stitch
{rbtgr_build_json}
with all step image references and substitution variables
.. {rbbc_submit}
`POST «GCB_PROJECT_BUILDS_URL»` with Build resource JSON via `builds.create`
.. All build parameters in stitched
{rbtgr_build_json}
-- zero custom substitution overrides
.. Waits for build completion
.. On success, the
{rbtga_ark_image}
artifact is created with the
{rbtga_consecration}
.. {rbtgr_provenance}
is auto-generated by Google as a signed attestation on the built image

//axhoot_typed_output
{rbtga_ark_image}

//axhoot_typed_output axd_fact
{rbf_fact_consecration}
The full
{rbtga_consecration}
string.

//axhoot_typed_output axd_fact
{rbf_fact_build_id}
The
{gcb_build}
job ID from the `builds.create` submission.
Used by
{rbtgo_ark_vouch}
for
{rbtgr_provenance}
verification.

//axhoot_typed_output axd_fact
{rbf_fact_gar_root}
{gar_service}
registry prefix.
Composable with
{rbf_fact_ark_stem}
for full artifact references.

//axhoot_typed_output axd_fact
{rbf_fact_ark_stem}
{rbrv_sigil}
and
{rbtga_consecration}
joined as `«VESSEL»:«CONSECRATION»`.
Composable with suffix for artifact tags.

//axhoot_typed_output axd_fact
{rbf_fact_ark_yield}
Per-platform
{rbtga_ark_image}
reference.
One fact file per platform discovered during the build.

. *Display Results*:
.. {rbbc_show}
vessel image built with consecration tag

//axhoc_completion
Successful execution creates the
{rbtga_ark_image}
in
{rbtgi_repository}.
The {rbtr_director} retains `«BUILD_ID»` and `«CONSECRATION»` from the conjure round-trip.
The `builds.create` job produces both `-image` and `-about` (about steps embedded via stitch).
`rbf_create` chains only {rbtgo_ark_vouch} after the combined job completes, passing `«BUILD_ID»` for DSSE provenance verification.
