{mkr_network_connect}

NOTE: Secondary network attachment is now managed declaratively by Docker Compose.
The sentry's compose service definition lists both
{at_transit_network}
and
{at_enclave_network}.
Compose attaches both networks when the container starts — there is no separate connect step.

IMPORTANT: Docker Compose controls neither interface naming (eth0, eth1) nor default route selection.
Compose `priority` keys are empirically NOT honored for route selection on linux Docker engine 28.x or mac Docker Desktop 28.x, and the compose file does not set them.
Docker assigns the default route to the gateway of the alphabetically-first attached network, which is not necessarily the uplink.
The load-bearing mechanism is sentry-side: the startup script discovers its interfaces by IP using `ip -o addr show` (never hardcoded interface names), derives the uplink gateway as the first usable address of the uplink subnet, and then replaces the default route via the uplink (`ip route replace default via «uplink_gateway»`) so egress leaves through transit rather than the enclave.
