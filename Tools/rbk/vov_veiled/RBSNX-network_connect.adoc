NOTE: Secondary network attachment is now managed declaratively by Docker Compose.
The sentry's compose service definition lists both `default` (priority 1000) and `enclave` (priority 500) networks.
Compose attaches both networks when the container starts — there is no separate connect step.

IMPORTANT: Docker Compose does not guarantee which network maps to which interface name (eth0, eth1).
The compose `priority` field influences default route selection but does NOT control interface enumeration order.
The sentry startup script discovers its interfaces by IP using `ip -o addr show to «enclave_ip»` and never references hardcoded interface names.
