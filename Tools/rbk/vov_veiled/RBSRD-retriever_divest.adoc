//axhob_operation
{rbtgo_retriever_divest}

NOTE: This operation divests a
{rbtr_retriever}
{giam_service_account}
from the
{rbtgi_depot_project}.
The
{rbtr_governor}
synthesizes the SA email from the supplied identity folio, issues a REST DELETE against the IAM API, and opportunistically removes the production RBRA file if its `client_email` matches the synthesized email.
A required identity folio (param1) is supplied by the operator; the SA email is composed as `${RBCC_account_retriever}-<identity>@${RBGD_SA_EMAIL_FULL}`.

//axhos_step
. *Authenticate as {rbtr_governor}*:
.. Execute
{rbtoe_governor_authenticate}
.. {rbbc_store}
`«GOVERNOR_TOKEN»` for subsequent API calls

//axhos_step
. *Synthesize Service Account Email*:
.. {rbbc_require}
identity folio (param1) is non-empty
.. {rbbc_store}
`«RETRIEVER_EMAIL»` as `${RBCC_account_retriever}-<identity>@${RBGD_SA_EMAIL_FULL}` (composed from the role prefix and the operator-supplied identity)

//axhos_step
. *Divest Retriever Service Account*:
.. {rbbc_call}
link:https://cloud.google.com/iam/docs/reference/rest/v1/projects.serviceAccounts/delete[iam.projects.serviceAccounts.delete] with:
... method: `DELETE`
... endpoint: `https://iam.googleapis.com/v1/projects/{rbtgi_depot_project}/serviceAccounts/«RETRIEVER_EMAIL»`
... header: `Authorization: Bearer «GOVERNOR_TOKEN»`
.. {rbbc_require}
HTTP 200 response OR HTTP 404 (already divested)
.. {rbbc_warn}
if HTTP 404: "Retriever service account not found in cloud (already divested)"

//axhos_step
. *Confirm Deletion Propagated*:
.. {rbbc_poll}
the SA GET endpoint until it returns HTTP 404 for `RBGC_GONE_CONFIRM_STREAK` consecutive observations, confirming the
{giam_service_account}
is durably gone. A delete-accepted response precedes propagation, and the post-delete read path flaps 404→200 across replicas (see <<rbscip_trade_study>>), so a single 404 is not durable proof — an immediately following same-name
{rbtgo_retriever_invest}
would otherwise race a still-visible account. Any intervening non-404 resets the streak.
.. NOTE: skipped when the DELETE returned HTTP 404 — the account was already absent, so there is nothing to wait for
.. {rbbc_require}
`RBGC_GONE_CONFIRM_STREAK` consecutive HTTP 404s within the eventual-consistency deadline

//axhos_step
. *Opportunistic RBRA Cleanup*:
.. If the role's RBRA file exists at
{rbrr_retriever_rbra_file},
source it and read `RBRA_CLIENT_EMAIL`
.. If `RBRA_CLIENT_EMAIL` equals `«RETRIEVER_EMAIL»`, remove the file
.. NOTE: The removal is conditional — if the installed RBRA's `client_email` does not match the divested `«RETRIEVER_EMAIL»` (e.g., a different identity is currently installed), the file is left in place. No fatal error is raised when the file is absent or non-matching.

//axhos_step
. *Display Result*:
.. {rbbc_show}
single-line success notice naming `«RETRIEVER_EMAIL»` and confirming divestiture, or a 404-already-gone notice if cloud state was already clean

//axhoc_completion
Successful execution removes the
{rbtr_retriever}
{giam_service_account}
from
{rbtgi_depot_project}.
If the installed RBRA at
{rbrr_retriever_rbra_file}
matches the divested identity, it is also removed; otherwise it is left intact.
A 404 response from the cloud DELETE is accepted as success — the
{giam_service_account}
is absent either way.
Proceed to
{rbtgo_retriever_invest}
to re-issue a fresh
{rbtr_retriever}
{giam_service_account}
when needed.
