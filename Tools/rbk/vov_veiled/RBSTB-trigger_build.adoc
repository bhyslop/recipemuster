//axhob_operation
{rbtgo_trigger_build}

NOTE: This operation dispatches a container build via the vessel's
{rbtgr_vessel_trigger}
using the committed
{rbtgr_build_json}
in its
{rbtgr_vessel_dir}.
All substitution values are baked at inscribe time — zero custom overrides at dispatch.

NOTE: This operation supports concurrent execution per the Concurrent Session Safety invariant.
Each invocation dispatches an independent trigger execution with a unique `BUILD_ID`.
Concurrent builds of the same vessel produce independent images with different
{rbtga_hallmark}
timestamps.

The
{rbtr_director}
dispatches a trigger-based remote container build:

==== Substitution Variable Inventory

All `_RBGY_*` substitution variables are baked into
{rbtgr_build_json}
at inscribe time.
Cloud Build resolves these from the committed JSON — no values are overridden at dispatch.

*Vessel identity*::
[cols="1,2"]
|===
| `_RBGY_MONIKER` | Vessel sigil. Source: {rbrv_sigil}.
| `_RBGY_DOCKERFILE` | Dockerfile path relative to build context. Source: {rbrv_conjure_dockerfile}.
| `_RBGY_PLATFORMS` | Target platforms. Source: {rbrv_conjure_platforms}.
|===

*Registry coordinates*::
[cols="1,2"]
|===
| `_RBGY_GAR_LOCATION` | Artifact Registry region. Source: {rbrr_gcp_region}.
| `_RBGY_GAR_PROJECT` | Depot project ID. Source: {rbrr_depot_project_id}.
| `_RBGY_GAR_REPOSITORY` | GAR repository name. Source: {rbrr_gar_repository}.
| `_RBGY_GAR_HOST_SUFFIX` | GAR Docker host suffix. Source: RBGC constant.
| `_RBGY_ARK_SUFFIX_IMAGE` | Image artifact tag suffix. Source: RBGC constant.
| `_RBGY_CRANE_TAR_GZ` | Crane release tarball URL. Source: RBGC constant.
|===

*Provenance*::
[cols="1,2"]
|===
| `_RBGY_GIT_COMMIT` | Main repo HEAD SHA. Placeholder in generated JSON; filled at inscribe time.
| `_RBGY_GIT_BRANCH` | Main repo branch at inscribe time.
| `_RBGY_GIT_REPO` | Main repo identifier (owner/repo).
| `_RBGY_RUBRIC_REPO` | Build definition repository URL. Placeholder in generated JSON; filled at inscribe time.
| `_RBGY_RUBRIC_COMMIT` | Build definition repository HEAD SHA after push. Placeholder in generated JSON; filled at inscribe time.
| `_RBGY_INSCRIBE_TIMESTAMP` | Inscribe timestamp (`YYMMDDHHMMSS`). Placeholder in generated JSON; filled at inscribe time. Used by rbgjb01 to construct {rbtga_hallmark}.
|===

Builder image references (step `name:` fields) are resolved to digest-pinned URIs from regime pins at inscribe time.
These are inlined directly into JSON — not expressed as substitution variables.

==== Script Inlining

Step scripts (`rbgjb/*.sh`) are the source of truth for build logic.
At stitch time, each script is read and inlined into the corresponding JSON `args` array of
{rbtgr_build_json}.

Developers edit `rbgjb/*.sh` in the main repo, then run the stitch function to regenerate
{rbtgr_build_json}
with the updated content.

==== Tag Construction

Each build produces a
{rbtga_hallmark}
with dual timestamps:

`c«YYMMDDHHMMSS»-r«YYMMDDHHMMSS»`

* `c«YYMMDDHHMMSS»` — conjure mode prefix + inscribe timestamp, baked as `_RBGY_INSCRIBE_TIMESTAMP`
* `r«YYMMDDHHMMSS»` — realized timestamp, derived at build time by `rbgjb01-derive-tag-base.sh`

Example: `c260224153022-r260224160530`

Step `rbgjb01` constructs: `HALLMARK="c${_RBGY_INSCRIBE_TIMESTAMP}-r$(date -u +%y%m%d%H%M%S)"`

Complete artifact tags combine vessel moniker with {rbtga_hallmark}:

* {rbtga_ark_image}: `«MONIKER»:«HALLMARK»«ARK_SUFFIX_IMAGE»`

About and vouch artifacts are created by subsequent pipeline stages ({rbtgo_ark_about}, {rbtgo_ark_vouch}).

Multiple builds of the same inscribed JSON sort together by inscribe prefix,
each distinguished by its realized timestamp.
