//axhrb_regime
{rbrr_regime}

This regime specifies variables that define repository-wide configuration and installation-wide settings within the Recipe Bottle system.
Each
{rbrr_regime}
instance represents foundational infrastructure configuration that rarely changes and is shared across all Recipe Bottle operations.
RBRR assignment files are found in the operator's local filesystem (typically `../station-files/rbrr.env`) and configure the depot identity tuple `(RBRR_CLOUD_PREFIX, RBRR_DEPOT_MONIKER)` (from which depot project ID, GAR repository, pool stem, and bucket fall out as RBDC constants), regional settings, and service account credential file paths.

//axhrgb_group
{rbrr_group_resource_prefixing}

Operator-chosen identifiers that scope resource names across the cloud and local-runtime namespaces.
Enables multi-regime isolation on shared infrastructure — distinct depots or developers can coexist in the same GCP project or on the same developer workstation without name collisions.
All three variables must be populated — the marshal-zero path blanks them so a fresh regime requires an explicit operator decision before `rbrr_validate` passes.
The two prefixes must be 2–11 characters, start with a lowercase letter, and end with a hyphen (regex `^[a-z][a-z0-9-]*-$`).

//axhrgv_variable
{rbrr_cloud_prefix}

Prefix prepended to all cloud-visible resource names under the depot: GAR image paths, vessel package names,
{gcb_service}
substitutions, and other GCP-scoped identifiers.

//axhrgv_variable
{rbrr_depot_moniker}

Stem identifier that combines with
{rbrr_cloud_prefix}
to form the depot identity tuple.
Must be 1–26 characters, start with a lowercase letter, and contain only lowercase alphanumerics (regex `^[a-z][a-z0-9]*$`).
The tuple derives `RBDC_DEPOT_PROJECT_ID`, `RBDC_GAR_REPOSITORY`, `RBDC_GCB_POOL_STEM`, and `RBDC_GCS_BUCKET` at kindle time.

//axhrgv_variable
{rbrr_runtime_prefix}

Prefix prepended to machine-local resource names: container names, Docker network names, and other runtime identifiers on the operator's workstation.

//axhrv_variable
{rbrr_gcp_region}

The default
{gcp_service}
region for regional services like
{gar_service}
and
{gcb_service}.
Applied when creating repositories and submitting builds unless overridden by operation-specific configuration.

//axhrgb_group
{rbrr_group_service_accounts}

Credential file paths derived from a single enrolled directory for each operational role.
Each RBRA file contains the minimum identity and signing material needed for authentication to
{gcp_service}
via service account.
The payor RBRO file contains OAuth refresh credentials for
{rbtr_payor}
operations.

//axhrgv_variable
{rbrr_secrets_dir}

Directory on the operator's local filesystem outside the source repository that contains all credential files.
Enrolled and validated to exist; individual file paths are derived from this directory at lock time.

//axhrgv_variable
{rbrr_retriever_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `retriever/rbra.env`.
Enables the operator to assume the retriever role for container image operations.

//axhrgv_variable
{rbrr_director_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `director/rbra.env`.
Enables the operator to assume the director role for build submission and image management operations.

//axhrgv_variable
{rbrr_governor_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `governor/rbra.env`.
Enables the operator to assume the governor role for
{gcp_project_id}
administration operations.

//axhrgv_variable
{rbrr_payor_rbro_file}

Path derived from
{rbrr_secrets_dir}
as `payor/rbro.env`.
Enables the operator to perform
{rbtr_payor}
OAuth operations using locally stored refresh credentials.

//axhrgb_group
{rbrr_group_container_runtime}

Variables that configure container runtime and deployment environment settings.
These settings control DNS resolution and vessel directory layout.

//axhrgv_variable
{rbrr_vessel_dir}

Directory path containing vessel configuration subdirectories.
Each subdirectory defines a vessel image source (bind or conjure mode).

//axhrgv_variable
{rbrr_dns_server}

DNS server IP address used by sentry containers for name resolution.
Injected into container environment to control DNS behavior.

//axhrgb_group
{rbrr_group_build_configuration}

Operator-chosen
{gcb_service}
build parameters.
Builder images are sourced from the reliquary;
platform targets and VM supply chain are in the
{rbrm_regime}.

//axhrgv_variable
{rbrr_gcb_machine_type}

Compute Engine machine type for the private worker pool, consumed at pool creation time by
{rbtgo_depot_levy}.
Valid families: `e2`, `n2d`, `c3`.
Examples: `e2-standard-2`, `e2-highcpu-8`, `n2d-standard-4`.

//axhrgv_variable
{rbrr_gcb_min_concurrent_builds}

Minimum number of concurrent Cloud Build executions required.
With private pools, the relevant quota metric is `concurrent_private_pool_build_cpus` on the pool's host project.
Machine type is determined by the pool configuration at creation time, not by individual build requests.

//axhrgv_variable
{rbrr_gcb_timeout}

Maximum duration for Cloud Build operations before timeout.
Specified in seconds or duration format (e.g., `3600s`).

//axhrgb_group
{rbrr_group_public_docs}

Public documentation URL for user-facing references.

//axhrgv_variable
{rbrr_public_docs_url}

Complete URL for the public documentation entry point.
Updated per-release (swap branch name) or per-incorporation (point at a fork).
Used by buh_tlt glossary links and user-facing output.

//axhro_kindle
{rbkro_kindle}

Enrolls fields via `buv_*_enroll` type-specific helpers under named groups.
Detects unexpected RBRR_ variables via `buv_scope_sentinel`.
Composes `ZRBRR_DOCKER_ENV` injection array and locks the regime.

//axhro_validate
{rbkro_validate}

Validates enrolled fields via `buv_report RBRR` against type-specific constraints from enrollment.
Validates directory existence for `RBRR_VESSEL_DIR` and `RBRR_SECRETS_DIR`.
Enforces `RBRR_DEPOT_MONIKER` and `RBRR_CLOUD_PREFIX` format and their joint-length cap (≤30 chars for GCP project ID limit).

//axhro_render
{rbkro_render}

Renders enrolled fields via `buv_render RBRR` with group structure and presentation gates.
Sections: Resource Prefixing, Vessel and Local Configuration, GCP Infrastructure, Google Cloud Build Configuration, Secrets Directory, Public Docs.
