//axhrb_regime
{rbrr_regime}

This regime specifies variables that define repository-wide configuration and installation-wide settings within the Recipe Bottle system.
Each
{rbrr_regime}
instance represents foundational infrastructure configuration that rarely changes and is shared across all Recipe Bottle operations.
RBRR assignment files are found in the operator's local filesystem and configure the runtime container prefix, vessel directory, container DNS, Cloud Build timeouts, secrets directory, and public docs URL.
Depot-time-immutable identity and pool configuration live in the sibling
{rbrd_regime}.

//axhrgb_group
{rbrr_group_resource_prefixing}

Operator-chosen identifier that scopes machine-local resource names — container names, Docker network names, and other runtime identifiers on the operator's workstation.
Enables multi-regime isolation on shared infrastructure — distinct developers can coexist on the same workstation without name collisions.
Must be populated — the marshal-zero path blanks it so a fresh regime requires an explicit operator decision before `rbrr_validate` passes.
Must be 2–11 characters, start with a lowercase letter, and end with a hyphen (regex `^[a-z][a-z0-9-]*-$`).

//axhrgv_variable
{rbrr_runtime_prefix}

Prefix prepended to machine-local resource names: container names, Docker network names, and other runtime identifiers on the operator's workstation.

//axhrgb_group
{rbrr_group_service_accounts}

Credential file paths derived from a single enrolled directory for each operational role.
Each RBRA file contains the minimum identity and signing material needed for authentication to
{gcp_service}
via service account.
The payor RBRO file contains OAuth refresh credentials for
{rbtr_payor}
operations.

//axhrgv_variable
{rbrr_secrets_dir}

Directory on the operator's local filesystem outside the source repository that contains all credential files.
Enrolled and validated to exist; individual file paths are derived from this directory at lock time.

//axhrgv_variable
{rbrr_retriever_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `retriever/rbra.env`.
Enables the operator to assume the retriever role for container image operations.

//axhrgv_variable
{rbrr_director_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `director/rbra.env`.
Enables the operator to assume the director role for build submission and image management operations.

//axhrgv_variable
{rbrr_governor_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `governor/rbra.env`.
Enables the operator to assume the governor role for
{gcp_project_id}
administration operations.

//axhrgv_variable
{rbrr_payor_rbro_file}

Path derived from
{rbrr_secrets_dir}
as `payor/rbro.env`.
Enables the operator to perform
{rbtr_payor}
OAuth operations using locally stored refresh credentials.

//axhrgb_group
{rbrr_group_container_runtime}

Variables that configure container runtime and deployment environment settings.
These settings control DNS resolution and vessel directory layout.

//axhrgv_variable
{rbrr_vessel_dir}

Directory path containing vessel configuration subdirectories.
Each subdirectory defines a vessel image source (bind or conjure mode).

//axhrgv_variable
{rbrr_dns_server}

DNS server IP address used by sentry containers for name resolution.
Injected into container environment to control DNS behavior.

//axhrgb_group
{rbrr_group_build_configuration}

Operator-chosen
{gcb_service}
build parameters that may change across builds within a depot's lifetime.
Depot-time-immutable machine type lives in the
{rbrd_regime}.

//axhrgv_variable
{rbrr_gcb_min_concurrent_builds}

Minimum number of concurrent Cloud Build executions required.
With private pools, the relevant quota metric is `concurrent_private_pool_build_cpus` on the pool's host project.
Machine type is determined by the pool configuration at creation time, not by individual build requests.

//axhrgv_variable
{rbrr_gcb_timeout}

Maximum duration for Cloud Build operations before timeout.
Specified in seconds or duration format (e.g., `3600s`).

//axhrgb_group
{rbrr_group_public_docs}

Public documentation URL for user-facing references.

//axhrgv_variable
{rbrr_public_docs_url}

Complete URL for the public documentation entry point.
Updated per-release (swap branch name) or per-incorporation (point at a fork).
Used by buh_tlt glossary links and user-facing output.

//axhro_kindle
{rbkro_kindle}

Kindles the {rbrr_regime} fields by enrolling each field with its type-specific constraints under named groups.
Detects unexpected RBRR_ variables.
Composes the docker environment injection set and locks the regime against further modification.

//axhro_validate
{rbkro_validate}

Validates enrolled fields against type-specific constraints from enrollment.
Validates directory existence for {rbrr_vessel_dir} and {rbrr_secrets_dir}.
Enforces format constraints on {rbrr_runtime_prefix}.

//axhro_render
{rbkro_render}

Renders the {rbrr_regime} grouped by section with presentation gates per group.
Sections: Runtime Prefix, Vessel and Local Configuration, Google Cloud Build Configuration, Secrets Directory, Public Docs.
