//axhrb_regime
{rbrr_regime}

This regime specifies variables that define repository-wide configuration and installation-wide settings within the Recipe Bottle system.
Each
{rbrr_regime}
instance represents foundational infrastructure configuration that rarely changes and is shared across all Recipe Bottle operations.
RBRR assignment files are found in the operator's local filesystem (typically `../station-files/rbrr.env`) and configure depot project identifiers, regional settings, repository locations, and service account credential file paths.

//axhrv_variable
{rbrr_depot_project_id}

The
{gcp_service}
{gcp_project_id}
where all Recipe Bottle resources are created and managed.
Used as the target
{gcp_project_id}
in API calls and as the audience claim in OAuth token exchange.

//axhrv_variable
{rbrr_gcp_region}

The default
{gcp_service}
region for regional services like
{gar_service}
and
{gcb_service}.
Applied when creating repositories and submitting builds unless overridden by operation-specific configuration.

//axhrv_variable
{rbrr_gar_repository}

The
{gar_service}
repository name within
{rbrr_depot_project_id}
that stores all bottle images and metadata.

//axhrgb_group
{rbrr_group_service_accounts}

Credential file paths derived from a single enrolled directory for each operational role.
Each RBRA file contains the minimum identity and signing material needed for authentication to
{gcp_service}
via service account.
The payor RBRO file contains OAuth refresh credentials for
{rbtr_payor}
operations.

//axhrgv_variable
{rbrr_secrets_dir}

Directory on the operator's local filesystem outside the source repository that contains all credential files.
Enrolled and validated to exist; individual file paths are derived from this directory at lock time.

//axhrgv_variable
{rbrr_retriever_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `rbra-retriever.env`.
Enables the operator to assume the retriever role for container image operations.

//axhrgv_variable
{rbrr_director_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `rbra-director.env`.
Enables the operator to assume the director role for build submission and image management operations.

//axhrgv_variable
{rbrr_governor_rbra_file}

Path derived from
{rbrr_secrets_dir}
as `rbra-governor.env`.
Enables the operator to assume the governor role for
{gcp_project_id}
administration operations.

//axhrgv_variable
{rbrr_payor_rbro_file}

Path derived from
{rbrr_secrets_dir}
as `rbro-payor.env`.
Enables the operator to perform
{rbtr_payor}
OAuth operations using locally stored refresh credentials.

//axhrgb_group
{rbrr_group_container_runtime}

Variables that configure container runtime and deployment environment settings.
These settings control DNS resolution and vessel directory layout.

//axhrgv_variable
{rbrr_vessel_dir}

Directory path containing vessel configuration subdirectories.
Each subdirectory defines a vessel image source (bind or conjure mode).

//axhrgv_variable
{rbrr_dns_server}

DNS server IP address used by sentry containers for name resolution.
Injected into container environment to control DNS behavior.

//axhrgb_group
{rbrr_group_build_configuration}

Operator-chosen
{gcb_service}
build parameters.
Builder image pins are in the
{rbrg_regime};
platform targets and VM supply chain are in the
{rbrm_regime}.

//axhrgv_variable
{rbrr_gcb_machine_type}

Compute Engine machine type for the private worker pool, consumed at pool creation time by
{rbtgo_depot_create}.
Valid families: `e2`, `n2d`, `c3`.
Examples: `e2-standard-2`, `e2-highcpu-8`, `n2d-standard-4`.

//axhrgv_variable
{rbrr_gcb_min_concurrent_builds}

Minimum number of concurrent Cloud Build executions required.
With private pools, the relevant quota metric is `concurrent_private_pool_build_cpus` on the pool's host project.
Machine type is determined by the pool configuration at creation time, not by individual build requests.

//axhrgv_variable
{rbrr_gcb_timeout}

Maximum duration for Cloud Build operations before timeout.
Specified in seconds or duration format (e.g., `3600s`).

//axhrgv_variable
{rbrr_gcb_pool_stem}

Worker pool base name (suffixed with -tether/-airgap by derived constants).
Derived constants `RBDC_POOL_TETHER` and `RBDC_POOL_AIRGAP` expand this stem into full resource paths for each pool variant.
Must be set to the base name of the pools created by
{rbtgo_depot_create}
for this depot.

//axhro_kindle
{rbkro_kindle}

Sets defaults for all RBRR_ variables via `VAR="${VAR:-}"` pattern.
Detects unexpected RBRR_ variables via compgen prefix scan.
Builds ZRBRR_ROLLUP serialization string and ZRBRR_DOCKER_ENV injection array.

//axhro_validate
{rbkro_validate}

Validates fields via buv_env_* type checkers.
Checks required fields: DEPOT_PROJECT_ID, GCP_REGION, GAR_REPOSITORY.
Validates directory existence for RBRR_SECRETS_DIR.
Validates machine type and timeout format constraints.

//axhro_render
{rbkro_render}

Renders via rbcr_* shared module.
Sections: GCP Infrastructure, Artifact Registry, Service Account Credentials, Container Runtime Configuration, Build Configuration.
Displays unexpected variables in red.
Validates after full render completes.
