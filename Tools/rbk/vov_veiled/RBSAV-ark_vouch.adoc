//axhob_operation
{rbtgo_ark_vouch}

NOTE: This operation produces the
{rbtga_ark_vouch}
artifact for an
{rbtga_ark}
in
{rbtgi_repository}
by running mode-aware verification in
{gcb_service}.
The
{rbtr_director}
submits a Cloud Build job via `builds.create` (direct submission), which runs as the
{rbtr_mason}
service account on the private pool (`RBRR_GCB_WORKER_POOL`).

NOTE: Vouch uses `builds.create` rather than trigger dispatch because the
{rbtga_consecration}
is known only after the image arrives in GAR.
The private pool matches conjure's execution environment.

NOTE: The vouch verdict depends on vessel mode.
Conjure: SLSA provenance verification via `slsa-verifier verify-image`.
Bind: digest-pin comparison against the upstream reference.
Graft: GRAFTED stamp — explicit acknowledgment of no provenance chain.

The
{rbtr_director}
vouches for an ark:

//axhop_parameter_from_type
{rbtga_ark}
. *Validate Input Parameters*:
.. {rbbc_require}
vessel directory provided as first argument
.. {rbbc_require}
consecration provided as second argument

//axhos_step
. *Load Vessel Configuration*:
.. Source vessel configuration from `rbrv.env`
.. {rbbc_store}
`«VESSEL_MODE»` from {rbrv_vessel_mode}

//axhos_step
. *Authenticate as
{rbtr_director}*:
.. Execute
{rbtoe_director_authenticate}
.. {rbbc_store}
`«DIRECTOR_TOKEN»` for subsequent API calls

//axhos_step
. *Gate on About Existence*:
.. {rbbc_call}
Docker Registry API: HEAD manifests for `«CONSECRATION»-about`
.. {rbbc_require}
HTTP 200 response -- {rbtgo_ark_about} must complete before vouch
.. If `«CONSECRATION»-vouch` already exists:
{rbbc_warn}
re-vouch in progress (see idempotency below)

//axhos_step
. *Submit Vouch Cloud Build*:
.. Construct Build resource JSON with `serviceAccount` as
{rbtr_mason},
substitution namespace `_RBGV_*`
.. Pass registry coordinates, vessel mode, verifier identity, and source URI as substitution variables
.. {rbbc_submit}
`POST «GCB_PROJECT_BUILDS_URL»` with Build resource JSON
.. {rbbc_require}
HTTP 200 response
.. {rbbc_store}
`«BUILD_ID»` from response `.metadata.build.id`
.. {rbbc_show}
Cloud Console URL for build monitoring

//axhos_step
. *Cloud Build Step 1: Download and Verify slsa-verifier* (conjure only):
.. Image: {rbrg_alpine_image_ref}
.. If `_RBGV_VESSEL_MODE` is not `conjure`: early-exit with success
.. Download slsa-verifier binary from {rbrg_slsa_verifier_url}
.. Compute SHA256 checksum of downloaded binary
.. {rbbc_require}
checksum matches {rbrg_slsa_verifier_sha256}
.. {rbbc_store}
verified binary at `/workspace/slsa-verifier`

//axhos_step
. *Cloud Build Step 2: Mode-Aware Verification*:
.. Image: {rbrg_gcloud_image_ref}
.. Fetch OAuth token via `gcloud auth print-access-token` (gcloud authenticates as {rbtr_mason} via metadata server)
.. {rbbc_call}
Docker Registry API: GET manifests for `«CONSECRATION»-image` to obtain OCI image index
.. Filter out attestation manifests before extracting platforms.
OCI image indexes may contain non-runnable attestation manifests (SLSA provenance, SBOMs) with platform `unknown/unknown`.
Exclude any manifest entry where _both_ `platform.os` and `platform.architecture` are `unknown`.
See: https://docs.docker.com/build/metadata/attestations/attestation-storage/[Docker Attestation Storage].
.. {rbbc_store}
`«PLATFORM_DIGESTS»` as list of `(platform, digest)` pairs from the remaining (non-attestation) manifest entries
.. {rbbc_store}
platform list to `/workspace/vouch_platforms.txt`
.. Branch on `_RBGV_VESSEL_MODE`:
+
**Conjure branch**:
.. For each `(platform, digest)` in `«PLATFORM_DIGESTS»`:
... {rbbc_call}
`gcloud artifacts docker images describe --format json --show-provenance` to fetch provenance from Container Analysis API
... {rbbc_store}
per-platform provenance to `/workspace/provenance-«PLATFORM_SUFFIX».json`
... {rbbc_call}
`slsa-verifier verify-image` with `--provenance-path`, `--source-uri «RBRR_RUBRIC_REPO_URL»` (sans `.git` suffix), `--builder-id=https://cloudbuild.googleapis.com/GoogleHostedWorker`, and `--print-provenance`
... {rbbc_store}
per-platform verification output to `/workspace/verify-«PLATFORM_SUFFIX».json`
.. {rbbc_require}
all platforms verify successfully
.. Compose `vouch_summary.json` via python3: consecration, vessel, vessel_mode, verifier identity (URL + SHA256), per-platform verification verdicts
.. NOTE: slsa-verifier does not auto-discover GCB provenance from the Container Analysis API.
Provenance must be fetched explicitly via `gcloud artifacts docker images describe --show-provenance` and passed via `--provenance-path`.
The `--builder-id` must be `https://cloudbuild.googleapis.com/GoogleHostedWorker`.
+
**Bind branch**:
.. {rbbc_call}
Docker Registry API: HEAD manifests for `«CONSECRATION»-image`
.. {rbbc_store}
`«ACTUAL_DIGEST»` from `Docker-Content-Digest` response header
.. Extract `«PINNED_DIGEST»` from the `@sha256:...` suffix of `_RBGV_BIND_SOURCE`
.. {rbbc_require}
`«ACTUAL_DIGEST»` equals `«PINNED_DIGEST»` — pin integrity verified
.. Compose `vouch_summary.json`: consecration, vessel, vessel_mode (`bind`), verdict (`DIGEST_PIN_VERIFIED`), bind_source, actual_digest, pinned_digest
+
**Graft branch**:
.. Compose `vouch_summary.json`: consecration, vessel, vessel_mode (`graft`), verdict (`GRAFTED`), graft_source from `_RBGV_GRAFT_SOURCE`
.. No verification performed — the GRAFTED verdict explicitly marks the absence of provenance chain

//axhos_step
. *Cloud Build Step 3: Assemble and Push Vouch Artifact*:
.. Image: {rbrg_docker_image_ref}
.. Create buildx builder instance (inspect-or-create pattern, same as conjure)
.. Build multi-platform `FROM scratch` container with summary and per-platform verification files as layers
.. Platforms: read from `/workspace/vouch_platforms.txt` (written by step 2 for all modes)
.. Push to registry tagged `«CONSECRATION»-vouch` via `docker buildx build --push`
.. {rbbc_require}
push succeeds

//axhos_step
. *Wait for Build Completion*:
.. {rbbc_poll}
`GET «GCB_PROJECT_BUILDS_URL»/«BUILD_ID»` every 5 seconds
.. Continue while status in [`PENDING`, `QUEUED`, `WORKING`]
.. {rbbc_fatal}
if timeout or final status is not `SUCCESS` -- no partial vouch artifacts are pushed

//axhoo_output_of_type
{rbtga_ark_vouch}
. *Display Results*:
.. {rbbc_show}
vouch complete with consecration and vouch artifact tag
.. {rbbc_show}
verdict type: `SLSA_VERIFIED` (conjure), `DIGEST_PIN_VERIFIED` (bind), or `GRAFTED` (graft)

//axhoc_completion
Successful execution creates a
{rbtga_ark_vouch}
artifact in the registry tagged as `«VESSEL»:«CONSECRATION»-vouch`.
The vouch artifact is a multi-platform container (verification results are architecture-independent metadata; multi-platform is for pull ergonomics so arm64 hosts receive no platform mismatch warning).
+
*Source URI matching* (conjure only): The `--source-uri` must be `«RBRR_RUBRIC_REPO_URL»` with the `.git` suffix stripped.
The slsa-verifier prepends `https://` when matching against the source recorded in provenance by trigger-dispatched builds.
Format mismatch causes verification failure.
+
*Re-vouch idempotency*: Re-running vouch re-executes verification and pushes a new vouch artifact, overwriting the previous.
Content is presumed deterministic for conjure and bind; graft re-vouch always produces an identical GRAFTED stamp.
+
*Inspect integration*: Vouch results are displayed by {rbtgo_ark_inspect} in the Vouch Results section, showing verifier identity and per-platform verification verdicts (conjure), digest-pin confirmation (bind), or GRAFTED acknowledgment (graft).

==== Vouch Substitution Variables

All `_RBGV_*` substitution variables are passed in the `builds.create` request body.

*Registry and identity*::
[cols="1,2"]
|===
| `_RBGV_GAR_HOST` | GAR Docker host (e.g., `us-central1-docker.pkg.dev`). Source: regime constants.
| `_RBGV_GAR_PATH` | GAR repository path (`«PROJECT»/«REPOSITORY»`). Source: regime constants.
| `_RBGV_VESSEL` | Vessel sigil. Source: loaded vessel configuration.
| `_RBGV_CONSECRATION` | Full consecration string. Source: input parameter.
| `_RBGV_VESSEL_MODE` | Vessel mode (`conjure`, `bind`, or `graft`). Source: loaded vessel configuration.
|===

NOTE: Platform list is discovered by step 2 and written to `/workspace/vouch_platforms.txt`, not passed as a substitution variable.
Step 3 reads from workspace rather than re-discovering from the registry.

*Verification* (conjure only)::
[cols="1,2"]
|===
| `_RBGV_SOURCE_URI` | `«RBRR_RUBRIC_REPO_URL»` (sans `.git` suffix). Source: regime configuration.
| `_RBGV_VERIFIER_URL` | slsa-verifier download URL. Source: {rbrg_slsa_verifier_url}.
| `_RBGV_VERIFIER_SHA256` | slsa-verifier binary checksum. Source: {rbrg_slsa_verifier_sha256}.
|===

*Mode-specific*::
[cols="1,2"]
|===
| `_RBGV_BIND_SOURCE` | Pinned upstream image reference (bind only). Source: {rbrv_bind_image}.
| `_RBGV_GRAFT_SOURCE` | Local image reference at push time (graft only). Source: recorded during {rbtgo_ark_graft}.
|===
