{rbtgo_ark_vouch}

NOTE: This operation produces the
{rbtga_ark_vouch}
artifact for an
{rbtga_ark}
in
{rbtgi_repository}
by running mode-aware verification in
{gcb_service}.
The
{rbtr_director}
submits a Cloud Build job via `builds.create` (direct submission), which runs as the
{rbtr_mason}
service account on the airgap private pool (`RBDC_POOL_AIRGAP`, derived from `RBDC_GCB_POOL_STEM` with `-airgap` suffix — no public egress).

NOTE: Vouch uses `builds.create` rather than trigger dispatch because the
{rbtga_hallmark}
is known only after the image arrives in GAR.
The private pool matches conjure's execution environment.

NOTE: The vouch verdict depends on vessel mode.
Conjure: DSSE envelope signature verification of GCB-signed provenance.
Bind: digest-pin comparison against the upstream reference.
Graft: GRAFTED stamp — explicit acknowledgment of no provenance chain.

NOTE: *In-pool reliquary preflight*. When vouch steps are embedded in graft's combined about+vouch Cloud Build job (`zrbfv_graft_metadata_submit`), step 0 of the combined job is the in-pool reliquary preflight that HEAD-checks reliquary tool images from the worker pool's vantage before about and vouch steps run. Standalone vouch (`zrbfv_vouch_submit`, used by conjure and bind ordain flows after the image+about CB completes) does not currently include the preflight.

. *Cloud Build Step 2: Mode-Aware Verification* (Python 3):
.. Image: gcloud (from reliquary)
.. Fetch OAuth token from GCE metadata server
.. {rbbc_call}
Docker Registry API: GET manifests for `{gar_hallmarks_namespace}/«HALLMARK»/image:«HALLMARK»` to obtain OCI image index

See: https://docs.docker.com/build/metadata/attestations/attestation-storage/[Docker Attestation Storage].

**Conjure branch**:
.. For each `(platform, digest)` in `«PLATFORM_DIGESTS»`:
... {rbbc_call}

... Extract payload (base64url), signature (base64url), and keyid from the DSSE envelope using Python `json` module
... Decode payload and signature via Python `base64.urlsafe_b64decode`
... Reconstruct Pre-Authentication Encoding (PAE): `DSSEv1 {payloadType_len} {payloadType} {payload_len} {payload}`
... {rbbc_call}
`openssl dgst -sha256 -verify «KEY_PEM» -signature «SIG_BIN» «PAE_BIN»` via subprocess, using attestor public key from step 1
... {rbbc_require}
signature verification succeeds -- proves GCB signed the provenance
... Read now-trusted provenance fields via Python `json`: `builder.id`, `buildType`, `invocationId`, subject digest
... {rbbc_store}
per-platform verification output to `/workspace/verify-«PLATFORM_SUFFIX».json`
.. {rbbc_require}

.. {rbbc_call}

.. {rbbc_store}

.. {rbbc_require}
`«ACTUAL_DIGEST»` equals `«PINNED_DIGEST»` -- pin integrity verified
.. Compose `vouch_summary.json`: `{hallmark, vessel, vessel_mode: "bind", verification: {method: "digest-pin", bind_source, actual_digest, pinned_digest, verdict: "DIGEST_PIN_VERIFIED"}}`
+
**Graft branch**:
.. Compose `vouch_summary.json`: `{hallmark, vessel, vessel_mode: "graft", verification: {method: "none", graft_source, verdict: "GRAFTED"}}`
.. No verification performed -- the GRAFTED verdict explicitly marks the absence of provenance chain

//axhoc_completion
Successful execution creates a
{rbtga_ark_vouch}
artifact in the registry at `{gar_hallmarks_namespace}/«HALLMARK»/vouch:«HALLMARK»`.
The vouch artifact is a multi-platform container (verification results are architecture-independent metadata; multi-platform is for pull ergonomics so arm64 hosts receive no platform mismatch warning).
+
*DSSE verification* (conjure only): Provenance is fetched via `gcloud artifacts docker images describe --show-provenance` as a DSSE envelope.
The verification script (Python 3) decodes base64url payload and signature, reconstructs PAE, and calls `openssl dgst -sha256 -verify` as a subprocess.
Attestor public keys are embedded in the step script (sourced from `projects/verified-builder/` KMS); no runtime KMS dependency.
The about/vouch steps run on the gcloud builder image and invoke `gcloud` directly (see {rbsk_no_gcloud} exception).
No jq, no slsa-verifier — Python 3 standard library + openssl only. Air-gap compatible (Private Google Access reaches Google APIs).
+
*Re-vouch idempotency*: Re-running vouch re-executes verification and pushes a new vouch artifact, overwriting the previous.
Content is presumed deterministic for conjure and bind; graft re-vouch always produces an identical GRAFTED stamp.
+
*Inspect integration*: Vouch results are displayed by {rbtgo_ark_inspect} in the Vouch Results section, showing per-platform DSSE verification verdicts (conjure), digest-pin confirmation (bind), or GRAFTED acknowledgment (graft).

==== Vouch Substitution Variables

All `_RBGV_*` substitution variables are passed in the `builds.create` request body.

*Registry and identity*::
[cols="1,2"]
|===
| `_RBGV_GAR_HOST` | GAR Docker host (e.g., `us-central1-docker.pkg.dev`). Source: regime constants.
| `_RBGV_GAR_PATH` | GAR repository path (`«PROJECT»/«REPOSITORY»`). Source: regime constants.
| `_RBGV_VESSEL` | Vessel sigil. Source: loaded vessel configuration.
| `_RBGV_HALLMARK` | Full hallmark string. Source: input parameter.
| `_RBGV_VESSEL_MODE` | Vessel mode (`conjure`, `bind`, or `graft`). Source: loaded vessel configuration.
|===

NOTE: Platform list is discovered by step 2 and written to `/workspace/vouch_platforms.txt`, not passed as a substitution variable.
Step 3 reads from workspace rather than re-discovering from the registry.

*Mode-specific*::
[cols="1,2"]
|===
| `_RBGV_BIND_SOURCE` | Pinned upstream image reference (bind only). Source: {rbrv_bind_image}.
| `_RBGV_GRAFT_SOURCE` | Local image reference at push time (graft only). Source: recorded during {rbtgo_ark_graft}.
|===

*Base image provenance* (conjure only, empty for other modes)::
[cols="1,2"]
|===
| `_RBGV_IMAGE_1` | Resolved base image reference for slot 1 (full GAR ref if anchored, upstream tag if pass-through). Empty if slot unused.
| `_RBGV_IMAGE_1_PROVENANCE` | Provenance type for slot 1: `anchored` (GAR-mirrored via enshrine) or `pass-through` (upstream direct). Empty if slot unused.
| `_RBGV_IMAGE_2` | Resolved base image reference for slot 2.
| `_RBGV_IMAGE_2_PROVENANCE` | Provenance type for slot 2.
| `_RBGV_IMAGE_3` | Resolved base image reference for slot 3.
| `_RBGV_IMAGE_3_PROVENANCE` | Provenance type for slot 3.
|===
