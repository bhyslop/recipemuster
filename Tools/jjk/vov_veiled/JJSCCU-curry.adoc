Read or write a
{jjdhr_heat_p}
paddock file.
Operates in two modes depending on whether content is provided.

*Getter mode* (no content in gazette file, `firemark` param required):
reads the paddock file and returns its contents,
followed by gazette-formatted output written to {jjdxo_gazette}.

*Setter mode* (gazette file present, or `firemark` + `content` params provided):
writes new paddock content to the file,
creates a discussion commit with optional note.

{jjds_arguments}

//axd_optional
* {jjdt_firemark}
— target
{jjdhr_heat}.
Required in getter mode and in direct setter mode.
Not required when {jjdxo_gazette} contains setter input (firemark is parsed from gazette).

//axd_optional
* `content` (string) — new paddock content (direct setter mode, when gazette file is empty).

//axd_optional
* `note` (string) — chalk description appended to commit message.
Setter mode only.
If absent, commit message is "paddock curried".

{jjds_stdout}

* Getter mode: paddock file contents; gazette-formatted paddock written to {jjdxo_gazette}.
* Setter mode: confirmation message.

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

*Getter:*

. Parse {jjdt_firemark}
. Load {jjdgr_gallops} (read-only, no lock)
. Verify {jjdhr_heat} exists
. Read paddock file from {jjdhm_paddock} path
. Output paddock contents
. Output gazette-formatted version ({jjezs_paddock} slug)

*Setter (direct):*

. Parse {jjdt_firemark}
. Load {jjdgr_gallops}
. Verify {jjdhr_heat} exists
. Write new content to paddock file
. Commit paddock file with discussion marker (`d` action code)

*Setter (gazette file):*

. Dispatcher reads and consumes {jjdxo_gazette}, parses firemark and content
. Proceed as direct setter
