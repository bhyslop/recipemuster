//axvd_sheaf axd_normative
This is a normative portion of the {jjs0_imprimatur}: binding — code, tests, and decisions may rely on it.
Read or write a
{jjdhr_heat_p}
paddock file — two commands, one file.
Mode is named by the command, never inferred:
{jjdo_paddock} is the reader and can never write;
{jjdo_curry} is the writer and can never read.
The pair split (260707) after gazette-presence mode inference
let a pre-staged notice flip a read-intent call
into a silent, auto-committed paddock wipe.

*{jjdo_paddock}* (reader; `firemark` param required):
reads the paddock file and returns its contents,
followed by gazette-formatted output written to `gazette_out.md`.
Staged `gazette_in.md` input rejects loud
(already consumed by the universal entry rule;
the rejection names {jjdo_curry} as the write path and directs a re-write).

*{jjdo_curry}* (writer; `gazette_in.md` present with one `jjezs_paddock` notice, no `firemark` param):
writes new paddock content to the file,
creates a discussion commit with optional note.
The notice body must be non-empty:
an empty or whitespace-only body is a loud parse rejection —
a revision replaces the whole paddock, never blanks it —
backstopped at the write funnel (`jjrg_curry_apply`),
which refuses empty content for every caller
(the {jjsoprd_revise_docket} batch's paddock notice rides the same parse law and funnel).

{jjds_stdout}

* {jjdo_paddock}: paddock file contents; gazette-formatted paddock written to {jjdxo_gazette}.
* {jjdo_curry}: confirmation message.

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

*{jjdo_paddock}:*

. Reject if `gazette_in.md` held staged input — a read call carries nothing to consume
. Parse {jjdt_firemark} from param
. Load {jjdgr_gallops} (read-only, no lock)
. Verify {jjdhr_heat} exists
. Read paddock file from {jjdhm_paddock} path
. Output paddock contents
. Output gazette-formatted version ({jjezs_paddock} slug)

*{jjdo_curry}:*

. Dispatcher reads `gazette_in.md` via universal entry rule; absent input rejects, naming {jjdo_paddock} as the read path
. Parse firemark and content from the `jjezs_paddock` notice; an empty or whitespace-only body rejects loud
. Load {jjdgr_gallops}
. Verify {jjdhr_heat} exists
. Write new content to paddock file
. Run size guard at {jjda_size_limit}
. Commit paddock file with discussion marker (`d` action code)
