Read or write a
{jjdhr_heat_p}
paddock file.
Operates in two modes depending on whether content is provided.

*Getter mode* (no content, no input):
reads the paddock file and returns its contents,
followed by gazette-formatted output for structured consumption.

*Setter mode* (content or input provided):
writes new paddock content to the file,
creates a discussion commit with optional note.

{jjds_arguments}

//axd_optional
* {jjdt_firemark}
— target
{jjdhr_heat}.
Required in getter mode and direct setter mode.
Not required when using gazette `input` (firemark is parsed from gazette header).

//axd_optional
* `content` (string) — new paddock content (setter mode, direct).

//axd_optional
* `note` (string) — chalk description appended to commit message.
Setter mode only.
If absent, commit message is "paddock curried".

//axd_optional
* `input` (string) — gazette markdown input (setter mode, alternative to firemark+content).
Format: `# paddock <firemark>\n\n<paddock content>`.
When present, firemark and content are parsed from the gazette.

{jjds_stdout}

* Getter mode: paddock file contents, then gazette-formatted output.
* Setter mode: confirmation message.

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

*Getter:*

. Parse {jjdt_firemark}
. Load {jjdgr_gallops} (read-only, no lock)
. Verify {jjdhr_heat} exists
. Read paddock file from {jjdhm_paddock} path
. Output paddock contents
. Output gazette-formatted version ({jjezs_paddock} slug)

*Setter (direct):*

. Parse {jjdt_firemark}
. Load {jjdgr_gallops}
. Verify {jjdhr_heat} exists
. Write new content to paddock file
. Commit paddock file with discussion marker (`d` action code)

*Setter (gazette):*

. Parse gazette input to extract firemark and content
. Proceed as direct setter
