Read or write a
{jjdhr_heat_p}
paddock file.
Operates in two modes depending on whether {jjdxo_gazette} input is present.

*Getter mode* (no `gazette_in.md`, `firemark` param required):
reads the paddock file and returns its contents,
followed by gazette-formatted output written to `gazette_out.md`.

*Setter mode* (`gazette_in.md` present with `jjezs_paddock` notice):
writes new paddock content to the file,
creates a discussion commit with optional note.

{jjds_arguments}

//axd_optional
* {jjdt_firemark}
— target
{jjdhr_heat}.
Required in getter mode.
Not required in setter mode (firemark is parsed from {jjdxo_gazette}).

//axd_optional
* `note` (string) — chalk description appended to commit message.
Setter mode only.
If absent, commit message is "paddock curried".

//axd_optional
* {jjda_size_limit}
+
Setter mode only.

{jjds_stdout}

* Getter mode: paddock file contents; gazette-formatted paddock written to {jjdxo_gazette}.
* Setter mode: confirmation message.

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

*Getter:*

. Parse {jjdt_firemark} from param
. Load {jjdgr_gallops} (read-only, no lock)
. Verify {jjdhr_heat} exists
. Read paddock file from {jjdhm_paddock} path
. Output paddock contents
. Output gazette-formatted version ({jjezs_paddock} slug)

*Setter:*

. Dispatcher reads `gazette_in.md` via universal entry rule, parses firemark and content from `jjezs_paddock` notice
. Load {jjdgr_gallops}
. Verify {jjdhr_heat} exists
. Write new content to paddock file
. Run size guard at {jjda_size_limit}
. Commit paddock file with discussion marker (`d` action code)
