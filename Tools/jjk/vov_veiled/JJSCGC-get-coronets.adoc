
List
{jjdt_coronet_s}
for a
{jjdhr_heat}
in order, one per line.

{jjds_arguments}

//axd_required
* `FIREMARK` — the
{jjdt_firemark}
to query

//axd_optional
* `--remaining` — Exclude
{jjdpe_complete}
and
{jjdpe_abandoned}
paces

//axd_optional
* `--rough` — Include only
{jjdpe_rough}
paces

{jjds_stdout} One
{jjdt_coronet}
per line, in
{jjdhm_order}
sequence — no silks, no docket.
A live pace emits the bare coronet with its `₢` prefix.
In the default listing an
{jjdpe_abandoned}
pace is tagged `<coronet>  [abandoned]`;
the coronet stays the first whitespace token, so a token-wise reader still parses it.
Under `--remaining` or `--rough` the abandoned paces are excluded, so no marker appears.

*Example output (default listing):*
----
₢AFAAW
₢AFAAX  [abandoned]
₢AFAAQ
----

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

. Parse
{jjdt_firemark}
. Read
{jjdgr_gallops}
. Locate
{jjdhr_heat}
by
{jjdt_firemark}
. For each
{jjdt_coronet}
in
{jjdhm_order}:
.. If `--remaining` flag, skip paces where state is
{jjdpe_complete}
or
{jjdpe_abandoned}
.. If `--rough` flag, skip paces where state is not
{jjdpe_rough}
.. Output
{jjdt_coronet}
— suffixed with `  [abandoned]` when the pace state is
{jjdpe_abandoned}
— followed by newline
