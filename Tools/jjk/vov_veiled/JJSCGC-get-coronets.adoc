//axvd_sheaf axd_normative
This is a normative portion of the {jjs0_imprimatur}: binding — code, tests, and decisions may rely on it.

List
{jjdt_coronet_s}
for a
{jjdhr_heat}
in order, one per line.

*Invocation:*
----
vvx jjx_coronets <FIREMARK> [--remaining | --rough]
----

{jjds_arguments}

//axd_required
* {jjdt_firemark}
— the
{jjdhr_heat}
to query

//axd_optional
* `--remaining` — Exclude
{jjdpe_complete}
and
{jjdpe_abandoned}
paces

//axd_optional
* `--rough` — Include only
{jjdpe_rough}
paces

{jjds_stdout} One
{jjdt_coronet}
per line, in
{jjdhm_order}
sequence — no silks, no docket.
Returned inline as the tool result, not written to the
{jjsz_gazette}.
A
{jjdpe_rough}
pace emits the bare coronet with its `₢` prefix.
An
{jjdpe_abandoned}
pace is tagged `<coronet>  [abandoned]`, and a
{jjdpe_bridled}
pace `<coronet>  [bridled <tier>]` — so a dropped pace cannot be mistaken for a live one,
and a designated pace reads as claimed work at the tier that claims it.
The coronet stays the first whitespace token either way, so a token-wise reader still parses it.
A
{jjdpe_bridled}
pace whose tier is absent renders `[bridled ?]` (see
{jjdo_apostille},
which forbids the shape).
`--remaining` excludes the abandoned paces, so their marker never appears there;
`--rough` excludes both markers, since it admits only
{jjdpe_rough}
paces.

*Example output (default listing):*
----
₢AFAAW
₢AFAAX  [abandoned]
₢AFAAK  [bridled opus]
₢AFAAQ
----

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

. Parse
{jjdt_firemark}
. Read
{jjdgr_gallops}
. Locate
{jjdhr_heat}
by
{jjdt_firemark}
. For each
{jjdt_coronet}
in
{jjdhm_order}:
.. If `--remaining` flag, skip paces where state is
{jjdpe_complete}
or
{jjdpe_abandoned}
.. If `--rough` flag, skip paces where state is not
{jjdpe_rough}
.. Output
{jjdt_coronet}
— suffixed with `  [abandoned]` when the pace state is
{jjdpe_abandoned},
or with `  [bridled <tier>]` when it is
{jjdpe_bridled}
— followed by newline
