Parse git history for steeplechase entries belonging to a
{jjdhr_heat}.

{jjds_arguments}

// ⟦axd_required⟧
* {jjdt_firemark}
(positional) — target
{jjdhr_heat}

// ⟦axd_optional⟧
* `--limit` — maximum entries to return (default: 50)

{jjds_stdout} Column-aligned plain text table of steeplechase entries (newest first):
[source]
----
Timestamp            Commit  Act  Affil     Subject
────────────────────────────────────────────────────────
2026-01-24 09:20   f4824e77  [T]  ₣AF       Tally: parade-remaining-markdown
2026-01-24 08:49   857d982c  [F]  ₢AFAAf    Executing bridled pace...
2026-01-24 08:47   a6b75159  [W]  ₢AFAAn    pace complete
----

Columns:
- `Timestamp`: date and time in "YYYY-MM-DD HH:MM" format
- `Commit`: abbreviated git commit SHA (8 chars)
- `Act`: action code in brackets (e.g., `[n]` for notch, `[A]` for added) or empty for heat-level operations
- `Affil`: affiliation - shows ₢CORONET for pace-level entries, ₣FIREMARK for heat-level operations
- `Subject`: commit message or operation description

Entries ordered newest first (most recent at top).

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

. Run `git log --all --extended-regexp --grep="^jjb:[^:]+:(₣XX|₢XX)" --format="%ai%x09%s"`
  (skips hallmark field; filtering is by identity)
. Parse each line: extract timestamp, hallmark, identity, action, and message
. For pace-level (₢CORONET): verify coronet belongs to target heat
. For heat-level (₣FIREMARK): verify firemark matches target heat
. Build JSON array, limit to requested count
. Output JSON
