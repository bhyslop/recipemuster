== Act II — The mounted work

//axhci_interaction
`jjcbs_session` `jjcbs_engine`
{jjdo_open} —
the officium opens; the engine identifies the billet from the filesystem alone (the same derivation dispatch ran — agreement by independent rediscovery, no env handoff) and reports trunk staleness loudly.
On a known-stale trunk the mutating doors below would refuse in a precision band naming the resync operation as remedy; open itself never refuses.
Pending: the identify contract and the resync operation.

//axhci_interaction
`jjcbs_session` `jjcbs_engine`
{jjdo_orient} —
the mount read: paddock and docket from the studbook, lock-free, staleness-tolerant.

//axhci_interaction axd_attended
`jjcbs_operator` `jjcbs_session`
The work itself: the human directs, the session edits, and changes land in the billet's tree.
No system contract governs this interaction — the attended license is its citation.

//axhci_interaction
`jjcbs_session` `jjcbs_engine`
{jjdo_record} —
the notch, with an explicit file list.

//axhci_interaction
`jjcbs_engine` `jjcbs_hippodrome`
Inside the notch: status, commit the explicit list on the billet branch, take the counterfoil, push the branch — the work half of the {jjdb_journal}, durable-first.

//axhci_interaction
`jjcbs_engine` `jjcbs_studbook`
The {jjdb_journal} proper: fetch, CAS-create the {jjdb_guidon}-bearing lock ref, advance to remote tip, mutate and commit the gallops change with the counterfoil naming the hippodrome commit, push content atomically under lease, release.
Durable-first is visible in the interaction order — the hippodrome commit of the previous interaction already exists before this journal names it.

== Act III — Wrap and teardown

//axhci_interaction
`jjcbs_session` `jjcbs_engine`
{jjdo_close} —
the wrap: remaining work committed, the pace state mutates, and the same {jjdb_journal} bracket carries it into the studbook (the previous two interactions' shape, replayed).

//axhci_interaction
`jjcbs_session` `jjcbs_operator`
The session ends and the operator is back at the shell.
The billet stands — nothing reaps at wrap; the branch is pushed, so the worktree holds only convenience.

//axhci_interaction
`jjcbs_operator` `jjcbs_dispatch`
Days later, the next saddle: Act I's sweep finds this pace closed and its billet past retention, and reaps it.
The teardown actor is the dispatch layer at next start — never the session, which cannot remove its own floor.
Pending: the sweep operation.
