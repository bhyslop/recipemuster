Test infrastructure for validating the
{jjsogrd_remote_dispatch}
command surface across SSH boundaries.

A
{jjdxf_profile}
is a named account shape that declares
{jjdxr_fundus}
preconditions — what is configured, what is deliberately absent, and what assertions the profile supports.
Each profile corresponds to an OS user account (`jjfu_*`) that can be installed on any target machine.
Tests are run against a host specified at invocation time;
the same profile set works whether the target is localhost or a remote machine.
Each profile is
{jjdxf_preflight}-probed
before execution — missing preconditions skip cleanly, not fail.

[[jjdxf_profile]]
//axve_entity
{jjdxf_profile}::
A named account shape for
{jjdxr_fundus}
scenario testing.
Specifies an OS username (`jjfu_*`), expected precondition state, and supported test assertions.
All profiles share the same
{jjdt_reldir}
string; the precondition difference is what exists at that path under each account's home directory.
Two families: happy-path (full command coverage) and failure-mode (one broken precondition each).

==== Account Names

All test accounts use the `jjfu_` prefix (JJ Fundus User).
The prefix is globally unique in the minting universe, collision-proof, and grepping a machine for `jjfu_*` accounts shows exactly what is installed for
{jjsogrd_remote_dispatch}
testing.

[cols="1,2"]
|===
| Account | Precondition shape

| `jjfu_full`
| SSH keys, project at {jjdt_reldir}, BUK installed, git origin configured

| `jjfu_nokey`
| OS account exists, no SSH key access from {jjdxr_curia}

| `jjfu_norepo`
| SSH keys, no project directory at {jjdt_reldir}

| `jjfu_nogit`
| SSH keys, project at {jjdt_reldir}, BUK installed, no git origin
|===

==== Happy-Path Profile

[[jjdxf_full]]
//axl_voices axt_enum_value
{jjdxf_full}::
Full happy-path account.
User: `jjfu_full`.
{jjdt_reldir}: `projects/rbm_alpha_recipemuster`.
Full suite:
{jjdo_bind},
{jjdo_send},
{jjdo_relay}
(including parallel dispatch),
{jjdo_check}
(both instant probe and polled await),
{jjdo_fetch},
{jjdo_plant}.

==== Failure-Mode Profiles

Failure-mode profiles each break exactly one
{jjdxr_fundus}
precondition to validate a specific error path.

[[jjdxf_nokey]]
//axl_voices axt_enum_value
{jjdxf_nokey}::
No SSH key configured for this account.
User: `jjfu_nokey`.
Expected:
{jjdo_bind}
fails with authentication error.

[[jjdxf_norepo]]
//axl_voices axt_enum_value
{jjdxf_norepo}::
SSH access works but no project at
{jjdt_reldir}.
User: `jjfu_norepo`.
Expected:
{jjdo_bind}
fails at Layer 3 probe per
{jjdk_reldir_safety}
(resolved-path check fails).

[[jjdxf_nogit]]
//axl_voices axt_enum_value
{jjdxf_nogit}::
SSH access and project directory exist, but git remote is not configured.
User: `jjfu_nogit`.
{jjdt_reldir}:
directory exists with BUK installed, but `git remote` returns no origin.
Expected:
{jjdo_plant}
fails (no origin to fetch from).
{jjdo_send}
still works (BUD dispatch does not require git).

==== Preflight Contract

[[jjdxf_preflight]]
//axve_entity
{jjdxf_preflight}::
Probe executed before running a
{jjdxf_profile_p}
test.
If preconditions are not met, the test fails with a diagnostic message directing the operator to provision accounts.
Tests must never silently pass when preconditions are absent — a passing test means the assertion was actually exercised.
+
Each profile requires a corresponding {xref_BURH} on the
{jjdxr_curia}.
The BURH alias is the addressing handle — preflight probes connect via the alias, not raw `user@host`.
Test account BURH profiles carry `BURH_COMMAND` appropriate to the target platform (empty for Linux/macOS default shell, platform-specific for Windows subsystems).

*{jjdxf_full} profile:*

. BURH resolution: kindle `jjfu_full-<suffix>/burh.env`, extract alias
. SSH connection: `ssh -o ConnectTimeout=5 -o BatchMode=yes <BURH_ALIAS> exit 0`
. {jjdt_reldir} existence: `ssh <BURH_ALIAS> test -d $HOME/RELDIR`
. BUK installation: `ssh <BURH_ALIAS> test -f $HOME/RELDIR/.buk/burc.env`
. Any step fails -> test fails with provisioning instructions

*Failure-mode profiles* ({jjdxf_nokey}, {jjdxf_norepo}, {jjdxf_nogit}):

. BURH profile must exist for the account (provides alias and host coordinates)
. Profile-specific precondition check via BURH alias (SSH reachability, directory state)
. Missing precondition -> test fails with provisioning instructions
. These profiles deliberately fail at different stages — the
{jjdxf_preflight}
checks only that the test can be meaningfully attempted

==== Tabtarget Surface

Suite colophon: `jjw-tfs`.
Single colophon: `jjw-tfS`.
Channel: imprint selects the target host.

*Suite tabtarget:*
`tt/jjw-tfs.TestFundusScenario.<host>.sh` — runs all profiles against `<host>`.

*Single tabtarget:*
`tt/jjw-tfS.TestFundusSingle.<host>.sh <test-function-name>` — runs one test function against `<host>`.
The argument is the exact Rust test function name (e.g., `full::bind_send`).

Each tabtarget:

. Sets `JJTEST_HOST` from the imprint
. Invokes `cargo test --test fundus_scenario -- --test-threads=1 --ignored [filter]`
. Individual test functions run their own {jjdxf_preflight} and fail if preconditions are not met

Test thread count is always 1 — tests within a suite are sequential
({jjdxr_fundus}
state is shared).

==== Test Assertions by Profile

*{jjdxf_full} — full happy-path:*

* {jjdo_bind} + {jjdo_send}: bind to {jjdxr_fundus}, run simple command, verify output
* {jjdo_plant}: reset {jjdxr_fundus} to known commit, verify working tree state
* {jjdo_relay} + {jjdo_check} (timeout=0): launch async job, instant probe shows {jjdxpe_running}
* {jjdo_relay} + {jjdo_check} (timeout>0): launch short job, await completion, verify {jjdxpe_stopped} + exit code
* {jjdo_check} file list: verify temp dir contents returned on terminal status
* {jjdo_fetch}: read specific file from {jjdxr_fundus} using absolute path from `BURX_TEMP_DIR`
* `BURX_LABEL` correlation: verify {jjdt_pensum} token round-trips through `BURE_LABEL` -> `BURX_LABEL`

*{jjdxf_nokey} — bind auth failure:*

* {jjdo_bind} returns non-zero exit status
* Error message indicates authentication failure

*{jjdxf_norepo} — bind probe failure:*

* {jjdo_bind} returns non-zero exit status
* Error message indicates {jjdt_reldir} validation failure (Layer 3 per {jjdk_reldir_safety})

*{jjdxf_nogit} — plant failure, send works:*

* {jjdo_bind} succeeds (SSH works, {jjdt_reldir} exists, BUK installed)
* {jjdo_send} succeeds (BUD dispatch does not require git)
* {jjdo_plant} returns non-zero exit status (no origin to fetch from)
