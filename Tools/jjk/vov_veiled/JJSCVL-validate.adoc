//axvd_sheaf jjs0_imprimatur axd_normative

Normalize the
{jjdgr_gallops}
to canonical form and report the verdict.

A deliberate normalize-and-report pass, not a read-only check: a valid but
non-canonical store is rewritten to canonical form and committed rather than
fataled.
Canonicalization is the same write-forward the
{jjdz_reprieve}
mechanism drives in migration mode, plus key-order and whitespace
normalization.
It never invents or repairs missing or contradictory data — structural or
invariant breakage is reported and the file left untouched, never a silent fix.

{jjds_arguments}

{jjds_stdout}
Every outcome names its bucket and prints a per-episode
{jjdz_reprieve}
census, so a clean result is positive evidence rather than mere silence.

{jjds_exit_enumerated}

* 0 — clean: valid and already canonical; no write.
* 2 — normalized: valid but non-canonical; rewritten to canonical form and committed under {jjda_size_limit}.
* 1 — broken: parse or invariant failure; file untouched.

{jjds_behavior}

The normalize commit is gallops-wide, affiliated with neither
{jjdhr_heat}
nor
{jjdpr_pace}.
When a merge is in progress it finalizes that merge (two parents) — validate is
the post-merge
{jjdgr_gallops}
cleanup step.
Re-running after a normalize yields clean (idempotent).

{jjdhr_heat}

{jjdhm_silks}

{jjdhm_created}
(YYMMDD),
{jjdhm_status}

{jjdhe_stabled},

{jjdhm_order}
(array),
{jjdhm_seed}

`paces` (object)
. {jjdhm_order}
array and `paces` object must have identical key sets
. Each
{jjdpr_pace}

{jjdhr_heat}

. Each
{jjdpr_pace}

{jjdkr_tack}
