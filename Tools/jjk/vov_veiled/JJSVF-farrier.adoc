This is a normative entity sheaf of the {jjs0_imprimatur}: binding — code, tests, and decisions may rely on it.

= Farrier

JJ's polymorphic revision-control driver: the layer that fits the same JJ intent to whatever
ground a repo runs on.
This sheaf is the trait's contract home: the facet split, the op census as method contracts,
the rejection-kind taxonomy, and the {jjdf_palisade} at the trait boundary.
The ceremonies above the trait — the {jjdb_journal}, the {jjdb_break}, the dispatch doors —
stay JJ's;
no driver ever owns a transaction.

== The entity

[[jjdf_farrier]]
//axo_entity
{jjdf_farrier}::
The polymorphic revision-control driver: one trait, one implementation per *farrier kind* —
plain git in the MVP, worktrees within it; submodule and multi-repo constellations later.
As a farrier shoes the same horse for different terrain, one driver interface serves every
backend;
the {jjdb_pedigree} records which farrier kind serves clones of its sire.
The {jjdb_studbook} itself is driven through the same interface — the degenerate single-repo
case.
+
The {jjdf_farrier} supplies *primitives*; the ceremonies that compose them are homed above
the trait, and every mutating primitive fails loud and specifically — sequence composition
is only safe on that footing.
+
Capability is structural, never declarative: the trait splits into three facets —
{jjdf_core}, {jjdf_lock}, {jjdf_billet} — and a farrier kind implements exactly the facets it
can honestly serve, so a kind lacking a facet is a compile-time fact.
The blotter carrier implements {jjdf_core} plus {jjdf_lock};
a hippodrome-serving kind implements {jjdf_core} plus {jjdf_billet}.
No runtime capability machinery exists in the MVP;
the degradation ladder stays spec prose until a second farrier kind arrives.

== Two axes, two generalization points

The eventual backends sit on two orthogonal axes;
each has one generalization point settled now so later rungs add without re-minting:

* *Constellation axis* — what a {jjdb_counterfoil} names: one repo vs. many members.
  Generalization point: the {jjdb_counterfoil} is always a member→SHA manifest, single-repo
  the one-element case (canon at the studbook sheaf).
* *Partition axis* — who writes concurrently: a shared checkout vs. isolated per-work
  checkouts.
  Generalization point: the billet — a root with a lifecycle.
  A worktree is the plain-git implementation;
  a full clone is the fallback for kinds without worktrees.
  Every op takes an explicit repo root, so a partition is just another root — op signatures
  never change as partition mechanics vary by kind.

== The vocabulary Palisade

[[jjdf_palisade]]
{jjdf_palisade}::
The trait surface is a vocabulary Palisade: git words stop at the driver implementation
boundary.
Below the trait, git vocabulary is the neighbor's own language (Palisade conduct — never
legislate the neighbor);
above it — toothings, specs, rejection kinds, everything a session reads — a git word is
interface contamination.
+
The rationale is the reflex set: a faithful-subset git name is correct only up to an
invisible boundary, and the boundary is exactly what a lesser model does not hold —
`commit` fires the amend/stage-all reflex, `push` fires force in a recovery scenario,
create/delete/read fire the CRUD reflex (a failed create invites a "cleanup" delete).
A fresh word fails loud — the session looks it up;
a familiar word fails silent.
+
Census names stay boring-register but git-free: distinct and vivid are different axes, and
no asterism budget is spent below the vivid layer (`advance` and `enfold` prove the
register).
Any op added to the census mints under MCM Lapidary behind this Palisade.

== The facets

[[jjdf_core]]
{jjdf_core}::
The ops every farrier kind serves: orientation ({jjdf_identify}), inspection (`comb`,
`sync_state`, `counterfoil`), deposit (`lodge`, `proffer`), and remote sync (`glean`,
`advance`, `consign`).

[[jjdf_lock]]
{jjdf_lock}::
The guidon verbs `stake`/`pluck`/`sight` over a {jjdb_blotter_p} lock ref — the
compare-and-swap primitives composed only by the {jjdb_journal} and the {jjdb_break}.

[[jjdf_billet]]
{jjdf_billet}::
The billet lifecycle ops
`billet_create`/`billet_seat`/`billet_adopt`/`billet_detach`/`billet_remove`/
`enfold`, plus the observations `line_exists`, `line_abroad`, `outstripped`, and
`reachable`, consumed by the dispatch doors and the {jjdd_stile_p} trailing step
({jjdd_saddle}, {jjdd_lunge}, {jjdd_stile} — dispatch sheaf).

== Op census — method contracts

Primitives are *capabilities, not commands* — almost none are ever human-typed.
Two cross-cutting laws:
every op takes its repo root explicitly and none reads the working directory (the no-cwd
rule);
every mutating op rejects in a named rejection kind (below), never a bare failure.

=== Core facet

[[jjdf_identify]]
{jjdf_identify}::
Resolve the tree at an explicit probe path: studbook-blind, network-silent, lock-free — the
first link of the cosmology's derivation chain, running before any record is open.
The consumer captures the path once (the door captures cwd);
{jjdf_identify} itself honors the no-cwd rule.

The answer is *claim-or-decline*, so {jjdf_identify} doubles as ground detection:
the consumer probes the farrier-kind roster, and a claim names the claiming kind.
(The stile's approach cross-checks the claiming kind against the kind recorded in the
{jjdb_pedigree} downstream;
a mismatch is the approach's named rejection — record/ground drift — never
{jjdf_identify}'s.)

A claimed tree resolves exactly four things:

* *root* — the tree's station-local absolute path: transient, never journaled
  (the no-worktree-paths rivet, studbook sheaf).
  The identity dirname derives as its basename.
* *canonical upstream key* — the kind owns canonicalization;
  one key per tree — a constellation kind keys on its own upstream.
  The key is derived address, never the {jjdb_pedigree} key itself:
  the approach's pedigree lookup resolves it in one indirection,
  derived key → sire → {jjdb_pedigree} (studbook sheaf).
* *seat* — primary or partition.
  A partition seat carries its primary's root;
  the linkage mechanics are the kind's own (worktree gitdir for plain git, a planted
  station-local marker for full-clone fallbacks).
* *line of work* — branch or detached: one seat-grain designation, even for constellations.
  The branch name is reported verbatim, in the kind's own ref dialect:
  reading JJ's {jjdd_livery} out of it is a consumer's act, above the {jjdf_palisade},
  never {jjdf_identify_p} — a kind knows ref names, not JJ's identities.

{jjdf_identify} is *total* for a claimed tree: detached, mid-merge, and dirty states report
faithfully, never reject;
the sole failure is foreign ground (no kind claims), and all refusal policy lives in
consumers.

Deliberately excluded from the return:
position — the member→SHA manifest shape has one home, the `counterfoil` op;
trunk-ness — pedigree-relative and classified downstream, so the seat says primary, never
trunk;
dirt and staleness — `comb`'s and `sync_state`'s.

`comb`::
Comb the tree for dirt: clean/dirty, per-path — feeds clean-tree gates and explicit-list
deposits.
Sequence-internal.

`sync_state`::
Ahead/behind vs. the remote, from the last `glean` — never blocks on the network.
Ambient at open and orient: the staleness warning's source.

`counterfoil`::
Take the {jjdb_counterfoil} at the tree's current position: the member→SHA manifest —
one-element for a single repo — annotated with branch and dirty flag.
The manifest shape's one home;
ambient on record.

`lodge`::
Formally deposit an explicit file list with a message — no stage-all, no amend (additive
discipline).
Sequence-internal.

`glean`::
Gather what the remote holds: update remote-tracking state, mutate nothing local.
Sequence-internal, opportunistic;
the {jjdf_farrier} fetches and never merges — {jjdd_refit} (dispatch sheaf) is the
staleness remedy.

`advance`::
Carry the line of work forward to the remote tip: *fast-forward-only*, a no-op when already
there, and never destructive.
A line holding commits the counterpart does not have rejects ({jjdf_diverged}) — under the
compose-then-push rivet (`JJr_b52`, {jjdb_journal}) the local branch adopts only positions
the remote has accepted, so that state is impossible and is surfaced for the attended
session, never auto-destroyed.
The {jjdf_farrier} never merges toward a remote and never rebases.
Unrelated working-tree scratch neither blocks the move nor is destroyed by it — the move
touches only the paths the tip changed.

`consign`::
Hand a branch into the remote's custody: plain fast-forward push (hippodrome and billet
branches; a content race rejects ({jjdf_diverged})).
{jjdb_blotter} content never passes here — it goes through `proffer`, which is what binds a
write to the held lock.
*Never force* — a rivet, cited in shipped code by the opaque token `JJr_d81`:
the consign and proffer implementations and the lease-guarded `pluck` of the {jjdb_break} are
its citation sites, and one `grep JJr_d81` returns this definition and every site it guards.

`proffer`::
Proffer a composed write for the remote's acceptance — the {jjdb_blotter} write primitive
(`JJr_b52`, compose-then-push):
compose one commit whose sole parent is the branch's remote counterpart's tip (as of the last
`glean`), carrying exactly the named files, *without moving the local branch*;
push it atomic-under-lease against the holder's own lock ref ({jjdb_journal});
and only on acceptance advance the local branch to the accepted position.
A refusal — ({jjdf_lock_broken}) when the lock no longer flies the leased guidon, ({jjdf_diverged}) on a
plain content race — leaves the local branch and its record untouched: no residue exists for
any later ceremony to scrub.
Never force (`JJr_d81`).
+
The lease is also what *protects a live writer from a broken lock* (`JJr_c4e`, journal
sheaf): a holder whose lock is broken and restaked under it fails this push entire and lands
nothing.
That is the safety case the {jjdd_cashier} rests on, and it lives here, in the lease.

`counterfoil_verify`::
Does a journaled SHA still exist, reachable?
*Deferred past MVP*: the designed answer for when a dangling {jjdb_counterfoil}
first annoys — merge-based refit and push-at-notch make them rare.

=== Lock facet

The guidon verbs, consumed only by the {jjdb_journal} and the {jjdb_break};
sequence-internal, never operator-typed.
The trio spends vividness below the vivid layer deliberately ({jjdf_palisade} rationale):
create/delete/read fire the CRUD reflex exactly as push fires force, and the lock is the
maximum-danger facet, where a reflex-firing name costs most.

`stake`::
Stake the {jjdb_guidon} in: atomically create the lock ref bearing it — the compare-and-swap
claim — or reject ({jjdf_lock_held}).

`pluck`::
Pluck the lock out: lease-guarded delete against an observed {jjdb_guidon}, never blind.

`sight`::
Sight whose {jjdb_guidon} flies: read the lock ref, read-only.

In the entity design the held lock is an RAII guard — object lifetime is lock lifetime, and
a nested acquire panics;
clearing someone else's stale lock is never a guard method but the {jjdb_break}, its own
sequence (journal sheaf).

=== Billet facet

`billet_create`::
Birth a billet: seat an isolated partition — worktree for plain git, full-clone fallback for
kinds without worktrees — on a fresh branch, or detached.
Both birth forms anchor at the trunk branch's remote counterpart, never the primary's own
checkout: the caller names the trunk, the kind resolves the counterpart itself (the
`advance`/`enfold` posture), because a birth at the primary's HEAD would carry the operator's
unpushed trunk work into the billet branch and publish it at the first `consign` — the
`enfold` no-exfiltration law, applied at birth.
A branch-name collision fails loud as a caller-contract violation: a durable branch re-seats
via `billet_seat`, never re-births.
The kind sets the born branch's upstream to its own eventual counterpart, not trunk's, so a
later `consign` lands where this station itself will look.

`billet_seat`::
Seat an *existing* branch into a fresh partition — the reuse form behind the {jjdd_stile_p} approach
billet-ensure: billets are session-transient, branches durable (dispatch sheaf), so a
destroyed billet's branch re-seats with its own WIP history, un-anchored and un-reset.
A branch the constellation already seats rejects in a named kind — {jjdf_seat_vestige} when the
recorded seat's root is gone, {jjdf_line_seated} when it stands — rather than failing loud as a
caller-contract violation: both are states the operator reaches by ordinary means, and both
carry a remedy the refusal can name.
`billet_create`'s collision arm keeps the caller-contract posture, being a JJ composition
fault rather than a state the operator arrives at.
A branch that does not exist at all remains a caller-contract violation here, since
`line_exists` is the observation the approach consults first.

`billet_adopt`::
Seat a branch this station does not yet carry, taking its position from the remote
counterpart — the adopt-never-fork arm behind the {jjdd_stile_p} approach billet-ensure.
A fresh local branch is minted AT the counterpart's tip, as of the last `glean`, and seated in
the fresh partition, so a station meeting work another station pushed rejoins THAT line of
work rather than forking a rival one from trunk that no ceremony would ever reconcile.
The kind sets the adopted line's upstream to the counterpart it was born from, so a later
`consign` lands where the other station is looking.
This is the sole birth anchoring at something other than trunk's counterpart, and the
`enfold` no-exfiltration law holds all the same: a counterpart is by definition already in the
remote's custody, so nothing unpushed is carried in.
Both halves of the caller contract are observed first and fail loud when violated: the branch
must not exist locally (`line_exists`, else `billet_seat` is the op) and its counterpart must
be known (`line_abroad`).

`billet_detach`::
Re-detach an existing billet at the trunk branch's remote counterpart — groom-billet reuse
(dispatch sheaf: "a groom billet in reuse re-detaches to trunk tip");
refuses on dirty.

`line_exists`::
Whether a named branch exists in the constellation — read-only, network-silent;
the observation behind the approach's create-or-seat choice at billet ensure.

`line_abroad`::
Whether a named branch stands ABROAD — whether its remote counterpart is known to the
constellation, as of the last `glean`;
the observation behind the approach's adopt-or-birth choice, consulted once `line_exists` has
answered no.
A line absent at home but standing abroad is another station's pushed work, and precisely the
line a birth here would rival.
Read-only and network-silent like its at-home sibling: the `glean` is the caller's beat, so an
unreachable remote leaves this answering from what was last seen rather than blocking a
dispatch.
Adopting late is the cost of working offline;
refusing the dispatch would be the larger one.

`outstripped`::
The staleness probe: does the trunk branch's remote counterpart — its position as of the last
`glean` — hold work the billet's position lacks?
A local ancestry check, network-silent, run after any `glean`;
`sync_state` cannot serve here, since it compares the billet's line against *its own*
counterpart, never against trunk's.
Answers false when no counterpart is locally known: nothing observed can be ahead, and the
staleness warning this probe feeds ({jjdd_refit}, dispatch sheaf) must not cry on ignorance.

`reachable`::
The stranding probe: is the tree's current position an ancestor of — or equal to — the named
branch's remote counterpart, as of the last `glean`?
A local ancestry check, network-silent, the {jjdd_stile_p} groom-litmus conjunct (dispatch
sheaf).
Answers false when no counterpart is locally known: nothing can be proven held on ignorance,
and the exit litmus this probe feeds must never destroy on an unproven claim — the opposite
polarity from `outstripped` above, stated so the two neighbors are never harmonized.

`billet_remove`::
Remove a billet from the yard.
Un-forced, it refuses on dirt — the {jjdd_stile_p} trailing-step form, whose refusal is the
same verdict `comb` renders (ignored files are invisible to both, so cold build residue
never blocks removal).
Behind an explicit force — the {jjdd_muck} door's confirmed destroy arm is the force's only
caller — it removes regardless: the one deliberate data-loss call in the taxonomy.

`enfold`::
Merge the trunk branch's remote counterpart — its position as of the last `glean` — *into* a
billet branch;
never rebase, fail-loud on conflict, resolution belonging to the attended session.
The caller names the trunk branch (trunk-ness is pedigree-relative, classified above the
trait); the kind resolves the counterpart itself, the `advance` posture — callers never speak
a kind-native ref dialect.
The operator's local trunk ref is never read, so a consigned billet can never carry unpushed
trunk work into remote custody: the ref stays unpushed while its content rides out as billet
ancestry, and a later rewrite of those still-mutable commits would strand the billet on
abandoned history.
The bare primitive beneath {jjdd_refit} (dispatch sheaf).

=== Deliberately absent

Trunk-ward merge (the operator's serializer — `enfold` merges only *into* billet branches),
rebase, clone, init, log, blame: operator's hands or {jjdo_log}.

== Toothing: officium open

{jjdxo_officium} open composes over the primitives above rather than adding one — no verb
names it (`jjrm_resolve_officium_billet`, `jjrm_mcp.rs`).
{jjdf_identify} resolves the tree at the officium's captured `cwd`: root, upstream key, seat,
line of work.
The composition keeps one of the four — seat — and adds the two the tree cannot supply: the
station name (`jjrm_station_name`, `jjrm_mcp.rs`) and the session, the officium's own bare id.
It holds the three as its billet: station, {jjdw_hippodrome}-or-{jjdd_billet}, session.
The middle member is carried as the seat itself — no separate enum: reading `Primary` as the
{jjdw_hippodrome} and `Partition` as a {jjdd_billet} is a premise the dispatch doors
underwrite (JJSVD-dispatch.adoc, "the doors are the sole entrance" — a hand-launched session
in a hand-made partition is a non-JJ session and opens no {jjdxo_officium}), and the deferred
billet-validation composition (JJSAS-state-repo.adoc) will enforce; it is never a fact
{jjdf_identify} itself asserts.
A station that reports no name opens no {jjdxo_officium} — there is no stand-in name, since
two unnamed stations would record the same station member and collapse the distinction it
exists to draw.
That refusal is {jjdxo_officium} open's own; {jjdf_identify}'s `ForeignGround` decline
propagates unchanged, and the composition adds no rejection kind of its own — the taxonomy
below stays tree-shaped.
+
Drained (260718, ₢B3·CAAAO): the staleness lead now composes in too —
`zjjrm_open_staleness_notice`, `jjrm_mcp.rs`.
It re-derives the same hippodrome/pedigree resolution `jjrds_plan` uses (dispatch sheaf), then
reuses the dispatch board's own post-glean ancestry probe verbatim
(`jjrds_staleness_notice`/`jjrfr_outstripped` — never `jjrfr_sync_state`, which compares a
tree only against its own counterpart and cannot tell a billet its *trunk* has moved).
Every resolution step is best-effort: foreign ground, no upstream key, an unfounded studbook,
or an unrecorded sire all silently yield no notice rather than blocking open — the probe must
never cry on ignorance, matching `jjrfr_outstripped`'s own contract.
Superseded: the aspirant toothing's `sync_state`-from-last-fetch staleness line
(JJSAF-farrier.adoc) named a different, weaker mechanism for this same row; this composition
does not use it.
+
The billet co-locates with the officium's exchange in the {jjdb_studbook}'s untracked
working area (JJSVS-studbook.adoc, "Officium exchange and record"): the enablement seam
(`jjrm_mcp.rs`, `JJRM_OFFICIUM_STUDBOOK_ENABLED`) flipped at the conversion heat
(JJSAS Founding-and-cutover), and the billet-resolution composition runs live, in place of
the former curia-resident path. The staleness lead above does not wait on that seam: it
re-runs {jjdf_identify} and the pedigree lookup itself, rather than reusing
`jjrm_resolve_officium_billet`'s result.

== Rejection kinds

One farrier-wide taxonomy of rejection *kinds*, wrapped in a context layer naming op, repo,
and detail.
A kind is a shared semantic fact — {jjdf_dirty_tree} means the same thing from
`billet_detach` or `billet_remove` — so consumers branch on kind, never on message text.
A new kind must be allocated in this sheaf, and named git-free per the {jjdf_palisade}.

The {jjdf_palisade} runs between kind and detail:
the kind is the git-free fact a consumer branches on;
the detail is the taxonomy's one membrane,
quoting the neighbor's evidence verbatim — stderr, porcelain lines,
and the exact foreign command an operator-typed remedy requires —
because precise characterization is Palisade conduct, never vocabulary adoption.
Detail text is for eyes alone:
no consumer parses it or branches on it,
and that standing prohibition is what makes the verbatim quote safe.

=== Which verdicts are surveyed

The taxonomy is closed and stays small: a driver classifies only the verdicts that earn a
kind, and every other failure keeps the loud unclassified panic — the panic is the
discipline, not a defect.
The Palisade's absorb-only-the-surveyed-signature conduct (CMK rules of engagement, the
headwater) is what bounds the survey; this is its farrier-side test.

A verdict is surveyed when all three hold:

* it is *field-observed*, or *structurally relied upon by a cinched invariant*;
* it is *probe-detectable* — exit code, porcelain output, or a registry read, never stderr
  prose, whose wording is the neighbor's to change without notice;
* it carries a *remedy the refusal can name*.

A signature failing any conjunct is left to the panic, which surfaces it whole for the
attended session rather than mislabeling it under a familiar kind.
Two consequences the census makes concrete: a failed push whose cause is an unreachable
remote rather than a ref rejection is not surveyed, because separating unreachable from
refused needs stderr prose;
and an `enfold` conflict is not surveyed, because its only remedy is the attended session's
own work.

=== The kinds

[[jjdf_foreign_ground]]
{jjdf_foreign_ground}::
{jjdf_identify_p} sole failure:
the probed tree is no claimed ground of any farrier kind.

[[jjdf_dirty_tree]]
{jjdf_dirty_tree}::
Uncommitted changes stand where the op requires a clean tree —
the same fact from `billet_detach` or `billet_remove`.

[[jjdf_diverged]]
{jjdf_diverged}::
The counterpart holds commits the offered line does not — a content race;
the remote's refusal of a non-fast-forward write, surfaced as fact.

[[jjdf_lock_held]]
{jjdf_lock_held}::
The guidon lock already flies another holder's claim.

[[jjdf_lock_broken]]
{jjdf_lock_broken}::
The lock no longer flies the leased guidon —
the holder's lease was severed out from under it.

[[jjdf_seat_vestige]]
{jjdf_seat_vestige}::
The constellation records a seat whose root is gone — an out-of-band removal of a billet
leaves the registration standing, and the next seat of that branch dies on it.
Detected by a registry read, never by message text, and remedied by pruning the registry —
*advice, never automatic*: the refusal names the command and the operator types it, because
a driver that tidies the operator's own destruction hides the fact that it happened.

[[jjdf_line_seated]]
{jjdf_line_seated}::
The named line of work is already seated in a live billet.
This is where the constellation's own seat exclusivity becomes legible to JJ: git enforces
one live checkout per branch for free, and this kind is that enforcement surfaced as a
domain fact, so an invariant resting on it rests on a named kind rather than on a foreign
refusal.
The remedy names the standing billet's root: work there, or reap it first.

The last two arrive at `billet_seat` from a single foreign refusal, rendered identically, and
are told apart only by the registry probe;
a probe that matches no seat at all leaves the panic standing, since classifying that arm
would mean reading the message text this sheaf bars.
