//axvd_sheaf axd_normative
This is a normative portion of the {jjs0_imprimatur}: binding — code, tests, and decisions may rely on it.
Create an L (landing) commit recording agent return from autonomous execution.

{jjds_arguments}

//axd_required
* coronet (positional) — {jjdt_coronet} of the pace that was executed

//axd_required
* agent (positional) — agent tier that executed (haiku, sonnet, opus)

//axd_optional
* `content` (string) — the landing's evidence body (defaults to empty), governed by the contract below.

The landing-content contract is normative and homed here; the Designee protocol in `claude-jjk-core.md` and the jjc-wrap ceremony cite it, never restate it.
A landing is _evidence for the reviewer, never advocacy_ — it points at what to verify, it does not argue the work is right.
Include only which commits are this pace's work, any mid-session operator ruling that changed scope or waived a docket clause (verbatim where possible), and what verification was run (suites / fixtures) with its outcomes.
Forbid arguments for why the work is correct, guide rules "considered", edge cases "handled", and scope-justification for unmet docket clauses — anything that pre-frames the review.
The reviewer reads the content as claims to verify, not findings to inherit; a designee writes it at land time, a full-ceremony agent never lands.

{jjds_stdout} Commit hash

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

. Parse coronet argument
. Format subject: `jjb:BRAND:₢XXXXX:L: {agent} landed`
. Format body: `content` parameter + Co-Authored-By
. Mint via `vvc::marker` primitive (index-invariant, guard-free)
. Output commit hash

{jjds_example}

MCP tool call with parameters: coronet=`AFABC`, agent=`sonnet`, content=`"Work: commit a1b2c3d. Verified: tt/vow-t green (639/0). Operator ruled the Done-when perf clause out of scope this pace."` — evidence pointers (commits, verification, the operator ruling), no argument that the work is correct.
