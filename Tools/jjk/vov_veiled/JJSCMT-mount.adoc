//axvd_sheaf axd_normative
This is a normative portion of the {jjs0_imprimatur}: binding — code, tests, and decisions may rely on it.
Return context needed to mount a
{jjdhr_heat}:
the current
{jjdpr_pace}
(first in
{jjdhm_order}
that is not
{jjdpe_resolved})
and its
{jjdkr_tack}
details.

{jjds_arguments}

//axd_required
* Target — the
{jjdhr_heat}
({jjdt_firemark})
or
{jjdpr_pace}
({jjdt_coronet})
to mount, self-typed by length.
Provided solely via a single {jjezs_halter} notice in `gazette_in.md` (lede = firemark or coronet); there is no target parameter.
{jjdo_orient}
mounts exactly one target — an absent gazette, an empty lede, or more than one
{jjezs_halter}
notice is a loud error, never a silent default.
This forces the agent's first gazette write to the start of the mount ceremony, where its
{jjdx_tool_s}
`Write` permission surfaces while the operator is present.

{jjds_stdout} Plain text format with labeled sections,
consumed by
{jjdo_orient}.
That operation additionally emits the paddock and next-pace docket as structured
{jjsz_gazette}
notices for the agent to read back.
Section `Racing-heats:` shows all racing heats in muster table format.
[source,text]
----
Racing-heats:
₣Fire    Silks                       Status  Done  Total
───────  ──────────────────────────  ──────  ────  ─────
₣XX      heat-silks-name             racing     3     10
...

Heat: <silks> (₣<firemark>) [<status>]
Paddock: <paddock_file_path>

Paddock-content:
  <line 1 of paddock content>
  <line 2 of paddock content>
  ...

Next: <pace_silks> (₢<coronet>) [<state>]

Docket:
  <line 1 of docket>
  <line 2 of docket>
  ...

Work-files (₢XXXXX — files this pace's commits touched):

  jjrg_gallops.rs  also: ₢XXXXB vos-add-routines-section, ₢XXXXD jjsa-session-probe-update
  jjrt_types.rs    also: ₢XXXXB vos-add-routines-section
  jjro_ops.rs

Recent-work:

-------------------------------
abcd1234  ₢XXXXX    ...
efgh5678  ₢XXXXX    ...
...

  haiku: claude-3-5-haiku-20241022
  sonnet: claude-sonnet-4-20250514
  opus: claude-opus-4-5-20251101
  host: macbook-pro.local
  platform: darwin-arm64
----

Sections `Next:` through `Work-files:` are absent if no actionable pace exists.
Field `pace_silks` is derived from `tacks[0].silks`.
Section `Paddock-content:` contains the full text of the paddock file, indented 2 spaces.
Section `Work-files:` is the mounted pace's digest: the files that pace's own commits touched (excluding `.claude/jjm/*`), sorted, one per line, each naming inline the other paces that touched it — collision awareness for the work about to begin.
A pace with no commits has no digest, and the section is absent entirely; that is the ordinary case at mount, since a pace is usually mounted before any work lands against it.
The digest names the colliding paces in full rather than by a legend mark, so the section is read without decoding anything outside it.
{jjdo_orient}
renders no heat-wide retrospective — neither the file-touch census nor the commit swim lanes that
{jjdo_show}
carries.
Both answer a planning question a mount is not asking: the heat-wide view serves the session choosing *what* to work, while a mount has already chosen, and on a wide heat the two blocks cost more context than the docket they precede.
The heat-wide census is reached deliberately, through
{jjdo_show}.
Section `Recent-work:` contains the last 10 steeplechase entries for the heat (from
{jjdo_log}),
formatted as column-aligned table.
Section `Session-probe:` contains model IDs and system context collected via session probe, indented 2 spaces, one key-value per line.

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

. Iterate
{jjdgm_order}
to collect heats with status
{jjdhe_racing}
in priority order.
Render column-aligned table using same format as
{jjdo_list}
(₣Fire, Silks, Status, Done, Total).
Output as `Racing-heats:` section.
. Read
{jjdgr_gallops}
. Resolve the target from the single
{jjezs_halter}
notice in `gazette_in.md` (firemark or coronet, self-typed by length); error if the gazette is absent, the lede empty, or more than one notice is present
. Verify
{jjdhr_heat}
exists with given
{jjdt_firemark}
. Error if
{jjdhm_status}
is
{jjdhe_stabled}
("Cannot mount stabled heat {firemark}")
. Output `heat_silks` from
{jjdhr_heat}
. Output `paddock_file` from
{jjdhm_paddock}
. Read paddock file contents, output as `paddock_content`
. Resolve the
{jjdpr_pace}
to mount, by the target's type:
.. A
{jjdt_coronet}
target is looked up directly in the
{jjdhr_heat}.
A
{jjdpe_resolved}
pace is a calling error naming its state: it is finished work, and
{jjdo_orient}
mounts work that stands.
.. A
{jjdt_firemark}
target walks
{jjdhm_order}
in sequence and takes the first
{jjdpr_pace}
that is not
{jjdpe_resolved}
(first match wins).
The predicate decides this, never the state name: a
{jjdpe_bridled}
pace resolves exactly as a
{jjdpe_rough}
one does, since a designation marks work as ready to execute, never as executed.
. For the resolved
{jjdpr_pace}:
.. Output `pace_coronet`,
`pace_silks` (from `tacks[0].silks`),
`pace_state`
.. Output `docket` from `tacks[0].text`
.. Query steeplechase entries for heat, filter by actionable pace coronet, run `git diff-tree` per commit, collect unique work file names (excluding `.claude/jjm/*`); for each such file, name the other paces in
{jjdhm_order}
whose commits also touched it; output as the `Work-files:` digest (2-space indented, sorted, one per line, colliding paces named inline by coronet and silks).
Emit nothing when the pace has no commits
. If no actionable pace found, output only `heat_silks`, `paddock_file`, `paddock_content`, and `recent_work`
. Get last 10 steeplechase entries for the heat via
{jjdo_log}
logic, output as `recent_work` array
. If session probe is needed (gap > 1 hour since last commit), call vosr_probe routine and output `Session-probe:` section with model IDs (haiku, sonnet, opus) and system context (host, platform)
. Apply the designation gate: judge the resolved identity against the calling session's tier.
A
{jjdpe_bridled}
{jjdpr_pace}
admits exactly the tier it was designated for — strict equality, no frontier carve-out, because the persisted
{jjdcm_tier}
is honest provenance of the session that executed the work, not a ceiling under which an abler session may pass.
A
{jjdpe_rough}
{jjdpr_pace}
is judgment work and admits frontier tiers alone; so is a
{jjdhr_heat}
that resolved no actionable
{jjdpr_pace},
which holds nothing designated for a sub-frontier session to execute.
The gate lands after resolution and never filters it.
A tier-mismatched
{jjdpr_pace}
is refused *at*, never walked past to a later one:
{jjdhm_order}
is the dependency tree, so the next actionable
{jjdpr_pace}
is the work that stands next whatever tier the caller holds.
A refusal is an
{jjdz_interdictum},
and takes its message law from that genre.
The barred act leaves nothing behind, so
{jjdo_orient}
discards the context it assembled: no
{jjsz_gazette}
notice is written, and no
{jjdxw_emblem}
is recorded.
This is what makes the refusal a halt rather than a speed bump — a refused mount hands the session no docket, no paddock, and no mounted identity to carry forward, so there is nothing to proceed *from*.
Only a frontier session's
{jjdo_apostille}
lifts the bar, which is what places the condition outside the call.
. Record the resolved work identity for the {jjdxw_emblem} — the actionable
{jjdt_coronet},
or the heat
{jjdt_firemark}
when no actionable pace exists.
As a mount,
{jjdo_orient}
asserts it unconditionally; the coronet-sticks rule that keeps a later groom from demoting a
{jjdt_coronet}
is stated once at
{jjdxw_emblem}.
