== The association graph

The scope containment of claim rules is a tree;
the declared interrelations make it a graph.
A swingletree is a typed edge between two tackles — *specifies*, *tests*, *crosscuts* are the
census-observed types; the taxonomy stays open — and an edge can carry its own governing
discipline:
ACG is the type specimen, governing the code↔spec citation relation (reference-the-home,
rivet residue) rather than either side alone.

*Declared, never deduced.*
Every edge is authored — the pedigree-ceremony posture again — and traversal is mechanical:
no similarity inference, no embedding neighborhood, no linguistic guess (the VOSMM
mechanical-never-linguistic ethos, VOr_m7w, applied to context provisioning).
This is what makes the graph trustworthy enough to *govern* rather than merely suggest.

*Boxes before arrows* (sequencing comfort): the concept model holds nodes and edges, but the
build sequence need not — an MVP of nodes alone is complete and useful on its own, with
swingletrees arriving when their first consumer (school, or an ACG-citation lint) actually
lands.
The canon sheaf's document members are that ruling enacted, as flattened edges;
what flattening drops is exactly the pairing a relation-consumer needs (that ACG governs the
citations *into that spec set*), so those members normalize into swingletrees at the same act
that lands the first consumer — ACG the first migrant.
The grammar stays tiny either way: a node is a row, an edge is a tagged pair of rows;
sophistication lives in what consumers do with the tags, app-side, per the substrate split.

=== The consumer horizon

Vision, not this sheaf's design.
A frontier-tier planning session judges "this implementation needs balancing against this
spec" *once*, captures the judgment as a declared edge, and every later provisioning replays
it mechanically — school walks the graph to queue exactly the context a weaker session needs
to execute cheaply.
A weak model cannot discover missing context; the declared graph is the precondition for
handing work down-tier at all.
The same freeze pattern as the grimoire: authored once with full wisdom, read forever by
exact match.

The type-specimen use case is the recurring failure where a model writes bash from ambient
training habits unless reminded to read BCG.
The binding is what lets school *mechanically* place a read-the-governing-guide beat in the
beat map for any beat whose file scope lands in a BCG domain — retiring the operator's memory
as the enforcement mechanism.
Tackle's part is only the binding and the integrity declarations; the beat-shape is gait
design, breeze-side.
Execution-side design (gait affinities, blaze resolution, exclusivity) remains joint work
with the breeze sheaf (JJSAB, Reconciliation section) and is deferred below.

== The substrate split (direction — not yet an API commitment)

Where tackle services divide between the shared substrate crate (the vvx-land library both
JJ and the matricula read) and the applications above it.
The dividing line is the matricula's own mechanical/judgment boundary (VOr_m7w), applied to
tackle.
The write posture and the name-blindness law are settled and canon (tackle sheaf: JJ sole
author, matricula reads and lints; names in data, shape in code);
what stays direction is the *interface*:

* *The crate owns the mechanical half* — everything that is set operations over declared
  rules, no judgment: schema and load, claim-rule evaluation (path → verdict), roster
  derivation (rule × tree state → files), coverage subtraction (bounds minus claims),
  declared-edge traversal.
  Four query surfaces, matching the two query directions the canon sheaf records: resolve one
  path, derive one roster, audit coverage, walk edges.
* *The applications own the judgment half* — what to *do* with verdicts: JJ dispatch decides
  provisioning policy (honing, conduct core, pull door, warn-vs-refuse at the loudness
  seats); the matricula maps governance members to vestures and scan roles; school composes
  beats.
  Policy never sinks into the crate; rules never float into the apps.
* *Not yet claimable*: signatures and cache mechanics.
  The shape above is a lean the matricula's interim allowlist reader should stay compatible
  with, not an interface to build against.
* *Distribution*: the crate ships (it is the reader the direction already places in the
  shipped substrate library); the matricula consumes it and never flows into it (VOr_q4f).

== Deferred (explicitly not MVP) — the banked maximal layer

Banked from the retired V4 workup so the ideas survive their source heat; each is a future
design decision, not a commitment:

* *₿ blazes, the deferred remainder* — the definition mechanism itself landed with the MVP

  Deferred: arbitrary per-domain *resource* blazes beyond documents (fixture data, scratch
  roots) that dockets and gaits reference and school resolves when writing warrants —
  breeze never sees ₿ symbols.
  Validation sketch stands: no duplicate definitions, no orphan references.
* *Actions* — named commands with exclusivity scopes (candidate enum: worktree / workstation /
  project), the orchestrator holding mutexes during concurrent dispatch when an action's
  scope demands it.
* *Gait affinities and read-also entries* — which gaits a tackle suits, and standing context
  attachments.
* *Integrity declarations, the deferred remainder* — the check members landed MVP (the
  CLAUDE.md after-code-changes suite table is this content, hand-maintained today).
  Deferred: quality requirements around them (retry-up-to-N / fail-fast) and the
  chukker-boundary policy that consumes them, breeze-side.
* *Tackle–gait interface* — gaits declare required blazes; compatibility validates at school
  time.
* *Registry shape* — tackles as silks-identified, mutable resource bundles in a top-level
  registry beside the gait registry.
* *Per-heat tackle mutability* — MVP tackle is per-sire standing config.
  A provisional reconciliation (unreviewed supposition — design review required):
  the per-sire tackle set is the standing base; a heat may later overlay heat-scoped
  additions (blazes, actions) that reference and extend the base; "tackle" names the layered
  whole.
  This bullet is that fork's one home, and it is one of the two channels that would re-open

== Open forks (the mulling surface)

* *The central-corpus home* — the reference form is canon: first-class registry entries keyed
  by ₿-sigiled role words, with the skip-when-absent corpus member naming a holding corpus
  when the document is not in the sire's own tree.
  The recorded rationale: a role outlives any particular binding (the day the bash guide
  changes, the key does not lie); one home per document, moved once; whittled per-project
  variants and version-pinned experiments hang naturally on an entry (intended
  partial-release whittling of BCG; one guide at two versions, outcomes compared);
  and late-bound roles make a crossbred row *more* self-contained semantically, not less.
  Open: which private, never-distributed home holds the central corpus —
  the studbook as its first documents tenant (operator inclination: one regional data
  home; friction: the blotter is engine-written and linear, so hand-authored documents
  need a declared carve-out area or an edit ceremony),
  the kit repo (chartered "engine + provisioned context", adds no repo),
  or a standalone guides repo (cleanest lifecycle isolation, but a new repo).
  The first `jjotrn_corpora` entries and their shape land with that home;
  the corpus members' values are corpora keys, plural by design.
* *Claim-rule kinds* — the aspirant-sheaf domain exposes a new need: some real domains are
  defined by a *declared in-file marker* (the `axvd_sheaf axd_aspirant` lintel), not by
  path shape — aspirant sheaves are only partly path-regular: the JJ family rides `JJSA*`,
  but VOSMM, RBSCV, and RBSPC carry the aspirant lintel inside their codex directories with
  no path marker.
  The lintel is designed for mechanical reading (AXLA), so a lintel-predicate claim would
  still be declared-never-deduced — the declaration simply lives in the file instead of
  the table.
  Lean, per simplicity: the claims stay glob-only and the census *audits* the lintel (a
  file whose lintel carries `axd_aspirant` outside the aspirant tackle's claims is a
  finding); a lintel-predicate claim kind is the anticipated growth if that audit's
  maintenance burden proves real.
* *Loudness seats* — where the coverage remainder fires: the matricula census (repo
  hygiene, batch), saddle time (the session is about to touch an unaffiliated file — the
  pull-door shape), or both; and warn vs refuse at each seat.
* *Check-command semantics* — deferred until the base farrier is built and the commands
  have a live consumer; needs real design, not a drive-by cinch: what order means on
  failure (run-all vs stop-on-first), whether the members interdepend (a test presumes its
  build), tier ladders (a fast check vs a release suite), and where declared semantics stop
  before they recreate gait-side policy.
  One shape point leans settled ahead of the rest: every check entry is a tabtarget —
  self-logging, dispatch-disciplined, and operator-authored, so a check is a deliberate
  authorship act rather than a raw string executed on faith; for a foreign body the
  tabtarget lives JJ-side (the vanilla-hippodrome premise bars planting one in the governed
  tree; its infield home rides this same deferred design), which contains — never removes —
  the inherent run-their-code exposure of verifying a foreign body.
  Until settled, consumers treat the check lists as surfaced advice, never an executed
  contract (as the canon sheaf states).
* *Edge-tag taxonomy* — specifies / tests / crosscuts are census-observed, not yet an enum;
  more census before it deserves one.
* *Verdict naming* — held with the estray question (Candidate vocabulary).
* *Remaining word elections* — verdict names; the corpus names (with the central-corpus
  home); role-word conventions for the registries, if practice wants any.
  Elections run gates at a census-style pace, not mid-mulling.
  Word-health evidence banked for that gate: tackle itself sits two letters from the
  minted gallops tack — an audible-register hazard; the family re-ratifies with this
  evidence weighed.
