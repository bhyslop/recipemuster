== Saddle, billet, unsaddle — the dispatch layer

*Cinched (260704): JJ opens the Claude Code sessions we work in.*
The operator knows the verb at chat-top; the session's billet must be right at launch
(mid-session cd neither reloads project context nor survives the cd prohibition).

* **saddle** — the work dispatch, run at the shell.
  A coronet saddles that pace; a firemark saddles the heat's *next actionable pace* — mounting
  a heat has always meant mounting its first pace, and the firemark spelling keeps that meaning
  (re-cinched 260704, retiring the earlier length-picks-protocol collapse that spent the
  firemark case on grooming).
  Either way the session gets a *pace billet* (worktree + §F branch at trunk tip).
  (The bash toothing cannot be "mount" — Unix owns that word; the two-layer split is the JJS0
  vocabulary-isolation doctrine appearing at a new boundary.)
* **lunge** — the groom dispatch (minted 260704): working the horse from the ground,
  deliberately unmounted.
  A firemark gets a *groom billet* (worktree at detached trunk tip — nothing must survive).
* Both doors share one *entrance spine*: ensure the billet, glean (never merge — refit is the
  staleness remedy), export per-billet BURV
  roots, provision JJ context, and launch Claude Code in it with the engagement verb as opening
  prompt — and the verb itself carries the tier signal (lunge → planning tier, saddle → work
  tier), so no protocol-sniffing.
* **billet** — the standing thing a session runs in: a root with a lifecycle on the
  partition axis (worktree the plain-git implementation; full clone the fallback for kinds
  without worktrees — re-anchored per the banked delta, canon at `JJSVF-farrier.adoc` Two
  axes) + branch-or-detached-tip + env posture.
  The lifecycle entity the officium records and validates; the object noun completing the
  verbs-act-on-nouns grammar (mount acts on a pace; saddle creates a billet).
  Board on arrival, kept for the duration, collected after.
* **unsaddle** — the reap: `billet_remove`, refuse-if-dirty.

Lifecycle doctrine:

* *Billets are chat-ephemeral; branches are durable.*
  The §F branch is the WIP carrier across chats, journaled in pace state; the worktree is
  disposable scaffolding.
  `git worktree list` is the authoritative local registry of which branch is seated where —
  **no worktree path is ever written into the studbook** (station-local fact, regional record).
* *Refit by merge, never rebase.*
  Refit merges trunk *into* the pace branch: journaled counterfoils survive, conflicts
  surface once, and the branch's interior history dies at the operator's trunk merge anyway.
  On conflict, the mount session resolves as its first order of business — *attended* is the
  load-bearing property, not tier; saddle may bump the tier when it detects a pending conflict.
  Tier-escalation machinery is deferred to breeze.
* *Mount absorbs cold start* — billet-ensure, glean, orient. No new engagement verb.
* *Every dispatched session gets its own billet; trunk belongs to the operator alone.*
  Session isolation is structural, not disciplinary — even a "read-only" groom can run builds.
  Grooms get billets too (detached), so concurrent sessions on one hippodrome never share a tree.
  Git itself enforces the split: a branch checked out in one worktree cannot be checked out in
  another — pace billets own unique §F branches and groom billets are detached, so the
  constraint never binds, and a billet mechanically *cannot* seat the trunk branch while the
  hippodrome holds it.
  (Worktrees share the hippodrome's object store — no history duplication, no fetch between
  billet and hippodrome; the billet-side cost is working files and cold build artifacts only.)
* *The dispatch verbs are the sole door to JJ* (cinched 260704; saddle and lunge share the one
  entrance spine).
  A bare hand-launched session carries no MCP registration and no JJ context — it is simply a
  non-JJ session, forever legitimate as such.
  No fallback paths until genuinely forced; the consequence is that the entrance must be
  pleasant very early in the MVP — it is load-bearing, not a convenience.
* Stale billets: the muck sweep at dispatch time (paddock cinch 260705, superseding the
  earlier reaped-at-next-saddle-if-clean line) — plan-then-confirm nuke-on-start, retiring
  billets whose pace is closed past a retention window equal to the officium exsanguination
  window; refuse-with-advice on dirty anomalies by default, auto-commit-push behind explicit
  confirm; pace states snapshot under the studbook lock, then reaping proceeds lock-free
  (completion is monotonic); liveness is the pure JJ-data join (live officia × billets), a
  platform process-probe only ever belt-and-braces.

== The BUK meld (studied 260704)

Findings from the dispatch spine (`z-launcher.sh` → `bul_launcher.sh` → regimes):

* BUK dispatch *self-anchors*: cwd normalizes to the invoked tree's root; moorings and burc.env
  are committed and present in any worktree.
  BUK cannot tell a billet from a clone.
* The station family (`../station-files/burs.env`, `../logs-buk`, `../temp-buk`,
  `../output-buk`) resolves to the parent directory — the infield.
  Billets placed as infield peers inherit station geography with zero setup; nesting billets
  deeper would break it, so infield-peer placement is load-bearing.
* The genuine hazard is concurrency, not geometry, and it predates billets:
  `../output-buk/current/` is cleared per dispatch and `logs-buk/last.txt` is a singleton.
  Saddle closes it by exporting per-billet `BURV_OUTPUT_ROOT_DIR` / `BURV_TEMP_ROOT_DIR`
  (absolute paths, theurge's own discipline).
* *Theurge needs no change*: it sets BURV via `Command::env()` with self-constructed absolute
  per-invoke paths — innermost setting wins, nobody composes with inherited values.
  The protocol is "whoever dispatches owns the knob."
* Named behavior change: chain facts (`current/`→`previous/`) become *billet-scoped* — a
  tabtarget inside the billet and one in the trunk terminal no longer see each other's flow.
  This also closes a latent cross-clone chain race that exists today.
* Elected (260704): *build* the `BURV_LOG_DIR` override — a few launcher lines mirroring the
  existing BURV pattern, exported by saddle beside its siblings, making the `last.txt`
  read-back race-free.
  Meticulous execution demanded by the operator; slots as an early standalone pace at cut time
  (self-testable via `buw-st`), not a groom-session side-edit.

== Launch-time provisioning (the vanilla-hippodrome consequence)

Hippodromes carry no JJ inserts (canon: the revision-control cosmology sheaf), which
re-gestalts JJK installation:

* MCP config, the verb table, and JJ context arrive *at session launch from saddle*
  (session-scoped config + opening prompt), not from repo-resident inserts.
  The project's own CLAUDE.md — whatever the vanilla project ships — still governs its code.
* The destination is *generated, work-scoped context* (guideline cinched 260704): the app
  prints context honed to the elected heat/pace — acronym rows the tackle bindings say this
  work touches, the verb table cut to the session type.
  Default stance until that intelligence exists: retain *everything* currently in
  CLAUDE.md / claude-jjk*.md content, removing pieces only as honing is actually built
  (perhaps with tackle refinement).
* Honing is prediction, and starvation fails *silent* — the full table is also the index of
  what exists, so a starved session doesn't know what it doesn't know.
  Two standing repairs: an invariant *conduct core* ships unhoned in every provision
  (never-reach-past, commit discipline, additive-only), and a *pull door* — a context-lookup
  verb named in the core — turns starvation into one extra round-trip instead of silent
  ignorance.
  Selection is safe only when paired with discoverability.
* Officium exchange (gazettes) relocates into the studbook clone's untracked working area
  (cinched — state-repo aspirant sheaf, Officium re-gestalt).
* Tackle bindings (tackle sheaf) supply per-scope discipline context at runtime, replacing
  repo-grain `@`-includes with file-grain binding.

Session tiers: the dispatch verb carries the tier — lunge at high-effort frontier tier
(groom/school-grade planning), saddle at the work tier per pace character, orchestration
(future breeze) at haiku.
Programmatic session dispatch for personal tooling is documented product surface (headless mode,
Agent SDK, `--model`); recording that standing determination here is what keeps it from being
re-litigated from memory.
