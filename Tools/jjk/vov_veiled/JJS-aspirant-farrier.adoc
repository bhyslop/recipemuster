== What the farrier is

The *farrier* is JJ's polymorphic revision-control driver: the layer that fits the same JJ intent
to whatever ground a repo runs on — plain git today, git-with-worktrees within it, submodules and
Android `repo` constellations later.
As a farrier shoes the same horse for different terrain, one driver interface serves every
backend; the pedigree (`JJSVS-studbook.adoc`, `jjdb_pedigree`) selects the farrier kind per
upstream.
The studbook itself is driven through the same interface's lock facet — the degenerate
single-repo case.

The farrier supplies *primitives*; the ceremonies above it (journal ceremony, break sequence,
saddle) stay JJ's.
No driver ever owns a transaction.

== Two axes, two generalization points

The eventual backends sit on two orthogonal axes, each with one generalization point that must be
right in the MVP so later rungs add without re-minting:

* *Constellation axis* — what a counterfoil names: one repo (scalar SHA) vs. many members.
  Generalization point: the **counterfoil** is always a member→SHA manifest; single-repo is
  the one-element case.
* *Partition axis* — who writes concurrently: shared checkout vs. isolated per-work checkouts.
  Generalization point: the **billet** (below); every op takes an explicit repo root, so a
  partition is just another root — op signatures never change when worktrees arrive.

== Hearting ops (MVP census)

Primitives are *capabilities, not commands* — almost none are ever human-typed.
Boring names by JJS0 Upper API doctrine, with a principled line (re-cinched 260704, superseding
the earlier keep-the-git-word-on-faithful-subsets rule): *the trait surface is a vocabulary
Palisade* — git words stop at the driver implementation boundary.
Below the trait, git vocabulary is the neighbor's own language (Palisade conduct: never
legislate the neighbor); above it — toothings, specs, error kinds, everything a session reads —
a git word is interface contamination.
A faithful-subset git name is correct only up to an invisible boundary, and the boundary is
exactly what a lesser model does not hold: `commit` fires the amend/stage-all reflex, `push`
fires force in a recovery scenario — the discipline's forbidden list *is* the git reflex set.
A fresh word fails loud (the session looks it up — the provisioning pull door exists for
precisely this); a familiar word fails silent.
Census names stay boring-register but git-free — distinct and vivid are different axes, and no
asterism budget is spent below the vivid layer (`advance`, `enfold`, `identify` prove the
register).

Re-minted at the census pace (260705, under MCM Lapidary, all gates run fresh):
`status`→`comb` (comb the tree for dirt), `commit`→`lodge` (formally deposit the explicit
list — the administrative filing sense), `fetch`→`glean` (gather what the remote holds),
`push`→`consign` (hand the branch into the remote's custody).
The lock trio settled as guidon verbs: `ref_cas_create`→`stake` (stake the guidon in — the
claim made atomic), `ref_cas_delete`→`pluck` (pluck it out), `ref_read`→`sight` (sight whose
guidon flies, read-only).
The trio spends vividness below the vivid layer deliberately, overruling the counter-lean:
create/delete/read fire the CRUD reflex exactly as push fires force — a failed create invites
a "cleanup" delete — and the lock is the maximum-danger facet, where a reflex-firing name
costs most.
Of the seed verbs, plant is taken (`jjx_plant`, the live foray command), strike is taken
(RBSME escheat's orphan sweep), and read is barred outright as the CRUD/git reflex word.
`advance` weighed and retained: git-free and boring-register — it is the register's own
proof-word; English-common is the hearting posture, not a defect, below the vivid layer.

|===
|Op |Facet |Shape |Consumer

|`identify` |orient |Resolve this tree: dirname, origin URL (pedigree key), branch or
detached state, HEAD, billet-of-which-hippodrome if a worktree. |saddle/lunge (dispatch-time);
ambient on `jjx_open`

|`comb` |inspect |Clean/dirty, per-path — feeds clean-tree gates and explicit-file commits.
|sequence-internal

|`sync_state` |inspect |Ahead/behind vs. remote, from last fetch (never blocks on network).
|ambient at open/orient

|`counterfoil` |inspect |HEAD member→SHA manifest + branch annotation + dirty flag. |ambient
on record

|`counterfoil_verify` |inspect |Does a journaled SHA still exist, reachable? *Deferred past
MVP* (260704): the designed answer for when a dangling counterfoil first annoys —
merge-based refit and notch-push make them rare. |deferred

|`lodge` |mutate |Explicit file list + message; no `-a`, no amend (additive discipline).
|sequence-internal

|`glean` |sync |Update remote refs. |sequence-internal, opportunistic

|`advance` |sync |Fast-forward-only move to remote tip; fails loud if diverged.
Deliberately not named "pull" — the farrier never merges or rebases. |journal ceremony

|`consign` |sync |Two flavors: plain fast-forward (hippodrome), `--atomic` + lease (studbook
content). Never force. |sequence-internal

|`stake` / `pluck` / `sight` |lock |Studbook/mews only; consumed by the
lock guard and break sequence. |sequence-internal

|`billet_create` / `billet_remove` |billet |Birth/reap a billet (below); remove refuses on
dirty. |saddle / unsaddle

|`enfold` |billet |Merge trunk *into* the billet branch (never rebase); fail-loud on
conflict — resolution belongs to the attended session. |refit (the session-facing resync
operation)
|===

Cross-cutting rules: every op takes the repo root explicitly (no cwd dependence); every mutating
op rejects in a named typed-error band (dirty-tree, diverged, lock-held, lock-broken) — sequence
composition is only safe when primitives fail loud and specifically.
The taxonomy is settled (260704, RCG-checked): one farrier-wide enum of rejection *kinds* — a
kind is a shared semantic fact ("diverged" means the same from `advance` or `consign`) — wrapped
in a context layer naming op, repo, and detail.
RCG defers error idioms by design; the binding disciplines (Zeroes, Identity Rule, String
Boundary, Output) all pass, and the typed taxonomy is a deliberate step up from the crate's
habitual `Result<T, String>` — settled early because error shapes calcify into every op
signature.
Deliberately absent: trunk-ward merge (the operator's serializer — `enfold` merges only
*into* billet branches), rebase, clone, init, log, blame — operator's hands or `jjx_log`.

Entity design: a driver trait with one implementation per farrier kind; the lock is an RAII
guard (object lifetime = lock lifetime, panic on nested acquire); break is a separate sequence
acting on someone else's lock, never a guard method.

== Toothing compositions

The journal ceremony (its own routine sheaf, `JJSVJ-journal.adoc`) is the shared write
bracket; toothings compose the primitives:

|===
|Toothing |Composition

|*(officium open — not a spoken verb)* |`identify` (billet, pedigree, branch, HEAD) +
`sync_state` from last fetch

|mount / orient |studbook read (no lock) + `sync_state` staleness line

|muster / parade / groom |studbook reads only

|notch |hippodrome: `comb` → `lodge` (explicit files) → `counterfoil` — then the journal
ceremony carrying the counterfoil

|wrap |as notch, plus pace-state mutation in the journal

|slate / reslate / paddock |journal ceremony only — no hippodrome ops

|saddle / lunge |shared entrance spine: `identify` → pedigree lookup →
`billet_create`-or-reuse → `glean` (the spine never merges — staleness reports at open and
refit is the remedy; groom billets re-detach to tip) → BURV export →
launch session

|unsaddle |`billet_remove`, refuse-if-dirty

|break |`sight` every JJ lock (studbook + mews) → report guidons → lease-guarded

A ring-survey toothing was considered and *demoted* (260704): the studbook dissolved
cross-clone state sync, billets partition parallel work, saddle housekeeping reaps stale
billets, and `sync_state` reports at open — if a whole-infield status panel ever earns its
keep, it is a report mode, not a verb.

== Saddle, billet, unsaddle — the dispatch layer

*Cinched (260704): JJ opens the Claude Code sessions we work in.*
The operator knows the verb at chat-top; the session's billet must be right at launch
(mid-session cd neither reloads project context nor survives the cd prohibition).

* **saddle** — the work dispatch, run at the shell.
  A coronet saddles that pace; a firemark saddles the heat's *next actionable pace* — mounting
  a heat has always meant mounting its first pace, and the firemark spelling keeps that meaning
  (re-cinched 260704, retiring the earlier length-picks-protocol collapse that spent the
  firemark case on grooming).
  Either way the session gets a *pace billet* (worktree + §F branch at trunk tip).
  (The bash toothing cannot be "mount" — Unix owns that word; the two-layer split is the JJS0
  vocabulary-isolation doctrine appearing at a new boundary.)
* **lunge** — the groom dispatch (minted 260704): working the horse from the ground,
  deliberately unmounted.
  A firemark gets a *groom billet* (worktree at detached trunk tip — nothing must survive).
* Both doors share one *entrance spine*: ensure the billet, glean (never merge — refit is the
  staleness remedy), export per-billet BURV
  roots, provision JJ context, and launch Claude Code in it with the engagement verb as opening
  prompt — and the verb itself carries the tier signal (lunge → planning tier, saddle → work
  tier), so no protocol-sniffing.
* **billet** — the standing thing a session runs in: worktree + branch-or-detached-tip + env
  posture.
  The lifecycle entity the officium records and validates; the object noun completing the
  verbs-act-on-nouns grammar (mount acts on a pace; saddle creates a billet).
  Board on arrival, kept for the duration, collected after.
* **unsaddle** — the reap: `billet_remove`, refuse-if-dirty.

Lifecycle doctrine:

* *Billets are chat-ephemeral; branches are durable.*
  The §F branch is the WIP carrier across chats, journaled in pace state; the worktree is
  disposable scaffolding.
  `git worktree list` is the authoritative local registry of which branch is seated where —
  **no worktree path is ever written into the studbook** (station-local fact, regional record).
* *Refit by merge, never rebase.*
  Refit merges trunk *into* the pace branch: journaled counterfoils survive, conflicts
  surface once, and the branch's interior history dies at the operator's trunk merge anyway.
  On conflict, the mount session resolves as its first order of business — *attended* is the
  load-bearing property, not tier; saddle may bump the tier when it detects a pending conflict.
  Tier-escalation machinery is deferred to breeze.
* *Mount absorbs cold start* — billet-ensure, glean, orient. No new engagement verb.
* *Every dispatched session gets its own billet; trunk belongs to the operator alone.*
  Session isolation is structural, not disciplinary — even a "read-only" groom can run builds.
  Grooms get billets too (detached), so concurrent sessions on one hippodrome never share a tree.
  Git itself enforces the split: a branch checked out in one worktree cannot be checked out in
  another — pace billets own unique §F branches and groom billets are detached, so the
  constraint never binds, and a billet mechanically *cannot* seat the trunk branch while the
  hippodrome holds it.
  (Worktrees share the hippodrome's object store — no history duplication, no fetch between
  billet and hippodrome; the billet-side cost is working files and cold build artifacts only.)
* *The dispatch verbs are the sole door to JJ* (cinched 260704; saddle and lunge share the one
  entrance spine).
  A bare hand-launched session carries no MCP registration and no JJ context — it is simply a
  non-JJ session, forever legitimate as such.
  No fallback paths until genuinely forced; the consequence is that the entrance must be
  pleasant very early in the MVP — it is load-bearing, not a convenience.
* Stale billets: reaped at next saddle if clean; flagged, never auto-destroyed, if dirty.

== The BUK meld (studied 260704)

Findings from the dispatch spine (`z-launcher.sh` → `bul_launcher.sh` → regimes):

* BUK dispatch *self-anchors*: cwd normalizes to the invoked tree's root; moorings and burc.env
  are committed and present in any worktree.
  BUK cannot tell a billet from a clone.
* The station family (`../station-files/burs.env`, `../logs-buk`, `../temp-buk`,
  `../output-buk`) resolves to the parent directory — the infield.
  Billets placed as infield peers inherit station geography with zero setup; nesting billets
  deeper would break it, so infield-peer placement is load-bearing.
* The genuine hazard is concurrency, not geometry, and it predates billets:
  `../output-buk/current/` is cleared per dispatch and `logs-buk/last.txt` is a singleton.
  Saddle closes it by exporting per-billet `BURV_OUTPUT_ROOT_DIR` / `BURV_TEMP_ROOT_DIR`
  (absolute paths, theurge's own discipline).
* *Theurge needs no change*: it sets BURV via `Command::env()` with self-constructed absolute
  per-invoke paths — innermost setting wins, nobody composes with inherited values.
  The protocol is "whoever dispatches owns the knob."
* Named behavior change: chain facts (`current/`→`previous/`) become *billet-scoped* — a
  tabtarget inside the billet and one in the trunk terminal no longer see each other's flow.
  This also closes a latent cross-clone chain race that exists today.
* Elected (260704): *build* the `BURV_LOG_DIR` override — a few launcher lines mirroring the
  existing BURV pattern, exported by saddle beside its siblings, making the `last.txt`
  read-back race-free.
  Meticulous execution demanded by the operator; slots as an early standalone pace at cut time
  (self-testable via `buw-st`), not a groom-session side-edit.

== Launch-time provisioning (the vanilla-hippodrome consequence)

Hippodromes carry no JJ inserts (canon: the revision-control cosmology sheaf), which
re-gestalts JJK installation:

* MCP config, the verb table, and JJ context arrive *at session launch from saddle*
  (session-scoped config + opening prompt), not from repo-resident inserts.
  The project's own CLAUDE.md — whatever the vanilla project ships — still governs its code.
* The destination is *generated, work-scoped context* (guideline cinched 260704): the app
  prints context honed to the elected heat/pace — acronym rows the tackle bindings say this
  work touches, the verb table cut to the session type.
  Default stance until that intelligence exists: retain *everything* currently in
  CLAUDE.md / claude-jjk*.md content, removing pieces only as honing is actually built
  (perhaps with tackle refinement).
* Honing is prediction, and starvation fails *silent* — the full table is also the index of
  what exists, so a starved session doesn't know what it doesn't know.
  Two standing repairs: an invariant *conduct core* ships unhoned in every provision
  (never-reach-past, commit discipline, additive-only), and a *pull door* — a context-lookup
  verb named in the core — turns starvation into one extra round-trip instead of silent
  ignorance.
  Selection is safe only when paired with discoverability.
* Officium exchange (gazettes) relocates into the studbook clone's untracked working area
  (cinched — state-repo aspirant sheaf, Officium re-gestalt).
* Tackle bindings (tackle sheaf) supply per-scope discipline context at runtime, replacing
  repo-grain `@`-includes with file-grain binding.

Session tiers: the dispatch verb carries the tier — lunge at high-effort frontier tier
(groom/school-grade planning), saddle at the work tier per pace character, orchestration
(future breeze) at haiku.
Programmatic session dispatch for personal tooling is documented product surface (headless mode,
Agent SDK, `--model`); recording that standing determination here is what keeps it from being
re-litigated from memory.
