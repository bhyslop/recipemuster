//axheb_entity
{jjsz_gazette}

A two-level map from slug to lede to content, constructed with a fixed vocabulary of allowed slugs.
Used for multiline data exchange between {jjdx_tool_s} and Claude.
Implementation module: `jjrz_gazette.rs` (prefix `jjrz_`).

== Invariants

*Freeze-on-disclosure.*
A gazette has two lifecycle phases: *building* (unfrozen, accepts mutations) and *frozen* (immutable, accepts queries).
A parse-constructed gazette is born frozen.
A build-constructed gazette freezes permanently when any method discloses notice map contents to the caller.
Internal invariant enforcement during mutation does not constitute disclosure.
Mutating a frozen gazette is fatal.

*Ordering is opt-in, and only within a slug.*
Across slugs there is no ordering semantic at all: a caller must not depend on the sequence in which `jjrz_query_all` or `jjrz_emit` walk the slug space.
Within one slug the gazette keeps two readings and the caller names which it wants.
`jjrz_query_by_slug` discloses entries keyed by lede — that the keying happens to yield lexical order is an artifact of the map, never contract, and a caller must not lean on it.
`jjrz_query_by_slug_ordered` discloses them in the order they were authored (file order on parse, insertion order on build), and *that* order is contract: the gazette records the authored sequence alongside the keyed map precisely so this reading survives.
The live order-significant use-site is the {jjsoprd_revise_docket} batch, where {jjezs_slate} notice order is pace order (Mixed Batch Input, below).
A command whose contract defines file-order semantics reads through the ordered query and says so; every other caller gets no order guarantee.

*Unique keys.*
Each notice is uniquely keyed by its (slug, lede) pair.
Inserting a duplicate key is fatal with a diagnostic explaining the conflict.
Lede may be absent for slug types that identify a single item; at most one lede-absent entry per slug.

== Slug Enumeration

{jjdt_slug} defines the set of valid category identifiers for gazette notices.
Each use site constructs a gazette with a vocabulary that is a subset of these values.

[[jjezs_slate]]

Lede: {jjdt_silks} of the new pace (no {jjdt_coronet} exists yet).
Content: docket text for the initial tack.

[[jjezs_halter]]
//axl_voices axt_enum_value axd_string
{jjezs_halter}::

Direction: input.

Direction: output.
Lede: {jjdt_firemark} of the heat whose history was read.
Content: every steeplechase entry, newest first, one stanza each — a header line carrying timestamp, commit, action code, and affiliation, then the subject untruncated beneath it.
The inline result clips each subject to a bounded line, so this notice is the only reading of a subject in full; it is always populated, empty body when the heat has no entries yet.
Serialized as `jjezs_steeplechase` in gazette markdown.

== File Exchange Protocol

Gazette file exchange uses two directional files within the {jjdxo_exchange}:

* `gazette_in.md` — Agent to server.
Functionally a parameter: the agent writes this file before calling a setter command.
* `gazette_out.md` — Server to agent.
Functionally a return value: the server writes this file after a getter command completes.

=== Universal Entry Rule

Every {jjdx_tool_s} call, unconditionally, before dispatch:

. Read `gazette_in.md` into memory (if present)
. Delete `gazette_in.md`
. Delete `gazette_out.md`

Gazette content has single-MCP-call lifetime.
After the entry rule, the {jjdxo_exchange} is always clean of gazette files.

Setter commands (enroll, redocket, curry) use the in-memory content from step 1.
If content was absent, the setter fails with a diagnostic.
Getter commands (orient, show, paddock, log) write fresh `gazette_out.md` after processing.
The paddock reader additionally rejects loud when step 1 found staged content —
a read call carries nothing to consume, and executing one anyway is how a pre-staged notice once became a silent write.
Non-gazette commands silently discard any in-memory content read in step 1.

=== Bidirectional Slugs

The {jjezs_paddock} slug is marked bidirectional.
It appears in `gazette_in.md` as write input ({jjdo_curry}, or a {jjsoprd_revise_docket} batch)
and in `gazette_out.md` as read output ({jjdo_paddock}, {jjdo_orient}, {jjdo_show}).
Each operation is unidirectional — no single operation touches both files.

=== Mixed Batch Input

{jjsoprd_revise_docket} accepts a heterogeneous single-heat batch:
one `gazette_in.md` carrying at most one {jjezs_paddock} notice,
zero or more {jjezs_reslate} notices,
and zero or more {jjezs_slate} notices,
applied in a single {jjsodp_command_lifecycle} pass and landing as one commit.
This section is the batch's contract home; the command sheaves cross-reference it rather than restating it.

*One heat, guarded.*
Every notice in a batch must name the same {jjdhr_heat}.
The batch {jjdt_firemark} is resolved from the {jjezs_paddock} lede and from the parent {jjdt_firemark} of each {jjezs_reslate} notice's {jjdt_coronet};
disagreement among them is a loud rejection.
{jjezs_slate} notices carry {jjdt_silks}, not an identity, so they inherit the resolved {jjdhr_heat} —
which means a slate-only batch has no heat anchor at all and is rejected, naming {jjdo_enroll} as the path for a bare slate.
The guard closes a legacy mass-reslate defect that keyed the whole run off the first {jjdt_coronet} and misattributed the rest.

*Slate order is file order.*
{jjezs_slate} notices apply in the order they appear in the file — notice order is pace order — read through `jjrz_query_by_slug_ordered`.
Positioning ({jjda_before}, {jjda_after}, {jjda_first}) aims the whole *run*, not one notice:
the first {jjezs_slate} takes the cursor and each later one follows its predecessor,
so the batch folds in as one unbroken sequence at the position aimed at.
Absent positioning, the run appends to the end of {jjdhm_order}, still in file order.
{jjezs_reslate} notices are order-free — each names its own {jjdt_coronet} — and the lone {jjezs_paddock} notice is positionless.

== Wire Format

Gazette markdown uses `#`-headers to delimit notices:

* `#` at line start marks a notice boundary
* First token after `#` (whitespace-separated) is the slug
* Remainder of the header line is the lede (may be empty)
* Content runs from the line after the header to the next `#`-header or EOF
* Leading and trailing blank lines in content are trimmed

Round-trip guarantee: for any gazette `g` constructed by parse with vocabulary `v`, `parse(v, emit(g))` produces a gazette with identical notice map contents.

== Methods

//axhems_scoped_method
=== jjrz_parse

Construct a frozen gazette from raw markdown per the wire format.
All validation errors are collected and returned as a diagnostic list rather than failing on the first.
Each diagnostic is self-contained and LLM-legible: unknown slug (with near-match suggestion if applicable), malformed header (with line number), or structural violation.

//axhems_scoped_method
=== jjrz_query_by_slug

Retrieve all notices with the given slug, keyed by lede.
The resulting sequence carries no contract — see the ordering invariant.
Discloses notice map contents.

//axhempm_motif_parameter axd_required axt_string
Slug.

//axhemom_motif_output axd_repeated axt_object
Set of (lede, content) pairs.
Empty set if no notices match.

//axhems_scoped_method
=== jjrz_query_by_slug_ordered

Retrieve all notices with the given slug in authored order — file order on a parsed gazette, insertion order on a built one.
The order is contract, and this is the only method that supplies it: a caller whose semantics depend on the sequence the agent wrote reads here, never through `jjrz_query_by_slug`.
The {jjsoprd_revise_docket} batch's slate run is the live consumer.
Discloses notice map contents.

//axhempm_motif_parameter axd_required axt_string
Slug.

//axhemom_motif_output axd_repeated axt_object
Sequence of (lede, content) pairs in authored order.
Empty sequence if no notices match.

//axhems_scoped_method
=== jjrz_query_all

Retrieve all notices.
Discloses notice map contents.

//axhemom_motif_output axd_repeated axt_object
Set of (slug, lede, content) triples.

//axhems_scoped_method
=== jjrz_emit

Produce formatted markdown per the wire format.
Discloses notice map contents.

Formatted markdown string.
