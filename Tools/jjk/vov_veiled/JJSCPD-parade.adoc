Display
{jjdhr_heat}
or
{jjdpr_pace}
information in human-readable text format.
Output varies by target type.

{jjds_arguments}

// ⟦axd_optional axd_defaulted⟧
* {jjda_file}

// ⟦axd_optional axd_defaulted⟧
* TARGET (positional) —
{jjdt_firemark}
(heat view) or
{jjdt_coronet}
(pace view).
Target type detected by length: 2 base64 chars = heat, 5 base64 chars = pace.
If omitted, uses first
{jjdhe_racing}
heat.
Returns error if no racing heats exist.

// ⟦axd_optional⟧
* `--detail` — Show paddock and full pace dockets (heat mode only)

// ⟦axd_optional⟧
* `--remaining` — Show only remaining paces (exclude
{jjdpe_complete}
and
{jjdpe_abandoned})


{jjds_stdout} Human-readable text (not JSON).

*Example output — heat list (default):*
----
 No  State       Pace                    Coronet
-------------------------------------------------
  1  abandoned   vvw-workbench-tabtarget AAAAC
  2  complete    install-arch-decision   AAAAI
  3  rough       vvx-push-rename         AAAAJ
----

*Example output — heat list with `--remaining`:*
----
Heat: my-heat (₣AA) [racing]
Progress: 1 complete | 1 abandoned | 1 remaining (1 rough, 0 bridled)

 No  State     Pace                Coronet
-------------------------------------------
  1  rough     vvx-push-rename     AAAAJ

Next: vvx-push-rename (₢AAAAJ) [rough]
----

*Example output — heat list with file-touch bitmap (always shown):*
----
File-touch bitmap (x = pace commit touched file):

  1 B vos-add-routines-section
  2 D jjsa-session-probe-update
  3 * heat-level

BD*
xxx jjrg_gallops.rs, jjrt_types.rs
··x JJSA-GallopsData.adoc
x·· jjro_ops.rs
----

*Example output — commit swim lanes (always shown after file-touch bitmap):*
----
Commit swim lanes (x = commit affiliated with pace):

  1 B vos-add-routines-section
  2 D jjsa-session-probe-update
  3 * heat-level

123456789ab
··xx·xx····  D  4c
xx··x······  B  3c
··········x  *  1c
----

Commit swim lanes show paces × commits: each column is a commit (chronological, dense-encoded 1-9 then a-z then A-Z for 61 max), each row is a pace ordered by first chronological appearance.
`*` row captures heat-level commits.
Row label (terminal char) and commit count suffix appear right of the bitmap.
Default truncation: last 35 commits; if truncated, prefixed with `(showing last 35 of N commits)`.

*Example output — pace commits (always shown after tack history):*
----
Pace commits (x = commit touched file):

  1 abc1234  Fix the main bug
  2 def5678  Update tests
  3 ghi9abc  Refactor helper

123
xx· jjrpd_parade.rs
x·x jjro_ops.rs
·xx jjtpd_parade.rs
----

Pace commits show files × commits for a single pace: each column is a commit within the pace (chronological), each row groups files by identical touch pattern.
Legend shows commit SHA and subject line.
Administrative files (`.claude/jjm/*`) excluded from all bitmap outputs.

*Example output — heat full:*
----
Heat: my-heat (₣AA)
Status: racing
Created: 260119

## Paddock

[paddock content...]

## Paces

### pace-silks (₢AAAAJ) [rough]

[full tack text...]

Warrant: [if bridled]
----

*Example output — pace detail (single tack):*
----
Pace: vvx-push-rename (₢AAAAJ)
Heat: ₣AA

[0] rough (abc123f)
    Silks: vvx-push-rename

    [full tack text...]
----

*Example output — pace detail (multiple tacks, showing history):*
----
Pace: feature-name (₢AAAAK)
Heat: ₣AA

[0] rough (789abc0)
    Silks: initial-name

    Initial docket text here...

[1] bridled (def456a)
    Silks: feature-name
    Warrant: Agent: sonnet...

    Ready for autonomous execution.

[2] complete (abc123f)
    Silks: feature-name

    Final implementation complete.

Tacks are displayed in chronological order (oldest first, index 0).
Each tack shows its index, state, commit hash, silks (which may change over time), optional warrant (if bridled), and the full docket text.
Commit hash displays "(no commit)" for placeholder value `0000000`.

{jjds_exit_uniform} 0 success, non-zero error.

{jjds_behavior}

. Read
{jjdgr_gallops}
from file
. Detect target type by length (strip prefix if present):
.. 2 chars → heat view
.. 5 chars → pace view
.. Otherwise: error "target must be Firemark (2 chars) or Coronet (5 chars)"
. For heat view:
.. Parse as
{jjdt_firemark},
verify heat exists
.. If `--detail` flag:
... Read paddock file content
... Print heat header (silks, firemark, status, created)
... Print paddock section
... For each pace in
{jjdhm_order}
(respecting `--remaining`): print pace header and full tack text
.. Otherwise (default list):
... If `--remaining`: print plain text table with heat header, progress stats, remaining paces in aligned columns, and next up indicator
... If not `--remaining`: for each pace in
{jjdhm_order}:
print numbered line with state, silks, coronet
.. After pace listing, always append file-touch bitmap: query steeplechase entries for heat, run `git diff-tree` per commit to identify touched work files (excluding `.claude/jjm/*`), group by touch pattern, output bitmap with vertical legend; include column-header line of terminal characters above bitmap rows to enable positional decoding without counting
.. After file-touch bitmap, always append commit swim lanes: query steeplechase entries for heat (newest-first from git log), take last 35 (most recent), reverse for chronological column order; rows are paces ordered by first chronological appearance plus `*` row for heat-level commits; columns dense-encoded 1-9 then a-z then A-Z; row label and commit count suffix after bitmap; truncation indicator if >35 commits
. For pace view:
.. Parse as
{jjdt_coronet},
extract parent
{jjdt_firemark}
.. Verify heat and pace exist
.. Query steeplechase entries for parent heat, filter by coronet, run `git diff-tree` per commit, collect unique work file names (excluding `.claude/jjm/*`), display as "Work files:" line (bare filenames, sorted)
.. Print pace header (silks, coronet, state, heat)
.. Print full tack text from `tacks[0]`
.. If bridled, print warrant from `tacks[0]`
.. After tack history, always append pace commits bitmap: query steeplechase entries for heat, filter by coronet, reverse for chronological order; run `git diff-tree` per commit to get work files (excluding `.claude/jjm/*`); group files by identical touch pattern; legend shows commit SHA and subject; column-header line above bitmap rows
