== What it is

Job Jockey's own record — the gallops, and later the captured chat histories — lives in a git
repository *separate from the code being jockeyed and global across stations*: the *studbook*.
The gallops stays the live state-concept; the studbook is the physical vessel that holds it.
A dedicated vvx-driven engine manages the studbook safely; the human never hand-manages its git.

== Why decouple

* *Contamination.* Co-residence makes JJ bookkeeping commits part of the work repo's history and
  advances its HEAD — the recorded symptom is the pensum-seed eviction (JJ commits advanced HEAD
  past origin and tripped foray's curia-readiness guard).
  A separate repo dissolves the whole contamination class, not one instance.
* *Merge relief.* Across synced clones, every gallops change is today a 3-way merge of canonical
  JSON; a single global record edited under a lock has no cross-clone merge at all.
* *Identity collision.* Distributed next-free coronet allocation across divergent clones
  deterministically double-mints (the `₢BfAA3` incident,
  `Memos/memo-20260702-jjk-gallops-coronet-collision-incident.md`).
  One studbook behind one lock means one seed — the collision class dies at the root.
* *Vanilla hippodromes.* State out of the work repo is what lets a work repo carry no JJ inserts
  at all (see The peer ring below).

== The peer ring (garth)

All JJ-managed trees are peers in one parent directory (candidate word: *garth*):

* `../job-jockey-repo` — the kit (engine, specs); found by peer-ring convention, no per-repo pointer.
* `../studbook` — this record repo.
* Work clones (*hippodromes*) and their transient worktree *billets* (farrier sheaf).
* The BUK station family (`../station-files/burs.env`, `../logs-buk`, `../temp-buk`,
  `../output-buk`) already lives at this level, so every peer tree shares station geography
  with zero setup.

== Hippodrome — the vanilla work clone

A *hippodrome* is a peer repo clone JJ operates from.
It carries **no JJ inserts**: no committed dossier, no `.claude/jjm` state, no JJK includes —
a vanilla external project is a working hippodrome the moment its pedigree exists.
Identity is the directory name (v1 ruling: no clone-local state at all);
the upstream key is the clone's origin URL.
Several clones of one upstream are several hippodromes — the operator's standing
concurrent-parallel-clone work model is preserved, not replaced.

== Pedigree — the per-upstream record

One *pedigree* per hippodrome upstream type, keyed by origin URL, stored in the studbook:

* the farrier kind (which revision-control driver serves clones of this upstream) and its
  parameters (trunk name, remotes posture) — see the farrier sheaf;
* tackle bindings (file scope → governing discipline) as sibling entries — see the tackle sheaf;
* reserved, parked: a per-upstream edit-scope wildcard list, if billets-per-pace ever prove
  insufficient for collision avoidance.

JJ launched anywhere derives origin URL → pedigree → everything it needs.
A new upstream needs a founding pedigree entry before JJ serves it — a deliberate ceremony.

== The journal ceremony

Every studbook write brackets identically (the *journal ceremony*), and the ordering is
*durable-first*: the work artifact commits before the journal that references it.

. *Hippodrome commit* (lock-free): explicit file list, on the current branch or billet branch.
  The pin now exists.
  (Whether this step also pushes the hippodrome is an open fork.)
. *Lock the studbook*: `fetch` (sync lock state) → `ref_cas_create` (the CAS: atomically push
  the stamped lock ref, or be rejected) → `ref_read` to confirm the held stamp is ours.
  Lock refs live in a dedicated namespace outside `refs/heads/*`; the stamp carries officium,
  station, acquire-time, and operation, which is what makes break-on-error safe.
. *Advance*: fast-forward the local studbook to remote tip — always clean under the lock;
  loud failure if not.
. *Mutate and commit*: apply the gallops change plus the pin naming the hippodrome and its
  commit; `commit` with explicit files.
. *Push content*: `--atomic`, binding the content update to a lease on our own lock ref —
  if the lock was broken under us, the whole push fails.
  No corruption, no service but git.
. *Unlock*: `ref_cas_delete`, best-effort; a die here leaves a stamped stale lock for the
  break sequence.

Breaking a stale lock is *never a method on the holder's guard* — it is its own sequence
(lease-guarded delete against the observed stamp, never blind), specified in its own right.
There is no personified break entity.
An operator-deliberate break-all toothing exists in the farrier sheaf's op census.

An orphaned hippodrome commit (committed, never journaled) is recoverable;
a journal entry naming a nonexistent SHA is the worse failure — hence durable-first.

== Pin — how JJ commits know hippodrome commits

Every journaled entry carries a *pin*: a member→SHA manifest (one-element for a single repo,
generalizing unchanged to submodule/multi-repo constellations), annotated with branch and a
dirty flag.
The reference is deliberately one-way — studbook points at hippodrome, never the reverse;
the only hippodrome-side mark is the session-id commit trailer, a denormalized cache justified
by query convenience, never by recoverability.

*Rebase soft spot (known, accepted for now):* a rebase after journaling rewrites SHAs and
dangles the pin — degraded provenance, not corruption; nothing downstream builds on it.
Convergence discipline already prefers merge over rebase (the collision memo), which preserves
pins; billet branches never rebase (freshen is merge-based, farrier sheaf), closing the gap
structurally where billets are used.
An optional `pin_verify` op turns a dangling pin from a silent lie into a flagged one.

== Plans as commits (the gazette convergence)

With the studbook, every gallops mutation *is* a git commit — so ₣Ah's cinch ("warrants in
gallops, not git") and the long-held intent to keep breeze plans reviewable as commits become
the same act.
The commit message may carry the plan in gazette notice format as a review surface for
`git log`; the JSON in the gallops is truth, the message a denormalized presentation.

== Store shape

* Git stays source-of-truth and journal — additive history, commit-as-intent, `jjx_log`
  reconstruction, free immutability.
* The studbook and mews repos are *linear — they never branch* — and single-writer-under-lock,
  so a monotonic ordinal per repo is clean and authoritative.
  The deferred insignia sub-mint (glyph-prefixed revision aliases; the ordinal a denormalized
  label, the SHA the identity) stands as recorded in the 260630 memo.
* The lock supersedes the original sharding pressure (no cross-clone merge exists at all);
  per-heat sharding remains open only as a file-layout choice, not a concurrency need.
* Reads take no lock and tolerate slight staleness; consumers reach the gallops through the
  engine's read path, never raw JSON.

== Scope at birth

First tenant: the gallops only.
Captured chat histories and action-choices are the *intended second tenant*, deferred.

== Chat↔commit correlation (captured 260620, deferred with the second tenant)

Once chats live in the studbook, a commit ties to the chat turn that produced it:
commit → session → transcript → turn → model and full context.
The transcript is the normalized source of truth; any field stamped on a commit is a
denormalized cache.
Two small enabling changes close the loop — write the session id into every work commit, and
label the SHA the tool already emits.

== Officium re-gestalt

With one regional studbook, the officium becomes the identity that names *where* work happens:
station + hippodrome (or billet) + session.
`jjx_open` discovers its billet (or trunk hippodrome) via the farrier's `identify` and
validates it against the work being mounted.
Officium exchange (gazettes) must leave the work repo (vanilla-hippodrome premise);
proposed home: a working area beside the studbook clone in the garth — open fork.

== MVP framing

One MVP, staging adjustable before heats are cut: the studbook transform (this sheaf), the
farrier and its dispatch layer (farrier sheaf), and MVP tackle (tackle sheaf) land together.
Phasing within the MVP is cinched below: build under current JJ, rehearse, then cut over.

== Founding and cutover (cinched 260704)

* *Remote: a GitHub private repo.*
  Credentials already exist on every station, push-CAS semantics are proven, and custom ref
  namespaces are accepted for the lock refs.
  No new confidentiality exposure — the gallops is already committed to GitHub-hosted work
  repos today.
* *Scope: one studbook, multiproject and cross-workstation.*
  Every session on every station synchronizes through the one record.
* *Separate at birth.*
  The studbook and the JJ kit repo are distinct repos founded in the same hour.
  Co-residence would recreate the contamination class in JJ's own house and force one repo to
  be both a locked linear record and a branching sourcebase; splitting later means migrating
  live locked state — exactly the surgery this design avoids.
* *Build under current JJ, convert last.*
  The new world is built as heats/paces in today's in-repo gallops — JJ jockeys its own
  successor — as one binary with parallel code paths: the old path frozen and authoritative,
  the new surface inert until enabled (the reprieve delivery shape).
* *Rehearsal before conversion.*
  The new world proves itself against a *scratch* studbook first: two-station lock contention,
  stale-lock break, atomic-push lease failure, saddle round-trip.
  The rehearsal checklist is the seam — final pace of the build phase, entry gate of the
  conversion phase.
* *The conversion is a full cutover ceremony* — sole-operator, one afternoon: quiesce sessions,
  final-sync clones, founding import, switch every install together.
  The old in-repo store is replaced by a tombstone marker that old binaries fail loud on,
  never silently resurrect (`JJr_` rivet treatment: deliberately temporary, demolition
  condition attached).
  The same ceremony *de-inserts* JJK from consumer repos — CLAUDE.md includes, `.claude/jjm`,
  MCP config — completing the vanilla-hippodrome premise; context provisioning moves to saddle
  (farrier sheaf).
* *Founding import carries live state only* — racing and stabled heats, paddocks, dockets;
  retired trophies may ride along as plain files.
  Pre-cutover history does not migrate, and the new engine builds *no* old-side read path —
  the fossils stay in the work repos as human-readable provenance; `jjx_log` starts fresh at
  the founding import.
* *Multiproject import discipline.*
  Independent installs minted firemarks from independent seeds, so cross-project imports can
  collide.
  The founding import takes one lineage; the importer detects and refuses collisions; a later
  second-project import is its own designed ceremony, with the tombstone-transfer maneuver as
  template.
