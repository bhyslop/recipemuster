= Studbook

The regional record repo: the {jjdb_blotter_p} first instance, and the physical vessel of the
{jjdgr_gallops}.
Store mechanics — the lock ref and its {jjdb_guidon}, the write bracket, the break sequence —
are the {jjdb_blotter_p} contract and the journal sheaf's sequences;
this sheaf homes the tenant: what the {jjdb_studbook} holds, and how its entries know the work
they record.

== The entity

[[jjdb_studbook]]
//axo_entity
{jjdb_studbook}::
The regional record repo — one {jjdb_blotter} serving every station and every jockeyed
project.
The {jjdgr_gallops} stays the live state-concept;
the {jjdb_studbook} is the physical vessel that holds it.
As a {jjdb_blotter} it is linear, single-writer-under-lock, and engine-driven:
every write passes through the {jjdb_journal} ({jjdk_sole_door}),
reads take no lock ({jjdk_lockless_reads}),
and its own coordinates are engine-known, never pedigree-resolved.

== Pedigree

[[jjdb_pedigree]]
{jjdb_pedigree}::
The per-sire record: one {jjdb_pedigree} per *sire* — the registered external body of work
JJ serves, a transport-independent identity (a git clone today, perhaps a tarball someday
under another driver kind) — stored in the {jjdb_studbook} — the {jjdb_studbook} is a
register of {jjdb_pedigree_s}.
The {jjdb_pedigree} keys on the sire's minted immutable identity;
an origin URL is a kind-owned mutable address attribute, never the key.
A new sire needs a founding {jjdb_pedigree} entry before JJ serves it — a deliberate
ceremony;
the derivation chain that consumes the entry is world model, homed in the cosmology sheaf.

Each {jjdb_pedigree} carries:

* the {jjdf_farrier} kind — which revision-control driver serves clones of this sire —
  and its parameters (trunk name, remotes posture);
* the sire's tackle set — its filesets and the governance each carries, and in time their
  declared interrelations — as a sibling entry ({jjdl_tackle}, tackle sheaf).
  The {jjdl_bounds} seat in the tackle table itself, never here: coverage is tackle
  machinery, and a second home would be a second truth;
* reserved, parked: a per-sire edit-scope wildcard list, if billets-per-pace ever prove
  insufficient for collision avoidance.

== Counterfoil

[[jjdb_counterfoil]]
{jjdb_counterfoil}::
The retained stub every journaled entry carries, naming the issued work:
a member→SHA manifest — one-element for a single repo, generalizing unchanged to submodule
and multi-repo constellations — annotated with branch and a dirty flag.
The reference is deliberately one-way:
the {jjdb_studbook} points at the {jjdw_hippodrome}, never the reverse.
The only hippodrome-side mark is the session-id commit trailer,
a denormalized cache justified by query convenience, never by recoverability.

*Rebase soft spot (known, accepted):*
a rebase after journaling rewrites SHAs and dangles the {jjdb_counterfoil} —
degraded provenance, not corruption; nothing downstream builds on it.
Convergence discipline already prefers merge over rebase, which preserves
{jjdb_counterfoil_s};
{jjdd_billet} branches never rebase ({jjdd_refit} composes the merge-based `enfold`, a
{jjdf_billet} op), closing the gap structurally where {jjdd_billet_s} are used.
An optional `counterfoil_verify` op ({jjdf_farrier} op census, deferred past MVP) turns a
dangling {jjdb_counterfoil} from a silent lie into a flagged one.

== What never enters the record

The {jjdb_studbook} is a regional record; a worktree path is a station-local fact.
*No worktree path is ever written into the {jjdb_studbook}* —
issued work is named by member, SHA, and branch, never by filesystem location.
The proposition classifies as a rivet;
its opaque tail mints at implementation, citing this section.

== Journal entries as commits

Every {jjdgr_gallops} mutation is one {jjdb_journal} bracket, so every mutation *is* a git
commit.
The commit message may carry the plan in gazette notice format as a review surface for
`git log`;
the JSON in the {jjdgr_gallops} is truth, the message a denormalized presentation.

== Scope at birth

First tenant: the {jjdgr_gallops} only.
Captured chat histories and action-choices are the *intended second tenant*, deferred;
their contract — including the chat↔commit correlation — stays aspirant with them
(`JJSAS-state-repo.adoc`).

== Officium exchange and record

The {jjdb_studbook}'s local clone hosts more than its committed tenants above: a fixed,
untracked subdir (one per officium) co-locates two things that never enter the record until
they graduate — {jjdxo_exchange} (gazettes: wire format, single-call lifetime, forever
untracked) and the officium's *record* half (identity metadata — station,
hippodrome-or-billet, session, per {jjdf_identify}'s officium-open composition,
`JJSVF-farrier.adoc` "Toothing: officium open" — and counterfoils, eventually the captured
session transcripts of this section's second tenant).
Co-locate now, commit later: graduating the record half to committed history is a
gitignore-line change, never a relocation — the studbook already knows this one location on
every station.
Because graduation is that small, it forces no schema review of its own: what may enter the
record binds here, before the record is ever written.
The {jjdw_hippodrome}-or-{jjdd_billet} member enters as the seat's role reading alone — a
`Partition` carries its primary's root, and that root is a station-local worktree path, barred
by "What never enters the record" above (the rivet whose tail mints when the record write
lands).
