//axhob_operation
{bust_garrison_bash}

NOTE: {busn_garrison} ceremony with native bash as the workload shell,
for Linux and macOS nodes.
Workload-only: garrison destroys any prior workload account named
{bujb_workload_user} via the letter-agnostic
`zbujb_obliterate_workload` helper in {bujb_jurisdiction} — the
single greppable destroy verb. On Linux the helper runs `userdel -r`
and purges `/home/<user>`; on macOS it runs `sysadminctl -deleteUser`
and purges `/Users/<user>`. Each step is idempotent on absent state.
Garrison never reads or writes any `sshd_config` and never modifies
admin authorized_keys.

//axhopt_typed_parameter
{burp_investiture}

//axhoq_precondition
A {burp_regime} profile exists at the given {burp_investiture} and its
referenced {burn_regime} profile is reachable.

//axhoq_precondition
sshd is reachable on the node and {burp_privileged_user} authenticates
by key alone using {burp_privileged_key_file}. On Linux/Mac admin
trust is established via {busn_caparison}, which wraps the
operator-managed `ssh-copy-id` flow rather than performing autonomous
SSH-trust hardening (only Windows OpenSSH has the autonomous
phase-1/phase-2 ceremony).

//axhoq_precondition
{bust_invigilate_macos} or {bust_invigilate_linux} passes for this
{burp_investiture} (selected by the BURN-targeted OS): admin SSH
key trust is in place and host posture (Remote Login + pmset +
Tailscale launchd on macOS; sshd + sleep-target mask + tailscaled on
Linux) is correct. Garrison runs invigilate as its first action so a
missing or stale admin posture surfaces with a fact-named diagnostic
rather than as an opaque downstream failure.

//axhoq_precondition
{burp_workload_key_file} is unencrypted and exists on the operator's
station.

//axhos_step
. *Assert admin posture*: invoke `{bust_invigilate_macos}` or
`{bust_invigilate_linux}` for this {burp_investiture} (selected by
the BURN-targeted OS). Failure surfaces a fact-named diagnostic
({bust_handbook_macos}/{bust_handbook_linux} pointer for
operator-managed facts;
{bust_caparison_macos}/{bust_caparison_linux} pointer for
caparison-managed facts) and aborts garrison before any destructive
workload work.

//axhos_step
. *Open admin SSH*: connect as {burp_privileged_user} using
{burp_privileged_key_file}; key-auth only. Failure here surfaces a
missing or stale admin trust.

//axhos_step
. *Obliterate workload*: invoke `zbujb_obliterate_workload`. On Linux
the helper runs `sudo -n userdel -r` and `sudo -n rm -rf /home/<user>`;
on macOS it runs `sudo -n sysadminctl -deleteUser` and
`sudo -n rm -rf /Users/<user>`. Idempotent on absent state.

//axhos_step
. *Create workload*: provision {bujb_workload_user} as an unprivileged,
ssh-only account.

//axhos_step
. *Place workload trust*: write the workload pubkey
(derived via `ssh-keygen -y`) to the workload's authorized_keys with
the shell-letter `b` `command=` directive below — locked spec content,
mirrored at `BUJB_command_b` in {bujb_jurisdiction}:
+
[listing]
----
command="/bin/bash -lc \"$SSH_ORIGINAL_COMMAND\"",no-port-forwarding,no-X11-forwarding,no-agent-forwarding
----

//axhos_step
. *Plant workload key*: copy {burp_workload_key_file} to the workload
account at the shell-letter `b` destination path hardcoded in
{bujb_jurisdiction}.

//axhos_step
. *Validate round-trip*: SSH as workload, run a no-op command, verify
exit 0.

//axhog_guarantee
On success the node has exactly one workload account named
{bujb_workload_user}, ssh-only, with both inbound (from operator) and
outbound (to GitHub or equivalent) key trust in place.

//axhoc_completion
Exit 0 when the round-trip validation succeeds.
Non-zero on invigilate assertion failure, admin SSH failure,
account-management failure, key-write failure, or workload validation
failure.
