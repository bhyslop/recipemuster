=== Windows Node Access

Platform-specific ceremony and contracts for Windows node access.

==== Status

Stub.
Elaborate content to be drafted by subsequent paces — Windows is the
canonical worked example for the Remote Node Access ceremony, requiring
detailed coverage of OpenSSH server provisioning, the
`administrators_authorized_keys` special case, ACL fixes via `icacls`,
and the temp-password bootstrap handshake.

==== Privilege Contract

The {burn_user} of a Windows privileged BURN must be a member of the local
Administrators group.
Rationale: Windows OpenSSH consults `C:\ProgramData\ssh\administrators_authorized_keys`
for users in the Administrators group, ignoring the per-user
`~/.ssh/authorized_keys` for those users.
A non-admin path would require parallel implementation with no return for
the supported use cases.

Asserted at every ceremony participant before any modification.

==== Platform Letters

* `c` — Cygwin (`buw-npgc`, `buw-nwcc`)
* `w` — WSL (`buw-npgw`, `buw-nwcw`)
* `p` — PowerShell (`buw-npgp`, `buw-nwcp`)

==== Out of Band

The console phase of the Windows ceremony is performed at the Windows
physical keyboard (or RDP) by the operator and is described in the BUK
handbook surface, not in this spec.
