//axhob_operation
{bust_caparison_windows}

NOTE: {busn_caparison} on Windows -- three phases. Phase 1: admin SSH
key trust + `sshd_config` hardening. Phase 2: key-only reconnect.
Phase 3: post-trust admin posture (WSL distribution, powercfg,
Tailscale service). Closes with {bust_invigilate_windows}.

NOTE: Nuclear -- every invocation purges and re-establishes the
caparison-managed posture; no skip-if-present logic.
{bust_handbook_windows} owns operator-manual posture; caparison owns
the idempotent admin-shell remainder.

NOTE: Phase 1's `Restart-Service sshd` terminates its own ssh session
-- the post-restart disconnect is the expected phase boundary, not a
failure. PowerShell-side operations use `Get-Content`, `Set-Service`,
`Start-Service`, `Restart-Service`, `wsl.exe --import`; bash-side
BCG-compliant matching extracts effective `sshd_config` directives
from the raw bytes.

The {busn_caparison} operation on Windows:

//axhog_guarantee
On success:

* Admin SSH is key-only with admin pubkey installed in
  `administrators_authorized_keys` under correct ACLs.
* `sshd_config` carries `PubkeyAuthentication yes`,
  `PasswordAuthentication no`, `PermitEmptyPasswords no`.
* Post-restart admin session authenticates by key alone.
* `rbtww-main` WSL distribution is registered under the admin user.
* `powercfg` sleep and hibernate are disabled.
* Tailscale service is Automatic and running.
* {bust_invigilate_windows} confirms every posture fact end-to-end.

//axhoc_completion
Exit 0 when the post-completion {bust_invigilate_windows} passes.
Non-zero on any phase-1 op failure (admin SSH, pubkey, ACL,
`sshd_config`, `sshd -t`), phase-2 reconnect failure, any phase-3 op
failure (`wsl --import`, `powercfg`, Tailscale), or post-completion
invigilate failure.
