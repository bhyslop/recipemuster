//axhob_operation
{bust_conscript_wsl}

NOTE: Creates an ephemeral workload OS user on a Windows node with a
WSL shell, and provisions a corresponding workload {burn_regime}
profile locally.
Uses a named privileged BURN as authority for the remote-user
creation.
The WSL shell choice is independent of the privileged BURN's shell.

//axhoq_precondition
A privileged-tier {burn_regime} profile exists locally and is
{bust_knock}-responsive against the target node.

//axhoq_precondition
A WSL distribution is registered (or registrable) for the new workload
account on the target node.

//axhos_step
. *Authenticate via privileged BURN*: open an authenticated session to
the target node using the privileged profile

//axhos_step
. *Create workload user*: provision a fresh OS user, register the WSL
distribution under that user, and capture the temporary credential or
pubkey trust as defined by the platform ceremony

//axhos_step
. *Write workload profile*: persist a workload {burn_regime} profile
locally with `{burn_tier} = workload`, {burn_command} naming the WSL
launcher, and a reference to the privileged authority used

//axhos_step
. *Probe new account*: confirm passwordless SSH to the workload account
succeeds via {bust_knock}

//axhog_guarantee
A new workload OS user exists on the remote node with a registered WSL
distribution, a corresponding workload {burn_regime} profile is
persisted locally, and the new account is reachable over passwordless
SSH.

//axhoc_completion
Exit 0 when remote-user creation, WSL registration, local profile write,
and verification probe all succeed.
Non-zero on any of: privileged-BURN refusal, remote-user creation
failure, WSL registration failure, local profile write failure, or
final probe failure.
