//axhob_operation
{bust_garrison_wsl}

NOTE: {busn_garrison} ceremony with WSL bash as the workload shell,
for Windows nodes reached via the WSL distribution `rbtww-main`
(hardcoded in {bujb_jurisdiction}).
Workload-only: garrison destroys any prior workload account named
{burc_workload_user}
and recreates it from a clean slate.
Garrison never reads or writes any `sshd_config` and never modifies
admin authorized_keys; admin trust is established beforehand by
{busn_fenestrate}
on the Windows OpenSSH server, while sshd hardening inside the WSL
distribution remains operator-manual.

//axhopt_typed_parameter
{burp_investiture}

//axhoq_precondition
A {burp_regime} profile exists at the given {burp_investiture} and its
referenced {burn_regime} profile is reachable.

//axhoq_precondition
The default WSL distribution on the node is `rbtww-main` and its sshd
is installed; mismatch fails fast with a clear error.

//axhoq_precondition
{busn_fenestrate} has run for this {burp_investiture} so that
{burp_privileged_user} can authenticate by key alone using
{burp_privileged_key_file} into the Windows OpenSSH host (used as the
admin entry point); the WSL session is then opened from that
foothold.

//axhoq_precondition
{burp_workload_key_file} is unencrypted and exists on the operator's
station.

//axhos_step
. *Open admin SSH*: connect as {burp_privileged_user} using
{burp_privileged_key_file}; key-auth only into the WSL distribution.
Failure here surfaces a missing or stale admin trust.

//axhos_step
. *Destroy workload*: remove any existing {burc_workload_user} account
inside the WSL distribution via `userdel -r`.

//axhos_step
. *Create workload*: provision {burc_workload_user} as an unprivileged,
ssh-only Linux account inside `rbtww-main`.

//axhos_step
. *Place workload trust*: write the workload pubkey
(derived via `ssh-keygen -y`) to the workload's authorized_keys with
the shell-letter `w` `command=` directive from {bujb_jurisdiction}.

//axhos_step
. *Plant workload key*: copy {burp_workload_key_file} to the workload
account at the shell-letter `w` destination path hardcoded in
{bujb_jurisdiction}.

//axhos_step
. *Validate round-trip*: SSH as workload, run a no-op command, verify
exit 0.

//axhog_guarantee
On success the node has exactly one workload account named
{burc_workload_user} reachable via the WSL distribution `rbtww-main`.

//axhoc_completion
Exit 0 when the round-trip validation succeeds.
Non-zero on WSL distribution mismatch, admin SSH failure, account-
management failure, key-write failure, or workload validation failure.
