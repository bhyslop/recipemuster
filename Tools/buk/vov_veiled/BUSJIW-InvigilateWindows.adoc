//axhob_operation
{bust_invigilate_windows}

NOTE: {busn_invigilate} on Windows -- read-only host posture
verification. Asserts each posture fact matches expected; no
mutations.

NOTE: Invocation: {bust_caparison_windows} post-completion check;
{bust_garrison_cygwin} and {bust_garrison_wsl} precondition.

NOTE: Failure pointers discriminate operator-managed
({bust_handbook_windows}) from caparison-managed
({bust_caparison_windows}). PowerShell reads via
`Get-ItemPropertyValue`, `Get-Service`, `powercfg /a`, `wsl.exe --list
--quiet`; bash-side BCG-compliant matching extracts each fact.

The {busn_invigilate} operation on Windows:

//axhopt_typed_parameter
{burp_investiture}

//axhoq_precondition
A {burp_regime} profile exists at the given {burp_investiture} and its
{burn_regime} targets a Windows OpenSSH node reachable as
{burp_privileged_user} under key-only auth using
{burp_privileged_key_file}. Admin SSH failure surfaces as the first
posture fact -- invigilate presupposes trust, does not establish it.

//axhos_step
. *Open admin SSH session*:
.. {busc_call}
ssh as {burp_privileged_user} with {burp_privileged_key_file}; key-auth only
.. {busc_store}
`«ADMIN_SESSION»`
.. {busc_fatal}
on connection failure -- {bust_caparison_windows} scope

//axhos_step
. *Assert operator-managed registry keys*:
.. For each {bust_handbook_windows}-established registry value
(canonical: `DevicePasswordLessBuildVersion` under
`HKLM\Software\Microsoft\Windows NT\CurrentVersion\PasswordLess\Device`):
... {busc_call}
PowerShell `Get-ItemPropertyValue` over `«ADMIN_SESSION»`
... {busc_store}
`«REG_VALUE»`
... {busc_require}
`«REG_VALUE»` matches expected
... {busc_fatal}
on mismatch -- {bust_handbook_windows} scope

//axhos_step
. *Assert AoAc override*:
.. {busc_call}
PowerShell `Get-ItemPropertyValue` for
`HKLM\System\CurrentControlSet\Control\Power\PlatformAoAcOverride`
.. {busc_store}
`«REG_AOAC»`
.. {busc_require}
`«REG_AOAC»` equals `0`
.. {busc_fatal}
on mismatch -- {bust_handbook_windows} scope (Modern Standby override)

//axhos_step
. *Assert Tailscale service registered*:
.. {busc_call}
PowerShell `Get-Service Tailscale`
.. {busc_store}
`«SVC_TAILSCALE»`
.. {busc_require}
`«SVC_TAILSCALE»` is non-empty
.. {busc_fatal}
on absence -- {bust_handbook_windows} scope (install + Run-Unattended + first auth)

//axhos_step
. *Assert Tailscale service StartupType*:
.. {busc_call}
PowerShell `Get-Service Tailscale | Select-Object StartType`
.. {busc_store}
`«SVC_STARTTYPE»`
.. {busc_require}
`«SVC_STARTTYPE»` equals `Automatic`
.. {busc_fatal}
on mismatch -- {bust_caparison_windows} scope

//axhos_step
. *Assert standby-timeout policy = 0 (AC and DC)*:
.. {busc_call}
`powercfg /query SCHEME_CURRENT SUB_SLEEP STANDBYIDLE` over `«ADMIN_SESSION»`
.. {busc_store}
`«POWERCFG_STANDBYIDLE»`
.. {busc_require}
`«POWERCFG_STANDBYIDLE»` carries `Current AC Power Setting Index: 0x00000000`
and `Current DC Power Setting Index: 0x00000000`
.. {busc_fatal}
on mismatch -- {bust_caparison_windows} scope. Verifies the policy
{busn_caparison} sets (`powercfg /change standby-timeout-ac/dc 0`),
not the firmware-supplied "available" capability list -- legacy-S3
hardware always reports Standby (S3) as available regardless of
auto-sleep timeout policy.

//axhos_step
. *Assert hibernate disabled*:
.. {busc_call}
`powercfg /a` over `«ADMIN_SESSION»`
.. {busc_store}
`«POWERCFG_REPORT»`
.. {busc_require}
`«POWERCFG_REPORT»` does not list Hibernate in its "available on this
system" section
.. {busc_fatal}
on Hibernate present -- {bust_caparison_windows} scope

//axhos_step
. *Assert admin's rbtww-main WSL distribution registered*:
.. {busc_call}
`wsl.exe --list --quiet` over `«ADMIN_SESSION»`
.. {busc_store}
`«WSL_DISTRO_LIST»`
.. {busc_require}
`«WSL_DISTRO_LIST»` contains `rbtww-main` (canonical:
`BUJB_wsl_distribution` in {bujb_jurisdiction})
.. {busc_fatal}
on absence -- {bust_caparison_windows} scope

//axhog_guarantee
On success:

* Every posture fact matches its expected value.
* `«ADMIN_SESSION»` was used only for read commands.
* No mutation occurred.
* The node is in the shape {bust_caparison_windows} produces and
  {bust_garrison_cygwin} / {bust_garrison_wsl} require.

//axhoc_completion
Exit 0 when every posture fact matches expected. Non-zero on first
mismatch with diagnostic naming the failing fact, expected value,
observed value, and pointer ({bust_handbook_windows} for
operator-managed; {bust_caparison_windows} for caparison-managed).
