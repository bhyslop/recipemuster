//axhob_operation
{bust_invigilate_windows}

NOTE: The {busn_invigilate} ceremony on Windows — read-only host posture
verification. Asserts each posture fact matches its expected value; no
mutations, no remediation.

NOTE: Invocation contexts: {bust_caparison_windows} runs invigilate-windows
as its post-completion check; {bust_garrison_cygwin} and
{bust_garrison_wsl} run it as their precondition assertion before any
destructive workload work.

NOTE: Failure pointers discriminate operator-managed posture (→
{bust_handbook_windows}) from caparison-managed posture (→
{bust_caparison_windows}). PowerShell-side reads use
`Get-ItemPropertyValue`, `Get-Service`, `powercfg /a`, and `wsl.exe
--list --quiet`; bash-side BCG-compliant matching extracts each fact —
PowerShell stays out of the parsing business.

The {busn_invigilate} operation on Windows:

//axhopt_typed_parameter
{burp_investiture}

//axhoq_precondition
A {burp_regime} profile exists at the given {burp_investiture} and its
referenced {burn_regime} profile targets a Windows OpenSSH node
reachable as {burp_privileged_user} under key-only authentication using
{burp_privileged_key_file}. Invigilate does not establish that trust —
it presupposes it; admin SSH failure surfaces as the first posture
fact.

//axhos_step
. *Open admin SSH session*:
.. {busc_call}
ssh as {burp_privileged_user} using {burp_privileged_key_file};
key-auth only (no password fallback)
.. {busc_store}
`«ADMIN_SESSION»` for subsequent posture reads
.. {busc_fatal}
on connection failure with pointer to {bust_caparison_windows} —
admin trust is caparison-managed

//axhos_step
. *Assert operator-managed registry keys*:
.. For each {bust_handbook_windows}-established registry value
(canonical: `DevicePasswordLessBuildVersion` under
`HKLM\Software\Microsoft\Windows\CurrentVersion\PasswordLess\Device`):
.. {busc_call}
PowerShell `Get-ItemPropertyValue` over `«ADMIN_SESSION»`
.. {busc_store}
`«REG_VALUE»` from the response
.. {busc_require}
{busc_use}
`«REG_VALUE»` matches the expected value
.. {busc_fatal}
on mismatch with pointer to {bust_handbook_windows} — these edits
are security-sensitive and operator-manual

//axhos_step
. *Assert AoAc override*:
.. {busc_call}
PowerShell `Get-ItemPropertyValue` for
`HKLM\System\CurrentControlSet\Control\Power\PlatformAoAcOverride`
.. {busc_store}
`«REG_AOAC»` from the response
.. {busc_require}
{busc_use}
`«REG_AOAC»` equals `0`
.. {busc_fatal}
on mismatch with pointer to {bust_handbook_windows} — Modern Standby
silently demotes `powercfg` operations until this override is in
place; it is a registry write requiring elevation, established once
by the operator

//axhos_step
. *Assert Tailscale service registered*:
.. {busc_call}
PowerShell `Get-Service Tailscale`
.. {busc_store}
`«SVC_TAILSCALE»` from the response
.. {busc_require}
{busc_use}
`«SVC_TAILSCALE»` is non-empty
.. {busc_fatal}
on absence with pointer to {bust_handbook_windows} — install +
Run-Unattended toggle + first auth are operator-manual

//axhos_step
. *Assert Tailscale service StartupType*:
.. {busc_call}
PowerShell `Get-Service Tailscale | Select-Object StartType`
.. {busc_store}
`«SVC_STARTTYPE»` from the response
.. {busc_require}
{busc_use}
`«SVC_STARTTYPE»` equals `Automatic`
.. {busc_fatal}
on mismatch with pointer to {bust_caparison_windows} — caparison
sets the StartupType after Tailscale is installed; non-Automatic
indicates caparison either did not run or its `Set-Service`
operation did not take

//axhos_step
. *Assert powercfg sleep/hibernate disabled*:
.. {busc_call}
`powercfg /a` over `«ADMIN_SESSION»`
.. {busc_store}
`«POWERCFG_REPORT»` from stdout
.. {busc_require}
{busc_use}
`«POWERCFG_REPORT»` reports neither standby nor hibernate as
available
.. {busc_fatal}
on mismatch with pointer to {bust_caparison_windows} — disabling
sleep and hibernate is caparison's phase-3 operation; a node still
advertising those states means caparison either did not run or its
`powercfg` calls were demoted to no-ops by a missing AoAc override

//axhos_step
. *Assert admin's rbtww-main WSL distribution registered*:
.. {busc_call}
`wsl.exe --list --quiet` over `«ADMIN_SESSION»`
.. {busc_store}
`«WSL_DISTRO_LIST»` from stdout
.. {busc_require}
{busc_use}
`«WSL_DISTRO_LIST»` contains `rbtww-main` (canonical name hardcoded
as `BUJB_wsl_distribution` in {bujb_jurisdiction})
.. {busc_fatal}
on absence with pointer to {bust_caparison_windows} — caparison's
phase-3 `wsl --import` from the Ubuntu-24.04 seed stages this
distribution; missing registration means caparison either did not
run or its import step failed without surfacing

//axhog_guarantee
On success:

* Every posture fact has been asserted to match its expected value.
* `«ADMIN_SESSION»` was used only for read commands.
* No remote or local mutation occurred.
* The node is in the shape {bust_caparison_windows} produces and
  {bust_garrison_cygwin} / {bust_garrison_wsl} require.

//axhoc_completion
Exit 0 when every posture fact matches its expected value.
Non-zero on the first mismatch, with a diagnostic naming the failing
fact, its expected value, its observed value, and the pointer
({bust_handbook_windows} for operator-managed facts;
{bust_caparison_windows} for caparison-managed facts).
