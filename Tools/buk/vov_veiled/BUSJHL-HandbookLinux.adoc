//axhob_operation
{bust_handbook_linux}

NOTE: Operator-manual scope for Linux -- posture {busn_caparison}
cannot or should not automate. No remote contact, no profile mutation.

NOTE: Steps establish openssh-server install, Tailscale install +
first `tailscale up` auth, a known {burp_privileged_user} password
for the first-run `ssh-copy-id` exchange, and a sudoers NOPASSWD
entry. Mismatches surface from {bust_invigilate_linux} as failure
pointers back to this handbook.

NOTE: Steps missing here cause {bust_caparison_linux} to fail at the
point that depends on them.

The {bust_handbook_linux} procedure:

//axhos_step
. *Install openssh-server*:
.. {bush_show}
common variants -- Debian/Ubuntu:
+
----
sudo apt install openssh-server
----
.. Fedora/RHEL:
+
----
sudo dnf install openssh-server
----
.. {bush_show}
other distributions ship equivalent packages under similar names
(`openssh`, `openssh-sshd`) -- consult the distro's package index
.. {bush_require}
`which sshd` returns a path

//axhos_step
. *Install Tailscale*:
.. cross-distro one-liner:
+
----
curl -fsSL https://tailscale.com/install.sh | sh
----
.. {bush_show}
per-distro packages are also available via the Tailscale repo --
see https://tailscale.com/download for distro-specific instructions
.. {bush_require}
`tailscale version` reports a version

//axhos_step
. *First-run `tailscale up`*:
.. invoke:
+
----
sudo tailscale up
----
.. {bush_show}
the command prints an authentication URL -- operator opens it in a
browser and completes the auth flow
.. {bush_require}
the host appears in the operator's tailnet admin console
.. {bush_show}
binds the host to the tailnet so the BURN profile's tailnet hostname
(or `100.x.y.z` address) becomes a stable transport address that
persists across reboots

//axhos_step
. *Ensure {burp_privileged_user} has a known password*:
.. {bush_require}
{burp_privileged_user} has a password the operator knows
.. set or verify via:
+
----
sudo passwd <username>
----
.. {bush_show}
required for the operator-manual `ssh-copy-id` step below -- operator
types it once during the first-run exchange; neither this handbook
nor {bust_caparison_linux} ever handles the password

//axhos_step
. *First-run `ssh-copy-id`*:
.. from the operator's station, install the operator's public key on
the host:
+
----
ssh-copy-id -i <pubkey-path> {burp_privileged_user}@<host>
----
.. {bush_show}
operator types the {burp_privileged_user} password once when
prompted -- last point at which a password traverses the wire
.. {bush_require}
subsequent key-auth ssh as {burp_privileged_user} succeeds without
a password prompt

//axhos_step
. *Configure sudoers NOPASSWD entry*:
.. {bush_warn}
syntax mistakes in the sudoers file can lock the operator out of
admin -- `visudo` validates before saving, but review the line
carefully before exiting the editor
.. edit sudoers:
+
----
sudo visudo
----
.. add the line:
+
----
<username> ALL=(ALL) NOPASSWD: ALL
----
.. {bush_require}
`sudo -n true` as {burp_privileged_user} returns success without
prompting

//axhog_guarantee
On completion:

* No remote contact occurred and no local file was written by the
  handbook itself.
* openssh-server is installed and `sshd` is on the system path.
* Tailscale is installed and the host is authenticated to the
  operator's tailnet.
* {burp_privileged_user} has a known password and the operator's
  public key is installed via `ssh-copy-id`.
* Sudoers NOPASSWD entry for {burp_privileged_user} is in place.
* The operator-managed posture scope that {bust_caparison_linux}
  and {bust_invigilate_linux} depend on is established.
